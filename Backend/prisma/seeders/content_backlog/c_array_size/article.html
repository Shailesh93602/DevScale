Understanding Array Size in C

Managing the size of arrays is critical in C because arrays are fixed-size in memory and the language does not perform automatic 'bounds checking'.

1. Fixed-Size Declaration

When you declare an array, you must specify its size: int scores[10];. This size is fixed at compile-time and cannot be changed during the program's execution.

2. Calculating Size with sizeof

You can calculate the number of elements in an array by dividing the total memory size of the array by the size of one element:

int size = sizeof(myArray) / sizeof(myArray[0]);

3. The Dangers of Going "Out of Bounds"

If you try to access an index larger than the array's size (e.g., myArray[20] for an array of size 10), C will not warn you. Instead, it will access whatever data is in that memory location, potentially causing crashes or data corruption.

Conclusion

Being rigorous about array size and boundaries is a hallmark of an expert C developer, ensuring your programs are both efficient and bug-free.
