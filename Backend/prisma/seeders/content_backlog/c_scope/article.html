Understanding Scope in C

Scope defines the region of a program where a particular variable is 'visible' and can be accessed or modified.

1. Local Scope

Variables declared inside a function (or any block {}) have local scope. They are created when the block is entered and destroyed when it's exited. Other functions cannot access them.

2. Global Scope

Variables declared outside all functions (typically at the top of a file) have global scope. They can be accessed and modified by any function within the same file (and sometimes other files) throughout the entire program's execution.

3. Static Variables

By using the static keyword, a local variable can 'remember' its value between function calls, even though it's still only accessible within that specific function.

Conclusion

Managing scope is essential for preventing variable name collisions and ensuring your data is only accessible where it's truly needed, a key principle of the 'encapsulation' we saw in OOP.
