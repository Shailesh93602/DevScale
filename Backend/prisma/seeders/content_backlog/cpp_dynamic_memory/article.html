Deep Dive into Dynamic Memory (The Heap)

While standard variables live on the "Stack", C++ allows you to request memory from the "Heap" at runtime. This is called Dynamic Memory Allocation.

1. The new and delete Keywords

  • new : Requests a block of memory from the system and returns its address.
  • delete : Returns that memory back to the system.

2. Why use Dynamic Memory?

You use the heap when you don't know exactly how much memory you'll need until the program is already running—like loading a file of unknown size or building a complex tree structure.

3. The Danger of Memory Leaks

If you use new but forget to use delete, that memory remains "occupied" even after your program is finished. Professional C++ developers use Smart Pointers to automate this cleanup.

Conclusion

Dynamic memory is what gives C++ its legendary flexibility and power. Understanding the "Heap" is the final step in moving from a beginner to a true systems engineer.
