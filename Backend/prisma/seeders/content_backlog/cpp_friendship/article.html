Friend Functions and Classes

In C++, Encapsulation usually prevents outside functions from accessing a class's private data. However, sometimes two classes need to work very closely together—this is where "Friends" come in.

1. Breaking the Wall Safely

A class can declare a function (or another class) as its friend. This gives that specific outside code full access to all its private and protected members.

2. Syntax

Inside the class definition, you simply add: friend void someFunction(); or friend class OtherClass;.

3. Use Sparingly

Friendship is one-way (if A is a friend of B, B is not automatically a friend of A). Use it only when two classes are logically inseparable, like a Database class and its Record class.

Conclusion

Friend functions provide a strategic "backdoor" to encapsulation, allowing for high-performance integrations without making your private data public to the entire world.
