Understanding Database Migration

Database migration refers to the process of updating a database's schema (tables, columns, etc.) in a version-controlled and reversible manner.

1. Schema Evolution as Code

Modern applications change frequently. Database migrations treat these changes as Migrations as Code. Each change is a script that is tracked and applied automatically across different development, testing, and production environments.

2. Up and Down migrations

  • Up Migration: The script that applies the change (e.g., creating a new table).
  • Down Migration: The script that reverses the change, allowing you to 'roll back' to a previous version if something goes wrong.

3. Popular Tools

Common migration frameworks include Liquibase, Flyway, and built-in tools within web frameworks, such as Prisma (Node.js), Rails Migrations (Ruby), and Alembic (Python).

Conclusion

Database migration is an essential part of CI/CD, ensuring that your database and application code always stay in sync as they evolve together.
