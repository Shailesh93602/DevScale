Understanding Disjoint Set (Union-Find)

A Disjoint Set data structure, also called a Union-Find data structure, keeps track of a set of elements partitioned into a number of disjoint (non-overlapping) subsets.

1. Core Operations

  • Find: Determine which subset a particular element is in. This can be used for determining if two elements are in the same subset (e.g., to detect cycles in a graph).
  • Union: Join two subsets into a single subset.

2. Optimizations

To keep the operations efficient (near constant time), two main optimizations are used: Path Compression (making nodes point directly to the root during Find) and Union by Rank/Size (attaching the smaller tree to the larger one).

3. Applications

Disjoint Sets are most famously used in Kruskal's Algorithm for MST. They are also useful for keeping track of connected components in a social network or an image.

Conclusion

The Disjoint Set is a remarkably efficient and simple data structure for managing partitions of a set, forming a core component of many advanced graph algorithms.
