Introduction to Mongoose Middleware

Middleware (also called pre and post hooks) are functions which are passed control during execution of asynchronous functions at the schema level.

1. Types of Middleware

  • Pre Middlewares: Executed before a specific operation (e.g., hash a password before saving).
  • Post Middlewares: Executed after a specific operation (e.g., send an email after a user is created).

2. Pre-save Pre-middleware

This is commonly used for hashing passwords before they are persisted to the database.


userSchema.pre('save', async function(next) {
  if (this.isModified('password')) {
    this.password = await bcrypt.hash(this.password, 10);
  }
  next();
});

3. Document Middleware vs Query Middleware

Document middleware is executed on a document instance (like save() or remove()), while Query middleware is executed on a query (like find() or update()).

4. The next() Function

Middleware functions must call next() to pass control to the next middleware in the stack or the operation itself. If it doesn't call next(), the operation will hang.

Conclusion

Mongoose middleware is a powerful way to handle cross-cutting concerns like logging, validation, and data transformation at the model level, keeping your application logic clean and consistent.
