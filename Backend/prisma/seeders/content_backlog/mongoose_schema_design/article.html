Advanced Mongoose Schema Design

Mongoose schemas are the blueprint for your data. Effective schema design goes beyond simple type definitions to include validation, default values, and sophisticated structures.

1. Schema Types and Validation

Mongoose provides a wide range of built-in types and allows you to add custom validation logic to ensure data integrity.


const userSchema = new Schema({
  username: {
    type: String,
    required: [true, 'Username is required'],
    unique: true,
    trim: true,
    minlength: 3
  },
  email: {
    type: String,
    required: true,
    match: [/^\S+@\S+\.\S+$/, 'Please use a valid email address']
  },
  role: {
    type: String,
    enum: ['user', 'admin'],
    default: 'user'
  }
});

2. Subdocuments and Nested Data

You can embed schemas within other schemas to create hierarchical data structures, which is a powerful feature of document-based databases like MongoDB.


const addressSchema = new Schema({
  street: String,
  city: String
});

const userSchema = new Schema({
  name: String,
  address: addressSchema // Subdocument
});

3. Virtuals

Virtuals are properties that are not persisted to the database but can be computed on the fly from other fields.


userSchema.virtual('fullName').get(function() {
  return `${this.firstName} ${this.lastName}`;
});

4. Schema Options

Parameters like timestamps: true automatically add createdAt and updatedAt fields to your models.

Conclusion

Thoughtful schema design is the foundation of a robust and scalable application. By leveraging validation, subdocuments, and virtuals, you can create expressive and efficient data models in Mongoose.
