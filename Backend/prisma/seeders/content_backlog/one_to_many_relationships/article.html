Understanding One-to-Many Relationships

The One-to-Many (1:N) relationship is the most common relationship in database system design. It occurs when a single record in one table is associated with many records in another table.

1. Real-World Example

A classic example is an Author and their Books. An author can write many books, but each book typically has one primary author.

2. Modeling in SQL

Use a foreign key on the 'Many' side of the relationship to store the primary key of the matching record from the 'One' side.


CREATE TABLE books (
  id INT PRIMARY KEY,
  title VARCHAR(255),
  author_id INT, -- Foreign Key
  FOREIGN KEY (author_id) REFERENCES authors(id)
);

3. Implementation in ORMs

  • Sequelize: Uses hasMany on the parent model and belongsTo on the child model.
  • Mongoose: Can be achieved through embedding (storing an array of subdocuments) or referencing (storing IDs of children in an array or parent's ID in each child).

4. When to Use

Use it whenever an entity owns or belongs to multiple other entities, such as Users and their Posts, Categories and their Products, or Folders and their Files.

Conclusion

Mastering One-to-Many relationships allows you to build sophisticated data systems that accurately reflect real-world hierarchies.
