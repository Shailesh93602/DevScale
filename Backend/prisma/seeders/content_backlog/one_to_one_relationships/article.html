Understanding One-to-One Relationships

A One-to-One (1:1) relationship occurs when a single record in one table is associated with exactly one record in another table, and vice-versa.

1. Real-World Example

Consider a User and their UserProfile. Each user has exactly one profile, and each profile belongs to exactly one user.

2. Modeling in SQL

In a relational database, this is usually implemented by adding a foreign key to one of the tables and ensuring it is unique. For example, the user_profiles table could have a user_id column that is both a foreign key and has a unique constraint.


CREATE TABLE user_profiles (
  id INT PRIMARY KEY,
  user_id INT UNIQUE, -- Ensures 1:1 relationship
  bio TEXT,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

3. Implementation in ORMs

  • Sequelize: Uses hasOne and belongsTo associations.
  • Mongoose: Can be achieved by embedding the profile document directly into the user document or by using a reference with a unique constraint.

4. When to Use

We use 1:1 relationships to split large tables into smaller ones, keep sensitive metadata separate, or for optional data that isn't required for every record.

Conclusion

One-to-One relationships are the simplest form of database connection, providing a clean way to organize related data across multiple entities.
