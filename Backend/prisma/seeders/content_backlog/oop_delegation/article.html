Understanding Delegation

Delegation is an object-oriented design pattern that allows an object to forward or delegate a task to another helper object, known as the delegate.

1. Composition + Redirection

Delegation is effectively composition in action. Instead of performing a task itself, the parent object calls a method on its internal component to do the work. It says: "I don't know how to do this, but my helper does."

2. Why use Delegation?

It is a powerful alternative to inheritance. Delegation allows for greater flexibility because you can change the delegate at runtime, essentially changing the object's behavior dynamically.

3. Real-World Example

In many UI frameworks, a Table object delegates the task of 'getting data' or 'handling clicks' to a TableDelegate object. The table knows when to act, but not how to handle specific app logic.

Conclusion

Delegation is a core pattern for building flexible and decoupled systems, avoiding the rigidity that often comes with deep inheritance hierarchies.
