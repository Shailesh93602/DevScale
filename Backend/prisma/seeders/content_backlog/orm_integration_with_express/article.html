Integrating Sequelize and Mongoose with Express.js

Integrating an ORM or ODM with Express is a common pattern for building robust and scalable web applications.

1. Middleware Pattern

Both Sequelize and Mongoose suggest initializing the database connection once during application startup and making the models available throughout the app.

2. Folder Structure

A typical project structure includes a models/ directory to store your Sequelize or Mongoose schemas and configurations.


project/
  models/
    index.js   (Initialize database here)
    user.js    (User model)
    post.js    (Post model)
  routes/
    users.js   (User routes using models)
  app.js       (Imports models/index.js)

3. Using Models in Routes


// routes/users.js
const { User } = require('../models');

router.get('/:id', async (req, res) => {
  try {
    const user = await User.findByPk(req.params.id);
    if (!user) return res.status(404).send('User not found');
    res.json(user);
  } catch (error) {
    res.status(500).send(error.message);
  }
});

4. Connection Management

It's important to properly handle connection errors and ensure that the server doesn't start until the database is successfully connected.

Conclusion

By organizing your project with a clean models/ directory and following best practices for connection management, you can build powerful and maintainable Express applications with Sequelize or Mongoose.
