OWASP Top 10: Broken Access Control (BAC)

Ranked #1 on the OWASP Top 10, Broken Access Control is the most common and dangerous vulnerability in modern web applications. It occurs when a user can access data or functions they shouldn't.

1. Insecure Direct Object References (IDOR)

This happens when an app uses a simple ID in a URL (e.g., /api/user/123/profile). An attacker can simply change 123 to 124 to see another user's private data. **Always verify ownership** on the backend before returning data.

2. Vertical and Horizontal Privilege Escalation

  • Horizontal: User A accessing User B's data (same role).
  • Vertical: A standard User accessing Admin functions (different roles).

3. Principle of Least Privilege

The core solution to BAC. Every user and service should only have the absolute minimum permissions required to do their job. Deny everything by default, and only allow what is necessary.

Conclusion

Mastering Access Control is the single most important step in securing an application. By implementing strong, centralized authorization logic, you prevent the vast majority of data breaches before they even start.
