Understanding Sequelize Associations

In relational databases, tables are linked together by relationships. In Sequelize, these relationships are known as associations. There are four main types of associations in Sequelize:

1. The Four Types of Associations

  • HasOne: One-to-One relationship where the foreign key is defined on the target model.
  • BelongsTo: One-to-One or Many-to-One relationship where the foreign key is defined on the source model.
  • HasMany: One-to-Many relationship where the foreign key is defined on the target model.
  • BelongsToMany: Many-to-Many relationship where a join table is used to link models.

2. One-to-One Relationship


// Profile belongs to User, and User has one Profile
User.hasOne(Profile);
Profile.belongsTo(User);

3. One-to-Many Relationship


// User has many Posts
User.hasMany(Post);
Post.belongsTo(User);

4. Many-to-Many Relationship


// User can have many Projects, and Projects can have many Users
const UserProjects = sequelize.define('UserProjects', {}); // Join Table
User.belongsToMany(Project, { through: UserProjects });
Project.belongsToMany(User, { through: UserProjects });

5. Eager Loading with Associations

Eager loading allows you to include associated models in your queries using the include property.


const users = await User.findAll({
  include: Profile
});

Conclusion

Associations allow you to model complex real-world data structures and perform powerful queries that span across multiple tables. Mastering Sequelize associations is critical for any developer working on data-heavy applications.
