Getting Started with Sequelize

Sequelize is a powerful, promise-based Node.js ORM for Postgres, MySQL, MariaDB, SQLite, and Microsoft SQL Server. It features solid transaction support, relations, eager and lazy loading, read replication, and more.

1. Installation

To use Sequelize, you first need to install it and the driver for your database of choice.


npm install sequelize
# For PostgreSQL
npm install pg pg-hstore
# For MySQL
npm install mysql2
# For SQLite
npm install sqlite3

2. Connecting to a Database

You can connect to a database by creating a new Sequelize instance and passing the connection URI or options.


const { Sequelize } = require('sequelize');

const sequelize = new Sequelize('database', 'username', 'password', {
  host: 'localhost',
  dialect: 'postgres' // or 'mysql', 'sqlite', etc.
});

3. Defining Models

Models are the essence of Sequelize. A model is an abstraction that represents a table in your database. Models are defined using sequelize.define or by extending the Model class.


const User = sequelize.define('User', {
  firstName: {
    type: Sequelize.STRING,
    allowNull: false
  },
  lastName: {
    type: Sequelize.STRING
  }
});

4. Synchronization

You can synchronize your models with the database using sequelize.sync(). Note that in production, you should use migrations instead of sync.

Conclusion

Sequelize Basics provide the foundation for interacting with relational databases in a Node.js environment. Once you've mastered connections and model definition, you can move on to more advanced features like associations and querying.
