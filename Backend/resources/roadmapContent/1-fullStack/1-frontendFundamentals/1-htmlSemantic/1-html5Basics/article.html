The Foundation: HTML5 Basics

HTML (HyperText Markup Language) is the backbone of every web page. It defines the structure and meaning of content. Other technologies — CSS for styling and JavaScript for behaviour — build on top of it.

The Standard Boilerplate

Every HTML5 document starts with the same skeleton:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello World</h1>
  </body>
</html>

Key Head Elements

  • <meta charset="UTF-8"> — Ensures text renders correctly in all languages.
  • <meta name="viewport"> — Critical for mobile-responsive design.
  • <title> — Appears in the browser tab and Google search results.

Semantic HTML

Semantic elements convey meaning, not just appearance. They improve both accessibility and SEO:

<header>  — Site-wide header / navigation
<nav>     — Navigation links
<main>    — Primary page content (one per page)
<section> — Thematic grouping of content
<article> — Self-contained piece of content (blog post, news item)
<aside>   — Sidebar / supplementary content
<footer>  — Site-wide or section footer

Block vs Inline Elements

Block elements take the full width available and start on a new line:

<div>, <p>, <h1>–<h6>, <ul>, <section>, <article>

Inline elements flow within text without breaking the line:

<span>, <a>, <strong>, <em>, <img>, <code>

Heading Hierarchy

Headings <h1>–<h6> define a document's outline. Use them in order and never skip levels:

<h1>Page Title (one per page)</h1>
  <h2>Major Section</h2>
    <h3>Subsection</h3>
      <h4>Sub-subsection</h4>

Best Practices

  • Always declare <!DOCTYPE html> as the very first line.
  • Use semantic tags over generic <div> whenever possible.
  • Keep exactly one <h1> per page for proper SEO hierarchy.
  • Add alt attributes to all <img> tags for accessibility.
  • Always specify the lang attribute on <html>.
