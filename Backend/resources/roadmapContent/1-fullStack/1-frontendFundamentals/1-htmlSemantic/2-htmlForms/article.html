HTML Forms & User Input

Forms are how users interact with your web application — login pages, search bars, checkout flows, contact forms. Mastering HTML forms is fundamental for every web developer.

The Form Element

<form action="/submit" method="POST">
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="password">Password:</label>
  <input type="password" id="password" name="password" minlength="8" required>

  <button type="submit">Sign In</button>
</form>

Essential Input Types

  • type="text" — General single-line text
  • type="email" — Browser validates email format automatically
  • type="password" — Masks the input characters
  • type="number" — Numeric input with min/max/step support
  • type="checkbox" — Boolean toggle
  • type="radio" — Single selection from a group
  • type="file" — File upload
  • type="date" — Browser date picker
  • type="range" — Slider
  • type="color" — Color picker
  • type="hidden" — Data submitted but not visible to user

HTML5 Validation Attributes

<input type="text"
  required               <!-- field cannot be empty -->
  minlength="3"          <!-- minimum character count -->
  maxlength="50"         <!-- maximum character count -->
  pattern="[A-Za-z]+"   <!-- regex pattern to match -->
  placeholder="Username">

The Label Element

Always pair every input with a <label> using the for attribute. This is essential for accessibility and also increases the clickable area:

<label for="username">Username:</label>
<input type="text" id="username" name="username">

Dropdown, Textarea & Fieldset

<!-- Dropdown -->
<select name="country" id="country">
  <option value="">Select a country</option>
  <option value="in">India</option>
  <option value="us">United States</option>
</select>

<!-- Multi-line text -->
<textarea name="bio" rows="4" placeholder="Tell us about yourself..."></textarea>

<!-- Group related fields -->
<fieldset>
  <legend>Preferred Contact Method</legend>
  <input type="radio" id="email-opt" name="contact" value="email">
  <label for="email-opt">Email</label>
  <input type="radio" id="phone-opt" name="contact" value="phone">
  <label for="phone-opt">Phone</label>
</fieldset>

Best Practices

  • Always use method="POST" for sensitive data — GET puts data in the URL.
  • Pair every input with a <label>.
  • Use HTML5 validation as a first layer — always validate on the server too.
  • Add autocomplete attributes to help password managers.
  • Use novalidate on the form if you handle all validation with JavaScript.
