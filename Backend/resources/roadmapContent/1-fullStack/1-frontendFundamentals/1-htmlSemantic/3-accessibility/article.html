Web Accessibility (a11y)

Accessibility ensures your website can be used by everyone — including people with visual, motor, auditory, or cognitive disabilities. It is also a legal requirement in many jurisdictions (ADA, WCAG, Section 508).

Why Accessibility Matters

  • Over 1 billion people worldwide have some form of disability.
  • Improves SEO — search engines and screen readers process HTML the same way.
  • Increases usability for everyone (e.g., keyboard navigation, high contrast).
  • Required by law in many countries for public-facing websites.

WCAG Core Principles (POUR)

  • Perceivable — All content can be seen or heard by all users.
  • Operable — All functionality works via keyboard alone.
  • Understandable — Content and UI are clear and predictable.
  • Robust — Compatible with current and future assistive technologies.

Practical Techniques

<!-- 1. Always add alt text to images -->
<img src="chart.png" alt="Bar chart showing Q4 revenue growth of 23%">
<img src="decorative.png" alt=""> <!-- empty alt for decorative images -->

<!-- 2. aria-label for icon-only buttons -->
<button aria-label="Close dialog">×</button>

<!-- 3. Skip navigation link for keyboard users -->
<a href="#main-content" class="skip-link">Skip to main content</a>

<!-- 4. Announce dynamic content changes -->
<div aria-live="polite" id="status"></div>

<!-- 5. Describe input purpose -->
<input type="text" aria-describedby="username-hint">
<p id="username-hint">Must be 8–20 characters, letters and numbers only.</p>

ARIA Landmark Roles

ARIA roles help screen readers understand the purpose of page regions:

role="banner"         — Site header
role="navigation"     — Nav landmark
role="main"           — Main content
role="complementary"  — Sidebar/aside
role="contentinfo"    — Footer
role="search"         — Search form

ARIA State Attributes

<button aria-expanded="false" aria-controls="dropdown-menu">Menu</button>
<div id="dropdown-menu" aria-hidden="true">...</div>

<input type="checkbox" aria-checked="true">
<div role="alert" aria-live="assertive">Error: Form invalid</div>

Keyboard Navigation

All interactive elements must be reachable via the Tab key in a logical order. Never remove the :focus outline without providing a visible alternative:

/* Never do this: */
:focus { outline: none; }

/* Instead, customize it: */
:focus-visible {
  outline: 2px solid #6366f1;
  outline-offset: 2px;
}

Colour Contrast

WCAG AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text. Use tools like WebAIM Contrast Checker to verify your colour choices.
