Links, Images & Media in HTML

Beyond text, every modern web page relies on links, images, and multimedia. Mastering these elements correctly affects performance, accessibility, and user experience.

Anchor Tags — The Backbone of the Web

The <a> element creates hyperlinks. Its key attributes:

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Visit Example</a>
<a href="#section-id">Jump to Section</a>  <!-- internal anchor -->
<a href="mailto:hi@example.com">Email Us</a>
<a href="tel:+911234567890">Call Us</a>
<a href="/file.pdf" download>Download PDF</a>

Critical: Always add rel="noopener noreferrer" when using target="_blank" to prevent tab-napping security vulnerabilities.

Images — The <img> Element

The <img> element is a void element (no closing tag) that embeds images:

<img
  src="hero.webp"
  alt="A developer coding at a desk"
  width="800"
  height="450"
  loading="lazy"
/>
  • alt — Required for accessibility and SEO. Describe the image content, not its appearance.
  • width & height — Prevents Cumulative Layout Shift (CLS), a Core Web Vital metric.
  • loading="lazy" — Defers off-screen images, improving page load speed.

Responsive Images with <picture>

The <picture> element lets browsers choose the best image format and size:

<picture>
  <source media="(min-width: 1024px)" srcset="hero-lg.webp" type="image/webp">
  <source media="(min-width: 768px)"  srcset="hero-md.webp" type="image/webp">
  <img src="hero-sm.jpg" alt="Hero image" width="400" height="300" loading="lazy">
</picture>

Modern image formats in order of preference: AVIF > WebP > PNG/JPEG. Always provide a fallback <img>.

The <figure> and <figcaption> Elements

Use <figure> to associate an image with its caption semantically:

<figure>
  <img src="chart.png" alt="Bar chart showing revenue growth 2020-2024">
  <figcaption>Figure 1: Revenue growth over four years.</figcaption>
</figure>

Audio & Video

HTML5 provides native media elements without requiring Flash:

<!-- Video -->
<video controls width="720" poster="thumbnail.jpg" preload="metadata">
  <source src="video.webm" type="video/webm">
  <source src="video.mp4"  type="video/mp4">
  <p>Your browser doesn't support HTML video. <a href="video.mp4">Download it</a>.</p>
</video>

<!-- Audio -->
<audio controls>
  <source src="podcast.ogg" type="audio/ogg">
  <source src="podcast.mp3" type="audio/mpeg">
</audio>
  • controls — Shows native browser playback controls.
  • poster — Thumbnail shown before the video plays.
  • preload="metadata" — Only loads duration/dimensions, not the full video (better performance).
  • Always include multiple <source> formats for browser compatibility.

Iframes — Embedding External Content

<iframe
  src="https://www.youtube.com/embed/dQw4w9WgXcQ"
  title="YouTube video player"
  width="560"
  height="315"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media"
  allowfullscreen
  loading="lazy">
</iframe>

SVG Inline vs. External

SVGs can be embedded directly in HTML for maximum styling control:

<!-- Inline SVG (can be styled with CSS) -->
<svg width="24" height="24" viewBox="0 0 24 24" aria-hidden="true">
  <path d="M12 2L2 7l10 5 10-5-10-5z"/>
</svg>

<!-- External SVG (simpler but less styleable) -->
<img src="icon.svg" alt="Stack icon" width="24" height="24">

Best Practices Summary

  • Always add meaningful alt text; use alt="" only for decorative images.
  • Prefer WebP/AVIF over JPEG/PNG for smaller file sizes.
  • Specify width and height on all <img> tags to prevent layout shift.
  • Use loading="lazy" on images below the fold.
  • Never open external links without rel="noopener noreferrer".
