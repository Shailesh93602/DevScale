HTML Tables & Tabular Data

HTML tables are designed for one purpose: displaying tabular data — data that logically belongs in rows and columns. Never use tables for layout (use CSS Grid or Flexbox for that).

Basic Table Structure

<table>
  <caption>Monthly Revenue by Product</caption>
  <thead>
    <tr>
      <th scope="col">Product</th>
      <th scope="col">January</th>
      <th scope="col">February</th>
      <th scope="col">March</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Widget A</th>
      <td>$1,200</td>
      <td>$1,450</td>
      <td>$1,800</td>
    </tr>
    <tr>
      <th scope="row">Widget B</th>
      <td>$850</td>
      <td>$920</td>
      <td>$1,100</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">Total</th>
      <td>$2,050</td>
      <td>$2,370</td>
      <td>$2,900</td>
    </tr>
  </tfoot>
</table>

Key Table Elements

  • <table> — The container element.
  • <caption> — A title for the table, placed immediately after <table>. Critical for accessibility.
  • <thead> — Groups header rows. Browsers may repeat this when printing long tables.
  • <tbody> — The main body of data rows.
  • <tfoot> — Summary/total rows at the bottom.
  • <tr> — Table row.
  • <th> — Table header cell. Rendered bold by default.
  • <td> — Table data cell.

The scope Attribute (Accessibility)

The scope attribute tells screen readers which cells a header applies to:

<th scope="col">Month</th>   <!-- applies to the column below -->
<th scope="row">Total</th>  <!-- applies to the row to the right -->
<th scope="colgroup">Q1</th> <!-- applies to a group of columns -->

Colspan and Rowspan

These attributes make cells span multiple columns or rows:

<table>
  <thead>
    <tr>
      <th rowspan="2">Product</th>
      <th colspan="3">Quarter 1</th>
    </tr>
    <tr>
      <th>Jan</th>
      <th>Feb</th>
      <th>Mar</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Widget</td>
      <td>100</td>
      <td>120</td>
      <td>140</td>
    </tr>
  </tbody>
</table>

Column Grouping with <colgroup>

<colgroup> lets you apply styles to entire columns without repeating class names on every <td>:

<table>
  <colgroup>
    <col style="background-color: #f0f4ff;">
    <col span="3" style="background-color: #fff7ed;">
  </colgroup>
  <thead>...</thead>
  <tbody>...</tbody>
</table>

Making Tables Responsive

Wide tables break on mobile. Wrap them in a scrollable container:

<div style="overflow-x: auto; -webkit-overflow-scrolling: touch;">
  <table>
    ...
  </table>
</div>

Best Practices

  • Always include <caption> to describe the table's purpose.
  • Use scope attribute on all <th> elements for screen readers.
  • Never use tables for page layout — use CSS Grid/Flexbox.
  • For large tables, use <thead> so headers repeat when printing.
  • Wrap tables in a scrollable container for mobile responsiveness.
