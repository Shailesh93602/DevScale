SEO & Meta Tags in HTML

Search Engine Optimization (SEO) begins in the <head> of your HTML document. The right meta tags influence how search engines index your page and how your content appears in search results and social media previews.

Essential Meta Tags

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Best Running Shoes 2024 | ShoeStore</title>
  <meta name="description" content="Discover the top 10 running shoes of 2024 for all budgets. Expert reviews, buyer's guide, and price comparisons.">
  <meta name="robots" content="index, follow">
  <link rel="canonical" href="https://example.com/running-shoes">
</head>
  • title — Most important SEO tag. Keep under 60 characters. Place primary keyword first.
  • description — Shown in search results. Keep 150-160 characters. Write to entice clicks.
  • robots — Controls crawler behaviour. Use noindex for pages you don't want indexed.
  • canonical — Prevents duplicate content issues by specifying the preferred URL.

Open Graph Tags (Social Media)

Open Graph tags control how your page appears when shared on Facebook, LinkedIn, and WhatsApp:

<meta property="og:title" content="Best Running Shoes 2024">
<meta property="og:description" content="Top 10 running shoes reviewed by experts.">
<meta property="og:image" content="https://example.com/og-running-shoes.jpg">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta property="og:url" content="https://example.com/running-shoes">
<meta property="og:type" content="article">
<meta property="og:site_name" content="ShoeStore">

OG image recommendation: 1200×630px. This is shown as a large preview card on most platforms.

Twitter Card Tags

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@ShoeStore">
<meta name="twitter:creator" content="@AuthorHandle">
<meta name="twitter:title" content="Best Running Shoes 2024">
<meta name="twitter:description" content="Top 10 running shoes reviewed by experts.">
<meta name="twitter:image" content="https://example.com/twitter-running-shoes.jpg">

Card types: summary (small image), summary_large_image (large banner), app (app install), player (video).

Structured Data with JSON-LD

JSON-LD is Google's preferred format for structured data markup. It enables rich results (star ratings, FAQs, breadcrumbs, etc.) in search:

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Best Running Shoes 2024",
  "author": {
    "@type": "Person",
    "name": "Jane Smith"
  },
  "datePublished": "2024-01-15",
  "dateModified": "2024-03-01",
  "image": "https://example.com/og-running-shoes.jpg",
  "publisher": {
    "@type": "Organization",
    "name": "ShoeStore",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.png"
    }
  }
}
</script>

<link rel="apple-touch-icon" href="/apple-touch-icon.png">

Hreflang for International Sites

If your site serves multiple languages, tell search engines which language/region each page targets:

<link rel="alternate" hreflang="en" href="https://example.com/en/">
<link rel="alternate" hreflang="fr" href="https://example.com/fr/">
<link rel="alternate" hreflang="x-default" href="https://example.com/">

Best Practices Summary

  • Every page must have a unique <title> and <meta name="description">.
  • Use <link rel="canonical"> to avoid duplicate content penalties.
  • Always include Open Graph tags for better social sharing previews.
  • Add JSON-LD structured data to qualify for rich results in Google.
  • Test with Google's Rich Results Test and Facebook's Sharing Debugger.
