The CSS Box Model

Every HTML element is a rectangular box. The box model defines how its total size is calculated — understanding it is fundamental to controlling layout and spacing.

The Four Layers

┌──────────────────────────────┐
│           Margin             │  ← Space outside the border
│   ┌─────────────────────┐   │
│   │       Border        │   │  ← The visible outline
│   │  ┌───────────────┐  │   │
│   │  │    Padding    │  │   │  ← Space inside the border
│   │  │  ┌─────────┐  │  │   │
│   │  │  │ Content │  │  │   │  ← Text, image, etc.
│   │  │  └─────────┘  │  │   │
│   │  └───────────────┘  │   │
│   └─────────────────────┘   │
└──────────────────────────────┘

box-sizing: border-box

By default (content-box), width/height only measures the content area. Padding and border are added on top of that, making sizing confusing. Use border-box globally:

*,
*::before,
*::after {
  box-sizing: border-box; /* Padding & border are INCLUDED in width/height */
}

CSS Selectors Cheatsheet

/* Type selector */
p { color: gray; }

/* Class selector */
.card { background: white; }

/* ID selector (use sparingly) */
#hero { font-size: 2rem; }

/* Attribute selector */
input[type="email"] { border-color: blue; }

/* Pseudo-class */
a:hover { text-decoration: underline; }
li:nth-child(2n) { background: #f5f5f5; }
input:focus { border-color: #6366f1; }

/* Pseudo-element */
p::first-line { font-weight: bold; }
.btn::before { content: "→ "; }

/* Child combinator */
nav > a { margin: 0 8px; }

/* Adjacent sibling */
h2 + p { margin-top: 0; }

/* General sibling */
h2 ~ p { color: #555; }

/* Descendant */
.card p { font-size: 0.9rem; }

CSS Specificity

When multiple rules target the same element, specificity determines which wins:

  • Inline styles → 1,0,0,0
  • ID selectors → 0,1,0,0
  • Class/attribute/pseudo-class → 0,0,1,0
  • Element/pseudo-element → 0,0,0,1
/* Specificity: 0,0,1,1 */
.card p { color: blue; }

/* Specificity: 0,1,0,0 — wins! */
#hero { color: red; }

The Cascade

When specificity is equal, the last rule wins (cascade). Use this intentionally by ordering your CSS from general to specific.
