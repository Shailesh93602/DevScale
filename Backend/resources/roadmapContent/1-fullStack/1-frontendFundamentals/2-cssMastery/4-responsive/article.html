Responsive Web Design

Responsive design ensures your website looks and works well on every device — from 320px mobile screens to 4K monitors. The strategy: write CSS that adapts to the viewport size.

1. The Viewport Meta Tag (Required)

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Without this, mobile browsers render at ~980px and scale it down, making everything tiny.

2. Media Queries

/* Mobile-first: base styles target small screens */
.container { padding: 16px; font-size: 1rem; }

/* Tablet (768px+) */
@media (min-width: 768px) {
  .container { padding: 32px; }
  .grid { display: grid; grid-template-columns: 1fr 1fr; }
}

/* Desktop (1024px+) */
@media (min-width: 1024px) {
  .container { max-width: 1200px; margin: 0 auto; padding: 64px; }
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Large screens (1280px+) */
@media (min-width: 1280px) {
  .hero-title { font-size: 4rem; }
}

Common Breakpoints

/* Mobile:  < 768px (default, no query needed) */
/* Tablet:  768px - 1023px */
/* Desktop: 1024px - 1279px */
/* Large:   1280px+ */

Mobile-First vs Desktop-First

Mobile-first (recommended): Write base styles for mobile, add complexity for larger screens with min-width. Results in smaller CSS for most users.

Desktop-first: Write base styles for desktop, simplify for smaller screens with max-width.

Fluid Images & Video

img, video { max-width: 100%; height: auto; }

Fluid Typography with clamp()

/* Scales between 1rem (mobile) and 1.5rem (desktop) */
h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
p  { font-size: clamp(1rem, 2.5vw, 1.125rem); }

Container Queries (Modern CSS)

Container queries let you apply styles based on a parent container's size rather than the viewport — ideal for reusable components:

.card-container { container-type: inline-size; }

@container (min-width: 400px) {
  .card { flex-direction: row; }
}
