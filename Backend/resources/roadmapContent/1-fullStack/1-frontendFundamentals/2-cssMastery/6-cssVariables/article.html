CSS Custom Properties (Variables)

CSS Custom Properties — commonly called CSS Variables — allow you to store values in reusable named tokens. They are the foundation of modern, themeable design systems.

Defining and Using Variables

/* Define variables on :root to make them globally available */
:root {
  --color-primary: #6366f1;
  --color-secondary: #8b5cf6;
  --color-text: #1e293b;
  --color-bg: #ffffff;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 2rem;
  --radius-md: 0.5rem;
  --font-body: 'Inter', sans-serif;
}

/* Use variables with var() */
button {
  background-color: var(--color-primary);
  color: var(--color-bg);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius-md);
  font-family: var(--font-body);
}

Fallback Values

Provide a fallback in case a variable is undefined:

color: var(--color-accent, #f43f5e);  /* uses #f43f5e if --color-accent isn't set */

Dark Mode with CSS Variables

This is where CSS variables truly shine — toggle an entire theme by changing a handful of values:

:root {
  --color-bg: #ffffff;
  --color-text: #1e293b;
  --color-surface: #f1f5f9;
}

[data-theme="dark"],
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #0f172a;
    --color-text: #f8fafc;
    --color-surface: #1e293b;
  }
}

body {
  background-color: var(--color-bg);
  color: var(--color-text);
}

Toggle dark mode by setting document.documentElement.setAttribute('data-theme', 'dark') in JavaScript.

Scoped Variables

Variables follow the CSS cascade — you can scope them to specific components:

.card {
  --card-bg: #f8fafc;
  --card-border: 1px solid #e2e8f0;
  background: var(--card-bg);
  border: var(--card-border);
}

.card--highlighted {
  --card-bg: #eff6ff;
  --card-border: 2px solid #6366f1;
  /* background and border automatically update! */
}

Animating CSS Variables with JavaScript

// Update a variable dynamically
document.documentElement.style.setProperty('--scroll-progress', `${scrollPercent}%`);

// Progress bar that updates on scroll
.progress-bar {
  width: var(--scroll-progress, 0%);
  background: var(--color-primary);
  height: 4px;
}

CSS Variables vs. Preprocessor Variables (Sass)

  • Sass variables are compiled away at build time — they can't be changed dynamically.
  • CSS variables are live in the browser — they can be changed with JavaScript, media queries, or pseudo-classes.
  • CSS variables participate in the cascade and can be inherited by child elements.

Building a Design Token System

:root {
  /* Primitive tokens */
  --indigo-500: #6366f1;
  --indigo-600: #4f46e5;
  --slate-900: #0f172a;

  /* Semantic tokens (reference primitives) */
  --color-brand: var(--indigo-500);
  --color-brand-hover: var(--indigo-600);
  --color-text-primary: var(--slate-900);
}

This two-layer system (primitives → semantics) is used by design systems like Material UI, Radix, and Chakra UI.

Best Practices

  • Define all global variables on :root.
  • Use semantic naming (--color-primary not --blue).
  • Always include a fallback value when using var().
  • Use CSS variables for theme tokens — colours, spacing, typography, shadows.
  • Prefer CSS variables over JavaScript style injection for performance.
