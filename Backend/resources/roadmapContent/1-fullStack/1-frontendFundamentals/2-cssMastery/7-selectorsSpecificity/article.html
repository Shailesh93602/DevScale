CSS Selectors & Specificity

Understanding how browsers decide which CSS rule wins in a conflict is fundamental. Two key mechanisms control this: specificity and the cascade.

Selector Types & Their Specificity Weight

Specificity is calculated as a 4-part score: (inline, ID, class/attribute/pseudo-class, element/pseudo-element)

SelectorScoreExample
Inline style1,0,0,0style="color:red"
ID selector0,1,0,0#header
Class selector0,0,1,0.btn
Attribute selector0,0,1,0[type="text"]
Pseudo-class0,0,1,0:hover, :nth-child(2)
Element selector0,0,0,1p, div
Pseudo-element0,0,0,1::before, ::after
Universal selector0,0,0,0*

Calculating Specificity

/* Score: 0,0,1,1 — one class + one element */
.nav a { color: blue; }

/* Score: 0,1,0,0 — one ID */
#main { color: green; }  /* wins over .nav a */

/* Score: 0,0,2,1 — two classes + one element */
.sidebar .link:hover { color: red; }

Advanced Selectors

/* Descendant (any level) */
.card p { }

/* Direct child */
.nav > li { }

/* Adjacent sibling (immediately after) */
h2 + p { }

/* General sibling (all after) */
h2 ~ p { }

/* Attribute selectors */
[type="email"] { }           /* exact match */
[href^="https"] { }          /* starts with */
[href$=".pdf"] { }           /* ends with */
[class*="btn"] { }           /* contains */

/* Pseudo-classes */
:nth-child(odd)    { }       /* every odd li */
:nth-child(3n+1)   { }       /* every 3rd, starting from 1 */
:not(.disabled)    { }       /* exclusion */
:is(h1, h2, h3)    { }       /* group selector (lower specificity) */
:where(h1, h2, h3) { }       /* like :is() but 0 specificity */
:has(img)          { }       /* parent selector! */

/* Pseudo-elements */
p::first-line { }
p::first-letter { font-size: 2em; }
::selection { background: #6366f1; color: white; }

The Cascade Algorithm

When specificity is equal, the cascade resolves conflicts by these rules (in order):

  1. Origin — Browser stylesheet < User stylesheet < Author (your CSS)
  2. Importance — !important overrides all (avoid using it)
  3. Specificity — Higher score wins
  4. Source order — Later rule wins when specificity is equal

The :is() and :where() Pseudo-classes

/* Without :is() — repetitive */
header h1, main h1, footer h1 { color: #1e293b; }

/* With :is() — cleaner. Specificity = specificity of most specific arg */
:is(header, main, footer) h1 { color: #1e293b; }

/* :where() — same, but always 0 specificity (easy to override) */
:where(header, main, footer) h1 { color: #1e293b; }

The :has() Selector — The Parent Selector

/* Style a card differently when it contains an image */
.card:has(img) {
  padding: 0;
}

/* Style a label when its input is checked */
label:has(input:checked) {
  font-weight: bold;
  color: var(--color-primary);
}

Best Practices

  • Keep selectors as short as possible — avoid deeply nested selectors.
  • Avoid using !important — it breaks the natural cascade.
  • Prefer class selectors over element selectors for reusability.
  • Avoid ID selectors in CSS — they have too-high specificity and can't be reused.
  • Use :where() for low-specificity reset/base styles that are easy to override.
