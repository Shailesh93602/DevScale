JavaScript Syntax

JavaScript syntax is the set of rules that define a correctly structured JavaScript program. It describes how to write valid code that a browser or Node.js runtime can understand and execute.

1. Variables and Literals

In JavaScript, we work with two types of values: fixed values (Literals) and variable values (Variables).

  • Literal Numbers: 10.50, 1001.
  • Literal Strings: "John Doe", 'Some Text'.
  • Variables: Used to store data values (e.g., let x = 10;).

2. Case-Sensitivity

JavaScript is case-sensitive. A variable named lastName is different from lastname. This applies to all identifiers, including variable names, function names, and keywords.

3. Comments

Use comments to explain your code and make it more readable for others:

// This is a single-line comment

/*
  This is a 
  multi-line comment
*/

4. Identifiers (Names)

Identifiers are the names we give to variables, functions, and labels. They must follow certain rules:

  • Can start with a letter (A-Z or a-z), a dollar sign ($), or an underscore (_).
  • Subsequent characters can also be digits (0-9).
  • Cannot start with a digit.

5. Statements and Semicolons

A JavaScript program is a list of programming statements. It is a common practice to end statements with a semicolon (;) to clearly separate them, although it is not strictly required in all cases due to Automatic Semicolon Insertion (ASI).

Conclusion

Understanding the basic syntax is the foundation of becoming a proficient JavaScript developer. By following these rules, you'll ensure your code is error-free and easy to maintain.
