JavaScript Placement

Where you place your JavaScript in an HTML document can have a significant impact on performance and user experience. JavaScript can be added to an HTML file in three primary ways: Inline, Internal, and External.

1. Internal and External JS

Internal JavaScript is placed within a <script> tag inside the HTML file. External JavaScript is linked from a separate .js file using the src attribute:

<!-- Internal JS -->
<script>
  console.log("Hello from inside the HTML!");
</script>

<!-- External JS -->
<script src="script.js"></script>

2. Positioning the Script Tag

The position of the <script> tag matters:

  • In the <head> section: This can delay the rendering of the page because the browser must download and execute the script before it continues parsing the HTML.
  • At the end of the <body> section: Usually recommended, as it allows the browser to render the page content first before loading and executing the script.

3. The async and defer Attributes

Modern HTML provides two attributes to optimize external script loading:

  • async: The script is downloaded asychronously while the HTML continues to be parsed, but it will pause parsing to execute as soon as it's ready.
  • defer: The script is downloaded asynchronously while the HTML continues to be parsed, but it will only execute after the HTML has been fully parsed.

Conclusion

For most modern web applications, placing scripts at the end of the <body> or using the defer attribute in the <head> is the best practice for optimal performance and user experience.
