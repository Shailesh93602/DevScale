JavaScript Cookies

Cookies are small pieces of data stored on your computer by the web browser while you are browsing a website. They are primarily used to remember information about the user, such as preferences or login status.

1. Creating and Modifying Cookies

JavaScript can create, read, and delete cookies with the document.cookie property. To create a cookie, simply assign a string to it:

document.cookie = "username=John Doe";

2. Cookie Expiration

By default, a cookie is deleted when the browser is closed. To make it persistent, you can specify an expiration date (in UTC time) or a maximum age in seconds:

// Cookie expires in 1 hour
  document.cookie = "username=John Doe; max-age=3600; path=/";

3. Reading Cookies

When you read document.cookie, it returns all cookies for current page in a single string: cookie1=value; cookie2=value; .... You typically need to parse this string to get a specific value.

let allCookies = document.cookie;
  console.log(allCookies); // username=John Doe; theme=dark

4. Secure and SameSite Flags

Modern web development requires secure handling of cookies:

  • Secure: Cookie is only sent over HTTPS.
  • HttpOnly: Cookie cannot be accessed through JavaScript (prevents XSS attacks). This must be set by the server.
  • SameSite: Controls whether the cookie is sent with cross-site requests (values: Strict, Lax, None).

Conclusion

While modern applications often use LocalStorage or SessionStorage for client-side data, cookies remain essential for session management and server-side tracking. Understanding their lifecycle and security is vital for any web developer.
