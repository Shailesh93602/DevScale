JavaScript Page Redirects

JavaScript can be used to redirect a user to a different URL or to reload the current page. This is commonly used after a certain condition is met, such as after a successful login or a timer.

1. window.location.href

Assigning a new string to window.location.href will cause the browser to navigate to the new URL. This is the simplest way to perform a redirect.

// Simple redirect
  window.location.href = "https://eduscale.io";

2. assign() vs replace()

There are two methods for modifying the current location:

  • window.location.assign(): Loads a new document. The user can still click the browser's "Back" button to return to the current page.
  • window.location.replace(): Replaces the current document with a new one. The current page is removed from the session history, meaning the user cannot go back to it. This is useful for sensitive pages like payment success screens.
// Redirect with replace (history is cleared)
  window.location.replace("https://eduscale.io/success");

3. location.reload()

The location.reload() method forces the browser to reload the current page. You can pass true as an argument to force a server reload, ignoring the cache (though this is deprecated in many modern browsers).

// Reload the current page
  location.reload();

4. Meta Refresh (Alternative)

While JavaScript is powerful, you can also perform redirects using HTML <meta> tags. This is often used as a fallback if JavaScript is disabled.

<meta http-equiv="refresh" content="5;url=https://eduscale.io">

Conclusion

Page redirects are a powerful tool for controlling user flow. By choosing between href, assign, and replace, you can create a smooth and seamless navigation experience for your users.
