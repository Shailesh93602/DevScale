JavaScript Objects

In JavaScript, an object is a standalone entity, with properties and type. Compare it with a cup, for example. A cup is an object, with properties. A cup has a color, a design, weight, a material it is made of, etc. Similarly, JavaScript objects can have properties, which define their characteristics.

1. Creating an Object

The most common way to create an object is using object literals (a comma-separated list of name-value pairs wrapped in curly braces):

const car = {
    make: "Tesla",
    model: "Model 3",
    year: 2022,
    start: function() {
        console.log("Starting engine...");
    }
  };

2. Accessing Object Properties

You can access object properties in two ways:

  • Dot notation: car.make
  • Bracket notation: car["make"] (useful when the property name is stored in a variable)

3. Methods and 'this'

A method is a function stored as an object property. The this keyword refers to the object it belongs to.

const person = {
    firstName: "John",
    lastName: "Doe",
    fullName: function() {
        return this.firstName + " " + this.lastName;
    }
  };

4. Object Methods (Iterating)

  • Object.keys(obj): Returns an array of keys.
  • Object.values(obj): Returns an array of values.
  • Object.entries(obj): Returns an array of [key, value] pairs.

Conclusion

Objects are the fundamental building blocks of JavaScript. Almost everything (except primitives) is an object or behaves like one. Mastering how to structure and manipulate data using objects is crucial for any advanced developer.
