JavaScript Numbers

JavaScript has only one type of number. Numbers can be written with, or without, decimals.

1. Integer and Float

Unlike many other languages, JavaScript doesn't have multiple number types like int, float, or long. All numbers are stored as double precision floating point numbers.

let x = 10;      // integer
  let y = 3.14;    // floating point

2. Special Values: NaN and Infinity

  • NaN (Not-a-Number): A value returned when a mathematical operation fails or an illegal conversion is attempted.
  • Infinity / -Infinity: Values returned if you calculate a number outside the largest possible number.
let x = 10 / "apple"; // x is NaN
  let y = 1 / 0;        // y is Infinity

3. Number Methods

  • toString(): Returns a number as a string.
  • toFixed(n): Returns a string, with the number written with a specified number of decimals.
  • parseInt(): Parses a string and returns a whole number.
  • parseFloat(): Parses a string and returns a floating point number.
  • isInteger(): Checks if a value is an integer.

4. BigInt

Introduced in ES2020, BigInt is a way to represent integers larger than the 64-bit float can handle accurately. Add an n to the end of an integer to make it a BigInt.

let big = 9007199254740991n; // a very large integer

Conclusion

Understanding the single-number-type architecture of JavaScript and its limitations is key to avoiding precision bugs (especially with currency). Always use parsing methods when receiving data from users or APIs to ensure you're working with the right type.
