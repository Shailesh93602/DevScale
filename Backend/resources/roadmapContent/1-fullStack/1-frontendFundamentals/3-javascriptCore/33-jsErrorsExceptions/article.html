JavaScript Errors and Exceptions

In web development, errors are inevitable. JavaScript provides robust mechanisms to handle these errors gracefully using try...catch blocks and the throw keyword.

1. Try...Catch...Finally

The try block allows you to test a block of code for errors. The catch block allows you to handle the error. The finally block executes regardless of the result.

try {
    // Code to test
    let result = riskyOperation();
  } catch (error) {
    // Handle error
    console.error("Oops!", error.message);
  } finally {
    // Code always executed
    cleanup();
  }

2. The Throw Keyword

You can create custom errors using the throw statement. You can throw anything (strings, numbers, booleans, or objects), but it's best to throw an Error object.

function checkAge(age) {
  if (age < 18) {
    throw new Error("Must be 18+");
  }
}

3. Standard Error Types

  • ReferenceError: An illegal reference has occurred.
  • TypeError: A type error has occurred (e.g., calling a non-function).
  • SyntaxError: A syntax error has occurred.
  • RangeError: A number "out of range" has occurred.
  • URIError: An error in encodeURI() has occurred.

Conclusion

Proper error handling ensures that a single mistake doesn't crash your entire application. By catching exceptions and providing meaningful feedback, you can improve both a developer's experience (debugging) and a user's experience (graceful failure).
