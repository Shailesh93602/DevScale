JavaScript Form Validation

Form validation is the process of ensuring that user input is consistent with the expectations of the application. While HTML5 provides basic built-in validation, JavaScript offers much more control and a better user experience.

1. Why Use JavaScript for Validation?

JavaScript provides real-time feedback, custom error messages, and complex logic that cannot be achieved with simple HTML attributes alone. It's often used before a form is submitted to the server to prevent unnecessary network requests.

2. Basic Validation Pattern

function validateForm() {
    let name = document.forms["myForm"]["fname"].value;
    if (name === "") {
        alert("Name must be filled out");
        return false; // Stop form submission
    }
}

3. The Constraint Validation API

Most modern browsers include a built-in Constraint Validation API as part of the HTMLInputElement interface. Key properties and methods include:

  • checkValidity(): Returns true if an input element contains valid data.
  • setCustomValidity(message): Sets a custom message to display if validation fails.
  • validationMessage: Contains the message the browser will display to the user.
  • validity: Returns a ValidityState object with specific details on what failed (e.g., valueMissing, typeMismatch, tooShort).

4. Data Validation and Regex

Regular expressions are frequently used in JavaScript form validation for things like email addresses, phone numbers, and strong passwords.

function isValidEmail(email) {
  let re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return re.test(email);
}

Conclusion

Always remember: client-side validation is NOT a security measure. It is exclusively a usability feature. Because JavaScript can be bypassed, you must always perform robust validation on the server side as well.
