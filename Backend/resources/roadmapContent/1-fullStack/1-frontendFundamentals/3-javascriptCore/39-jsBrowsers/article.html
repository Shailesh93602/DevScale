JavaScript Browser Object Model (BOM)

The Browser Object Model (BOM) allows JavaScript to "talk to" the browser. There are no official standards for the BOM, but modern browsers have implemented almost the same methods and properties for interactivity.

1. The Window Object

The window object is the global object for all browser-based JavaScript. All global variables and functions are members of the window object. Some key properties include:

  • window.innerHeight: The inner height of the browser window (in pixels).
  • window.innerWidth: The inner width of the browser window (in pixels).
  • window.open() / window.close(): Opens a new browser window or tab.

2. The Location Object

The window.location object is used to get the current page address (URL) and to redirect the browser to a new page.

console.log(location.href);     // Full URL
console.log(location.hostname); // Domain name
console.log(location.pathname); // Path of the current page

3. The History Object

The window.history object contains the browsers history. It allows you to move back and forth between previously visited pages.

history.back();    // Same as clicking back
history.forward(); // Same as clicking forward
history.go(-2);    // Go back two pages

4. The Navigator Object

The window.navigator object contains information about the visitor's browser, including the name, version, and language.

  • navigator.userAgent: Returns the user-agent header sent by the browser.
  • navigator.onLine: Returns true if the browser is online.
  • navigator.language: Returns the browser's language setting.

Conclusion

The BOM is what makes JavaScript unique as a web-centric language. By understanding how to interact with the window, location, history, and navigator objects, you can build powerful and aware web applications that respond to the environment they run in.
