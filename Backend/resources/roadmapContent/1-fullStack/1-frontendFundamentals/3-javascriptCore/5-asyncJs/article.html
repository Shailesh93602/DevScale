Async JavaScript — Promises & Async/Await

JavaScript is single-threaded, but handles async operations (network requests, timers, file reading) without blocking via an event loop.

The Problem: Callback Hell

// Hard to read, maintain, and handle errors
fetchUser(id, function(user) {
  fetchPosts(user.id, function(posts) {
    fetchComments(posts[0].id, function(comments) {
      // Deeply nested, error-prone — "Pyramid of Doom"
    });
  });
});

Promises

A Promise is an object representing an eventual value — either resolved (success) or rejected (failure).

const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = true;
    if (success) resolve({ data: "Hello!" });
    else reject(new Error("Network error"));
  }, 1000);
});

fetchData
  .then(result => console.log(result.data))  // "Hello!" after 1s
  .catch(err => console.error(err.message))
  .finally(() => console.log("Request complete"));

Async/Await — Clean Async Code

async function loadDashboard(userId) {
  try {
    const user = await fetchUser(userId);      // Wait for user
    const posts = await fetchPosts(user.id);   // Then fetch posts
    const comments = await fetchComments(posts[0].id);
    return { user, posts, comments };
  } catch (error) {
    console.error("Failed:", error.message);
    throw error; // Re-throw so the caller can handle it
  }
}

Promise Combinators

// Run ALL in parallel — fails fast if any fails
const [user, posts, settings] = await Promise.all([
  fetchUser(id),
  fetchPosts(id),
  fetchSettings(id),
]);

// First one to settle (resolve OR reject) wins
const fastest = await Promise.race([fetchA(), fetchB()]);

// Wait for ALL to settle — gets results including failures
const results = await Promise.allSettled([fetchA(), fetchB()]);
results.forEach(r => {
  if (r.status === "fulfilled") console.log(r.value);
  else console.error(r.reason);
});

Fetching Data from APIs

async function getUser(id) {
  const response = await fetch(`https://api.example.com/users/${id}`, {
    headers: { "Authorization": `Bearer ${token}` }
  });

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  return response.json(); // Also returns a Promise
}

// Usage
const user = await getUser(42);

The Event Loop (How It Works)

  1. Call Stack — executes synchronous code line by line.
  2. Web APIs (browser) handle async tasks (setTimeout, fetch).
  3. Microtask Queue — resolved Promises are queued here (processed FIRST).
  4. Macrotask Queue — setTimeout, setInterval callbacks queued here (processed after microtasks).
