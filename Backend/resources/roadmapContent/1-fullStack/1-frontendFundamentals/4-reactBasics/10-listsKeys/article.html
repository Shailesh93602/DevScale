Lists and Keys in React

In React, you can build collections of elements and include them in JSX by wrapping them in curly braces. Rendering lists of data is a fundamental part of building dynamic web applications.

1. Rendering Multiple Components

To render multiple components from an array of data, use the JavaScript map() function. For each item in the array, you return a React element.

const fruits = ["Apple", "Banana", "Cherry"];
const listItems = fruits.map((fruit) =>
  <li>{fruit}</li>
);

function FruitList() {
  return <ul>{listItems}</ul>;
}

2. The 'key' Prop

When rendering a list of elements, React requires a special key prop for each item. Keys help React identify which items have changed, been added, or been removed, which is crucial for high-performance DOM updates.

const todoItems = todos.map((todo) =>
  <li key={todo.id}>
    {todo.text}
  </li>
);

3. Why Keys Must Be Unique?

Keys must be unique among siblings to ensure that React can correctly track individual elements. Using the array index as a key is generally discouraged if the order of items may change (e.g., sorting, adding/removing), as it can lead to UI bugs and performance issues.

Conclusion

Lists and keys go hand-in-hand in React. By using map() and providing unique keys, you can efficiently and accurately render large amounts of dynamic data in your user interface while maintaining React's optimal performance.
