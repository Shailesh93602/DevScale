Forms in React

In HTML, form elements like <input>, <textarea>, and <select> typically maintain their own state. In React, mutable state is usually kept in the component's state. We can combine these two by making the React state the "single source of truth." This is called a Controlled Component.

1. Controlled Components

To create a controlled component, you wire up an input's value to a piece of React state and its onChange handler to a function that updates that state.

function NameForm() {
  const [value, setValue] = useState("");

  const handleChange = (event) => {
    setValue(event.target.value);
  };

  const handleSubmit = (event) => {
    alert("A name was submitted: " + value);
    event.preventDefault();
  };

  return (
    <form onSubmit={handleSubmit}>
      <input type="text" value={value} onChange={handleChange} />
      <button type="submit">Submit</button>
    </form>
  );
}

2. Handling Multiple Inputs

If you have several inputs in a form, you can give each one a name attribute and use a single handler with computed property names to update the state.

const handleInputChange = (e) => {
  const { name, value } = e.target;
  setValues({ ...values, [name]: value });
};

3. Textarea and Select

In React, <textarea> and <select> tags work very similarly to simple <input> tags. They both use a value attribute for their content and a single onChange handler for updates.

Conclusion

Controlled components give you complete power over user input and enable complex validation, real-time transformations, and interactive form behavior with ease. By mastering these patterns, you can build reliable and user-friendly data entry experiences in your React applications.
