Lifting State Up

Often, several components need to reflect the same changing data. We recommend lifting the shared state up to their closest common ancestor. This technique allows multiple components to share the same information and stay in sync.

1. Why Lift State Up?

In React, state is local to each component. If two components (like a parent and child, or two siblings) need to access and modify the same data, the state should be moved to a component that is a parent to both of them. This follows React's "single source of truth" principle.

2. How to Implement

  1. Identify which components need the shared data.
  2. Find the closest common ancestor of all those components.
  3. Move the state from the child to that common ancestor.
  4. Pass the state down to the components as props.
  5. Pass any necessary state updater functions as callbacks to the children.

3. Example

function TemperatureInput({ temperature, onTemperatureChange }) {
  return (
    <input
      value={temperature}
      onChange={(e) => onTemperatureChange(e.target.value)}
    />
  );
}

Conclusion

Lifting state up is a powerful pattern that ensures your data remains consistent across your application. While it may require adding more code than simply passing global variables, it maintains the predictable data flow that makes React both powerful and easy to debug.
