JSX (JavaScript XML)

JSX is a syntax extension for JavaScript used with React. It looks a lot like HTML but is actually JavaScript. JSX allows you to write your UI structure using a familiar, tag-based syntax alongside your logic code.

1. Why Use JSX?

You don't have to use JSX with React, but most developers do because it's much easier to read and write than the underlying React.createElement() code it compiles into.

// With JSX:
const element = <h1>Hello, World!</h1>;

// Without JSX:
const element = React.createElement("h1", null, "Hello, World!");

2. Rules of JSX

  • className, not class: Since class is a reserved keyword in JavaScript, use className for CSS classes.
  • htmlFor, not for: Similarly, use htmlFor for <label> tags.
  • Single Parent: JSX must always be wrapped in a single parent element (use <div> or a Fragment <>...</>).
  • camelCase: HTML attributes become camelCase (e.g., onclick becomes onClick).

3. Embedding Expressions

You can embed any JavaScript expression inside JSX by wrapping it in curly braces {}.

const name = "Josh";
const element = <h1>Hello, {name}!</h1>;

Conclusion

JSX is what makes React both powerful and enjoyable to work with. By allowing you to describe your UI and logic in the same place, it simplifies development and improves code clarity. Once you master the basic rules, you'll never want to go back to manual DOM updates again.
