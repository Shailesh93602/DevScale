React Components

Components are the building blocks of every React application. They are independent and reusable bits of code that serve the same purpose as JavaScript functions, but work in isolation and return HTML-like JSX structures.

1. PascalCase Naming

In React, component names must always start with a capital letter (PascalCase). This is how React distinguishes between regular HTML tags (like <div>) and custom components (like <Header />).

2. Functional Components

Modern React applications are built using Functional Components. A functional component is literally just a JavaScript function that returns JSX.

function Welcome() {
  return <h1>Hello, Welcome to my app!</h1>;
}

export default Welcome;

3. Component Composition

The true power of React comes from building small, focused components and composing them together to create a large interface.

function App() {
  return (
    <div>
      <Header />
      <MainContent />
      <Footer />
    </div>
  );
}

4. Exporting and Importing

Usually, each component is created in its own file. You can share them across your app using export and import.

  • Default export: export default MyComponent; (imported as import MyComponent from "./MyComponent")
  • Named export: export const MyComponent = ...; (imported as import { MyComponent } from "./MyComponent")

Conclusion

By thinking in components, you break down complex UIs into manageable, testable, and reusable pieces. This modular approach is at the heart of React's philosophy and is what enables teams to build massive applications with consistency and speed.
