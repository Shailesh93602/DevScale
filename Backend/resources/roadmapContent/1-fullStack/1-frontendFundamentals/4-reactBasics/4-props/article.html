React Props (Properties)

Props are the way we pass data from one React component to another (unidirectionally, from parent to child). Just as functions take arguments, React components take props.

1. Passing and Using Props

To pass a prop to a component, use an attribute-like syntax in your JSX. To access it, look at the first argument of your functional component.

// Parent Component
<Welcome name="Alice" />

// Child Component
function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

2. De-structuring Props

It's more common and cleaner to use ES6 de-structuring to extract props directly from the function argument.

function Welcome({ name, age }) {
  return <h1>{name} is {age} years old.</h1>;
}

3. Props are Read-Only

One of the most important rules in React is that props are immutable. A component must never modify its own props. If you need to change data within a component, that's where State comes in.

4. The 'children' Prop

Every component automatically receives a special children prop, which contains any JSX content placed between its opening and closing tags.

<Card>
  <p>Inside the card content.</p>
</Card>

function Card({ children }) {
  return <div className="card-box">{children}</div>;
}

Conclusion

Props are the lifeblood of data flow in React. By using props, you can create truly dynamic and reusable components that look and behave differently based on the data they receive. Understanding that data moves down from parent to child is key to mastering React architecture.
