React Hooks (useState and useEffect)

Hooks were introduced in React 16.8 to allow functional components to use state and other React features without writing a class. The two most common hooks are useState and useEffect.

1. The useState Hook

useState is used to add local state to a functional component. It returns an array with two items: the current state value and a function to update it.

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicked {count} times
    </button>
  );
}

2. The useEffect Hook

useEffect is used for side effects in your component like data fetching, subscriptions, or manually changing the DOM. It runs after every render by default, but you can control it with a dependency array.

import { useEffect, useState } from "react";

function Message() {
  useEffect(() => {
    console.log("Component mounted!");

    // Optional cleanup function
    return () => console.log("Component unmounted!");
  }, []); // Empty array means run only once on mount
}

3. Rules of Hooks

  • Top Level Only: Never call hooks inside loops, conditions, or nested functions. Always call them at the top level of your component.
  • React Functions Only: Only call hooks from React functional components or custom hooks.

Conclusion

Hooks have paved the way for a more functional and cleaner React code base. By mastering useState and useEffect, you can handle almost any dynamic behavior needed in a modern web application while keeping your code concise and readable.
