React Lifecycle Methods

In React, every component goes through a lifecycle of events. You can think of it as three main phases: Mounting (being created), Updating (changing state), and Unmounting (being removed).

1. Mounting Phase

This is when the component is being initialized and inserted into the DOM. In older class-based components, componentDidMount() was used here. In functional components, this is handled by a useEffect() with an empty dependency array [].

2. Updating Phase

This phase occurs when a component's state or props change. In class components, componentDidUpdate() was the standard. In functional components, useEffect() with specific dependencies in the array handles this.

3. Unmounting Phase

This is when the component is being removed from the DOM. In class components, componentWillUnmount() was used for cleanup (like canceling API calls or timers). In functional components, you return a function from within useEffect().

useEffect(() => {
  // Mount/Update logic
  return () => {
    // Cleanup/Unmount logic
  };
}, [dependencies]);

Conclusion

While class-based lifecycle methods are still supported, the modern useEffect hook provides a more unified and simplified way to handle all three phases. Understanding how hooks map to these lifecycle events is essential for writing robust and bug-free React applications.
