Event Handling in React

Handling events with React elements is very similar to handling events on DOM elements. There are some syntactic differences, but the logic remains almost the same.

1. Syntactic Differences

  • React events are named using camelCase (e.g., onClick instead of onclick).
  • With JSX, you pass a function as the event handler rather than a string.
// HTML
<button onclick="activate()">Activate</button>

// React
<button onClick={activate}>Activate</button>

2. Passing Arguments to Handlers

If you need to pass an argument to your event handler, you can wrap it in an arrow function. This ensures the function is called only when the event actually occurs.

function deleteUser(id) {
  console.log("Deleting user", id);
}

<button onClick={() => deleteUser(5)}>Delete User 5</button>

3. Preventing Default Behavior

In React, you cannot return false to prevent default behavior. You must call event.preventDefault() explicitly.

function handleSubmit(e) {
  e.preventDefault();
  console.log("Form submitted!");
}

<form onSubmit={handleSubmit}>
  <button type="submit">Submit</button>
</form>

Conclusion

React event handling is intuitive and consistent across browsers, thanks to React's synthetic event system. By mastering how to wire up your components with user interactions, you can create truly interactive and responsive application logic.
