Conditional Rendering in React

In React, you can create components that encapsulate the behavior you need. Then, you can render only some of them, depending on the state of your application. Conditional rendering works the same way conditions work in JavaScript.

1. Using Variable with if...else

You can use basic JavaScript if or else statements outside your JSX and return different React elements based on the result.

function Welcome({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  }
  return <h1>Please sign up.</h1>;
}

2. Inline If with Logical && Operator

The && operator is the most common way to conditionally include an element in your UI. If the condition is true, the element will render; if false, React ignores it.

<div>
  <h1>Hello!</h1>
  {unreadMessages.length > 0 &&
    <h2>You have {unreadMessages.length} unread messages.</h2>
  }
</div>

3. Ternary Operator (condition ? ExprIfTrue : ExprIfFalse)

The ternary operator is useful when you need to choose between two different elements.

<div>
  The user is <b>{isLoggedIn ? "currently" : "not"}</b> logged in.
</div>

4. Returning Null

If you don't want a component to render anything at all, you can return null from its render function.

Conclusion

Conditional rendering allows you to build dynamic and responsive user interfaces that change based on user input, data availability, or application state. Mastering these different techniques is essential for creating robust and readable React applications.
