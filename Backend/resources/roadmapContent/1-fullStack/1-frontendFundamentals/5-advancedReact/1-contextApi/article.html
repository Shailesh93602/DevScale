React Context API

The Context API is a way for a React app to effectively produce global variables that can be passed around. This is the alternative to "prop drilling" or moving props from grandparent to child through many levels.

1. When to Use Context?

Context is designed to share data that can be considered “global” for a tree of React components, such as current authenticated user, theme, or preferred language.

2. createContext and Provider

To use context, first create it with createContext(). Then, wrap your component tree with the Provider and pass the data you want to share via the value prop.

const ThemeContext = createContext("light");

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>
  );
}

3. Consuming Context with useContext

The useContext() hook is the simplest way to read the data from a context provider in any functional component down the tree.

import { useContext } from "react";

function Button() {
  const theme = useContext(ThemeContext);
  return <button className={theme}>I am styled by context!</button>;
}

Conclusion

The Context API is a powerful tool for simplifying your component architecture and avoiding deep prop nesting. By mastering Context, you can handle global state with ease and clarity without the overhead of external libraries like Redux for simple data sharing.
