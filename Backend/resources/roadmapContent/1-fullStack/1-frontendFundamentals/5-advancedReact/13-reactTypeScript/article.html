React with TypeScript

Using TypeScript with React is the industry standard for building scalable and error-proof applications. It adds static type checking on top of JavaScript, which means many common bugs are caught by your editor before you even run your code.

1. Typing Functional Components

In modern React, you can type your functional components using the React.FC type or simply by typing the props argument directly.

interface Props {
  title: string;
}

const Header = ({ title }: Props) => {
  return <h1>{title}</h1>;
};

2. Typing State with Hooks

The useState hook can be given a generic type to ensure that your state only ever holds the correct data type.

const [count, setCount] = useState<number>(0);
// count must now always be a number!

3. Handling Events

TypeScript helps you correctly type your event handlers so you can access properties like event.target.value with full IntelliSense and type safety.

const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
  console.log(event.target.value);
};

Conclusion

React with TypeScript is a formidable combination that improves developer productivity and code reliability. By mastering how to type your components, props, and states, you can build truly professional React applications that are easier to maintain and reason about as they grow.
