React Portals

Portals provide a way to render children into a DOM node that exists outside the DOM hierarchy of the parent component. This is incredibly useful for UI elements that need to break out of their container's styling or layout, such as modals and tooltips.

1. createPortal() Syntax

The createPortal() function takes two arguments: the JSX you want to render and the DOM element (usually an ID or document.body) where it should be placed.

import { createPortal } from "react-dom";

function Modal({ children }) {
  return createPortal(
    <div className="modal">{children}</div>,
    document.getElementById("modal-root")
  );
}

2. Why Use Portals?

Standard React components are mounted as children of their parent's DOM node. This can cause problems with CSS properties like overflow: hidden or z-index. Portals allow these elements to live elsewhere in the document structure (like directly under document.body) while still behaving like React components.

3. Event Bubbling

Even though a portal can be anywhere in the actual DOM tree, it behaves like a normal React child in every other way. This includes **event bubbling**: an event fired inside a portal will still bubble up to its ancestors in the React tree, regardless of where the portal's DOM element is located.

Conclusion

Portals are a specialized but essential tool for building advanced user interfaces. By mastering how to breakout of the standard DOM hierarchy, you can ensure that complex UI components like modals and popovers are always rendered correctly and reliably while maintaining the benefits of React's component-based architecture.
