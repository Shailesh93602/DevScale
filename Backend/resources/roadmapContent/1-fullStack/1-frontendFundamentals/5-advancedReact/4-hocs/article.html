Higher-Order Components (HOCs)

A Higher-Order Component is an advanced technique in React for reusing component logic. They are not part of the React API, per se; they are a pattern that emerges from React’s compositional nature.

1. What is a HOC?

  1. It's a function.
  2. It takes a component as an argument.
  3. It returns a new component with additional props or behavior.
const EnhancedComponent = withExtraLogic(BaseComponent);

2. When to Use HOCs?

HOCs are excellent for cross-cutting concerns like logging, authentication checks, or data fetching where multiple components need the same underlying logic. By extracting this logic, you keep your individual components focused on their primary UI task.

3. HOC Convention

It's conventional to name an HOC starting with with (e.g., withAuth, withRouter, withLogger). This makes it clear to other developers that the component is being enhanced.

Conclusion

While React Hooks have replaced many use cases for HOCs, they remain a powerful and common pattern in existing codebases and libraries. By understanding how to compose components using higher-order functions, you can build truly scalable and modular React applications.
