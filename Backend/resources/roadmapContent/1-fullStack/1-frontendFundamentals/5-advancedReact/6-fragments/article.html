React Fragments

A common pattern in React is for a component to return multiple elements. Fragments let you group a list of children without adding extra nodes to the DOM.

1. The Problem with Extra Divs

Sometimes, we wrap elements in a <div> just because a React component must return a single element. This can clutter the DOM and lead to CSS issues with layout (e.g. flexbox/grid) that expect specific child structures.

2. Using Fragments

Fragments allow you to group multiple JSX elements together without any wrapping DOM node being created at all.

import React from "react";

function Columns() {
  return (
    <React.Fragment>
      <td>Hello</td>
      <td>World</td>
    </React.Fragment>
  );
}

3. Short Syntax

There is a shorter and newer syntax for declaring Fragments that doesn’t require importing React separately: <></>.

function Columns() {
  return (
    <>
      <td>Hello</td>
      <td>World</td>
    </>
  );
}

Conclusion

Fragments are a simple but effective tool for keeping your DOM clean and avoiding unnecessary structural nodes. By using <React.Fragment> or the short <></> syntax, you can return multiple sibling elements from any component without sacrificing CSS flexibility or performance.
