Vue.js Directives

Directives are special attributes with the v- prefix. They provide a declarative way to tell Vue to perform specific actions on a DOM element when data changes.

1. Common Built-in Directives

  • v-bind (:): Dynamically binds one or more attributes, or a component prop to an expression.
  • v-model: Creates a two-way data binding on form elements.
  • v-if / v-else: Conditionally renders an element based on the truthiness of an expression.
  • v-for: Renders a list of items based on an array.

2. Conditional Rendering (v-if vs. v-show)

While v-if physically adds or removes elements from the DOM, v-show only toggles their display CSS property. v-if is more expensive to toggle but is better if the condition is unlikely to change often.

3. Working with Lists (v-for)

Using v-for, you can render elements dynamically for each item in an array of data. Just like in React, remember to provide a :key attribute for performance and stability.

<li v-for="item in items" :key="item.id">
  {{ item.text }}
</li>

Conclusion

Directives are one of Vue’s most powerful and visible features. By using them, you can build dynamic and interactive web applications with significantly less code than traditional JavaScript, while maintaining a clear and declarative template structure.
