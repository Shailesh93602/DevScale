Event Handling in Vue.js

In Vue, we use the v-on directive (shorthanded as @) to listen to DOM events and run JavaScript when they’re triggered. This provides a clean and declarative way to handle user interaction.

1. Common Syntax

To listen for an event, such as a click, use v-on:click or @click with a method name or inlined JavaScript code.

<!-- Full syntax -->
<button v-on:click="count++">Add 1</button>

<!-- Shorthand syntax -->
<button @click="sayHello">Say Hello</button>

2. Method Handlers

If you have complex logic, you should use a method defined in your Vue component options. Vue will automatically pass the native DOM event object to the method unless you pass your own arguments.

methods: {
  sayHello(event) {
    alert('Hello from Vue!')
    if (event) {
      alert(event.target.tagName)
    }
  }
}

3. Event Modifiers

Vue allows you to handle common event patterns inside your template using dot-separated modifiers. This keeps your methods focused purely on application logic instead of DOM details like event.preventDefault().

  • .stop: Equivalent to event.stopPropagation().
  • .prevent: Equivalent to event.preventDefault().
  • .once: The event will only be triggered at most once.

Conclusion

Event handling in Vue is both powerful and developer-friendly. By mastering v-on, method handlers, and event modifiers, you can build interactive applications that respond gracefully to user input without cluttering your code with repetitive DOM housekeeping.
