Vue.js Watchers

While computed properties are ideal for synchronous calculations, you often need to perform **side effects** (like an API call, logging, or setting a timer) whenever a piece of data changes. This is what **Watchers** are for.

1. What are Watchers?

A watcher is a function defined in the watch option of your Vue instance. It is named after the data property it "watches" and will be executed every time that data property changes.

watch: {
  question(newQuestion, oldQuestion) {
    if (newQuestion.includes('?')) {
      this.getAnswer()
    }
  }
}

2. When to Use Watchers?

  • Asynchronous operations: Need to fetch data from a server when a user selects a filter.
  • Expensive operations: Performing a calculation that only needs to happen when specific inputs change.
  • Deep watching: Watching complex objects or arrays for internal changes (requires { deep: true }).

3. Watchers vs. Computed

In general, you should use **Computed properties** for synchronous logic that returns a simplified version of existing data. Use **Watchers** for side effects that don’t necessarily return a value but instead "do something" when data updates.

Conclusion

Watchers are an essential tool for creating truly interactive and asynchronous Vue applications. By mastering how and when to "watch" your data, you can build seamless user experiences that gracefully handle complex behaviors and real-time updates.
