Vue.js Methods

Methods are the place to define the logic for your Vue components. They are standard JavaScript functions that can be used for event handling, data manipulation, or any reusable piece of code within your application.

1. Defining Methods

Methods are defined within the methods option of your Vue instance. Because they have access to the component instance via this, they can easily read and update any data or properties.

methods: {
  greet() {
    alert('Hello, ' + this.name + '!')
  },
  incrementCount() {
    this.count++
  }
}

2. Using Methods in Templates

Methods are most commonly used in event handlers (e.g. @click="greet"), but they can also be called directly inside templates (e.g. {{ formatName() }}). **Note:** Calling methods in templates will re-run the method on every render, so use them sparingly for large or expensive calculations.

3. Arrow Functions

Avoid using arrow functions when defining your Vue methods, as arrow functions don’t have their own this context. To access the Vue instance data with this.someProperty, regular function syntax is required.

Conclusion

Methods are the workhorses of your Vue applications. By building out a clear and well-structured set of methods, you can group your logic into reusable and testable units that work seamlessly with Vue’s reactivity for a smooth and interactive user experience.
