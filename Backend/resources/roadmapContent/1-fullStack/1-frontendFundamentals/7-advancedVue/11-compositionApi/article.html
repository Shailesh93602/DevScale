Vue Composition API

The Composition API is a set of APIs that allows you to author Vue components using imported functions instead of declaring options. It is the built-in way in Vue 3 to share logic between components with maximum flexibility and TypeScript support.

1. Why the Composition API?

In large components using the Options API (data, methods, computed), related logic is often scattered across different options. The Composition API lets you group related code together, making it much easier to read, test, and reuse.

2. Core Concepts: setup(), ref(), and reactive()

  • setup(): The entry point for using the Composition API in a component. It runs before the component is created.
  • ref(): Used to create a reactive reference for any data type (strings, numbers, etc.). Access it via .value.
  • reactive(): Used to create a reactive object. You don't need .value to access its properties.
import { ref, onMounted } from 'vue'

export default {
  setup() {
    const count = ref(0)
    const increment = () => count.value++

    onMounted(() => console.log('Mounted!'))

    return { count, increment }
  }
}

3. Composables

The true power of the Composition API lies in **Composables**—functions that leverage Vue's reactive APIs to encapsulate and reuse stateful logic (e.g., useMouse(), useFetch()).

Conclusion

The Composition API is the future of Vue.js development. By mastering setup, ref, and reactive, you can build truly modular, scalable, and type-safe applications that are much easier to maintain than traditional options-based components.
