Vue.js Mixins

Mixins are a flexible way to distribute reusable functionalities for Vue components. A mixin object can contain any component options. When a component uses a mixin, all options in the mixin will be “mixed” into the component’s own options.

1. Defining and Using Mixins

A mixin is simply an object that looks like any other Vue component. To use it, you add it to the mixins array in your component's definition.

const myMixin = {
  data() { return { count: 0 } },
  created() { console.log('Mixin created!') }
}

const component = {
  mixins: [myMixin]
}

2. Merge Strategies

When options overlap (e.g., both the component and mixin define a data property or a created hook), Vue follows specific merging rules. For life-cycle hooks, both will be called, and the mixin’s hook is usually called first.

3. Modern Tool: Composables

While mixins are still prevalent in legacy Vue codebases, the **Composition API (Composables)** is now the recommended way to share logic in Vue 3, as it solves many problems with mixins, such as property collisions and lack of clarity on where a property comes from.

Conclusion

Mixins are a seasoned but powerful tool in the Vue developer's toolkit. By understanding how to group and reuse your component logic, you can build modular and maintainable applications, even when using the Options API in older or migrating projects.
