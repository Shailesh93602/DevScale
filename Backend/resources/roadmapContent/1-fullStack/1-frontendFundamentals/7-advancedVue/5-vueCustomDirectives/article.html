Vue.js Custom Directives

In addition to the core set of directives (like v-model and v-show), Vue also allows you to register your own custom directives. While components are for reusable chunks of UI, custom directives are primarily for **reusable low-level DOM access**.

1. Defining a Custom Directive

A custom directive object defines hook functions that are called at various points in the lifecycle of the element it’s attached to.

const vFocus = {
  mounted: (el) => el.focus()
}

createApp({
  directives: {
    focus: vFocus
  }
})

2. Hook Functions

Custom directives have several lifecycle hooks, including:

  • created: Called before element attributes or event listeners are applied.
  • mounted: Called when the element is inserted into the DOM.
  • updated: Called after the containing component has updated.

3. Why Use Them?

Custom directives are perfect for cases where you need to interact with a DOM element in a way that doesn't fit into standard data binding—for example, automatically focusing an input, adding a tooltip, or managing a 3rd-party library that needs a direct reference to an element.

Conclusion

By mastering custom directives, you can add specialized and reusable behaviors to your DOM elements with ease. Whether you're managing complex interactions or integrating external libraries, custom directives give you the low-level control you need while staying within Vue's declarative template system.
