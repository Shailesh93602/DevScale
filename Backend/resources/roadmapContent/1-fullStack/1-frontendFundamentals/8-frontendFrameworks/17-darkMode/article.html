Dark Mode in Frameworks

Dark mode has become a standard feature for modern web applications. Both Tailwind CSS and Bootstrap provide built-in, first-class support for implementing dark themes, allowing you to easily cater to user preferences and reduce eye strain.

1. Tailwind CSS Dark Mode

Tailwind uses the dark: variant to apply styles only when dark mode is active. You can trigger it using the user's system settings (the media strategy) or by adding a .dark class to your HTML (the selector strategy).

<div class="bg-white dark:bg-gray-800">
  <h1 class="text-black dark:text-white">Hello World</h1>
</div>

2. Bootstrap Dark Mode (5.3+)

In Bootstrap 5.3 and later, dark mode is implemented using the data-bs-theme attribute. By simply adding data-bs-theme="dark" to any element (even the <html> tag), all Bootstrap components will automatically switch their background and text styles.

3. User Experience and Auto-switching

The best dark mode implementations allow users to toggle between Light, Dark, and "System" (which follows their OS setting). By storing the user's preference in localStorage, you can ensure a consistent experience across sessions.

Conclusion

By mastering dark mode in modern frameworks, you're building a truly professional and user-centric application. Whether you choose Tailwind's utility-first variants or Bootstrap's attribute-based themes, providing a high-quality dark mode experience is a hallmark of modern frontend engineering.
