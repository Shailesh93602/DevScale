Middleware in Express.js

The "heart" of any Express application is its **Middleware** system. A middleware function is simply a function that has access to the request (req) and response (res) objects, allowing you to run code or modify data before it reaches your final route handler.

1. How Middleware Works

Middleware functions run in a specific order. Each function can either end the request (by sending a response) or pass control to the next middleware in the stack using the next() function.

2. Types of Middleware

  • Application-Level: Defined using app.use(), these functions run for every single request to your server (e.g. logging or data parsing).
  • Route-Level: Defined only for a specific route (e.g. authentication for /admin).
  • Built-in: Express provides some utilities built-in, like express.json() for parsing JSON data and express.static() for serving images/CSS.

3. Third-Party Middleware

Modern developers use many community-built middlewares to handle common tasks. For example, **Helmet** adds security headers, **Morgan** logs incoming requests, and **CORS** allows your backend to talk to your frontend correctly.
