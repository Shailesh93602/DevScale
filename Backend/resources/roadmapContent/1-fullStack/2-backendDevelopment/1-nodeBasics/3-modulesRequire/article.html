Modules and Require

To keep your code organized, maintainable, and scalable, Node.js uses a **Module System**. This allows you to split your application into smaller, independent files (modules) that can be easily reused and tested across your entire project.

1. CommonJS (The Classic Way)

For many years, Node.js primarily used the **CommonJS** module system. It uses require() to import code and module.exports to share it with other files.

// math.js
const add = (a, b) => a + b;
module.exports = { add };

// app.js
const { add } = require('./math');
console.log(add(5, 10)); // 15

2. ES Modules (The Modern Web Standard)

Modern Node.js (and the browser) now supports **ES Modules (ESM)**. This uses the import and export keywords, which are more powerful and standard across the entire JavaScript ecosystem.

// math.mjs
export const add = (a, b) => a + b;

// app.mjs
import { add } from './math.mjs';

3. Built-in Modules

Node.js comes with a rich set of built-in modules that you don't need to install through NPM. For example, the path module helps you work with file paths correctly across different operating systems (Windows vs Linux).
