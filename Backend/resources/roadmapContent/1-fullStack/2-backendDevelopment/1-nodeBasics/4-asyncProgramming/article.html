Asynchronous Programming

To build truly high-performance servers, Node.js uses an **Asynchronous** (non-blocking) programming model. This allows your application to handle multiple tasks (like reading from a database or a file) simultaneously without stopping the entire application to wait for each one to finish.

1. Callbacks (The Foundation)

A callback is a function that you pass as an argument to another function, which will be executed once the task is complete. While powerful, "Callback Hell" (nested callbacks) can quickly make your code difficult to read and maintain.

2. Promises (.then() and .catch())

Promises were introduced to solve the complexity of callbacks. A Promise is a "job" that will either resolve (succeed) or reject (fail) in the future, providing a much cleaner way to handle asynchronous logic with .then() and .catch().

3. Async/Await (The Modern Standard)

Building on Promises, **Async/Await** allows you to write asynchronous code that looks and behaves like synchronous code. It is significantly more readable and is the industry standard for professional Node.js development.

async function fetchData() {
  try {
    const data = await getFromDatabase();
    console.log(data);
  } catch (error) {
    console.error("Oops!", error);
  }
}
