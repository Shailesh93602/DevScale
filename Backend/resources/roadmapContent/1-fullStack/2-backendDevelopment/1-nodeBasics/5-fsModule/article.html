File System (fs) Module

To interact with files and folders on the server, Node.js provides the **File System (fs)** module. This allows you to read, write, update, and delete files with ease, making it a critical tool for any backend application.

1. Reading Files

The fs module provides both synchronous (blocking) and asynchronous (non-blocking) ways to read files. In a production server, you should almost always use the asynchronous methods to ensure your application remains responsive.

const fs = require('fs');

// Read a file (Asynchronous)
fs.readFile('hello.txt', 'utf8', (err, data) => {
  if (err) console.error(err);
  console.log(data);
});

2. Writing and Appending Files

  • fs.writeFile: Creates a new file or overwrites an existing one with new content.
  • fs.appendFile: Adds new content to the end of an existing file (without deleting its current contents).

3. Using fs.promises (Modern Approach)

Instead of callbacks, you can use the **Promises-based** version of the fs module (fs.promises). This allows you to use the much cleaner async/await syntax you learned in the previous topic!
