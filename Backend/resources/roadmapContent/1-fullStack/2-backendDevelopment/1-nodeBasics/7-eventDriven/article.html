Event-Driven Programming

Node.js is an **Event-Driven** runtime. This means that instead of a linear, step-by-step process, your application is "listening" for events to happen (like a database query finishing or a user logging in) and then "reacting" to them by running a specific function.

1. The EventEmitter Class

The EventEmitter class is the heart of Node's event system. By creating an instance of this class, you can register listeners (.on()) and trigger events (.emit()) whenever you need to signal that something important has happened in your code.

const EventEmitter = require('events');
const myEmitter = new EventEmitter();

// 1. Register a Listener
myEmitter.on('login', (user) => {
  console.log(`Hello, ${user}!`);
});

// 2. Trigger an Event
myEmitter.emit('login', 'Alice');

2. Why Use Events? (Decoupling)

Events allow you to **decouple** your code. Instead of one giant function that handles every single part of a process, you can have many small, independent listeners that each react to the same event in their own way, making your application much easier to maintain.

3. Professional Usage

Most core Node.js modules (like http or fs.ReadStream) are built directly on top of the EventEmitter class. By mastering this pattern, you gain a deep understanding of how Node.js works "under the hood."

Conclusion

By mastering event-driven programming, you're becoming a faster and more efficient Fullstack engineer.
