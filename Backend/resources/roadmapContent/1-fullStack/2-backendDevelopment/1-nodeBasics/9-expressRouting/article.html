Routing in Express.js

To build a real-world web application, you need more than just a single page. **Routing** is the process of defining which code should run when a user visits a specific URL (like /users) or performs a specific action (like a POST request to /login).

1. HTTP Methods (GET, POST, etc.)

Express makes it incredibly easy to handle different types of network requests. You can define specific handlers for reading data (GET), creating data (POST), updating data (PUT), and deleting data (DELETE).

app.get('/users', (req, res) => {
  res.send('Get all users');
});

app.post('/users', (req, res) => {
  res.send('Create a new user');
});

2. Route Parameters (:id)

Professional applications often have dynamic URLs. For example, if you visit /users/123, Express can capture that 123 as a variable (route parameter) using a colon (:) in your route definition.

app.get('/users/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`User details for ${userId}`);
});

3. Query Parameters

For searching and filtering, you can also access **Query Parameters** (the part after the ? in a URL, like /search?q=node) through the req.query object.
