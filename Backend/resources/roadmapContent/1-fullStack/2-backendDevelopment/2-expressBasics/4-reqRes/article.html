Request and Response (req and res)

To build any web application, you must master the two most important objects in the Express framework: the **Request (req)** object and the **Response (res)** object. These allow your server to receive data from a user and send back professional, structured data.

1. The Request Object (req)

The req object contains everything the user is sending to your server:

  • req.body: Contains data sent from a form or an AJAX request (usually JSON).
  • req.params: Contains dynamic values from the URL (like :id).
  • req.query: Contains values from a query string (like ?search=node).
  • req.headers: Contains extra information about the request (like authentication tokens).

2. The Response Object (res)

The res object is used to send data back to the user's browser:

  • res.status(code): Sets the HTTP status code (e.g. 200 for OK, 404 for Not Found).
  • res.json(data): Sends a JSON response—the industry standard for modern APIs.
  • res.redirect(url): Sends the user to a completely different web page.
  • res.send(content): Sends a simple string or HTML content.

Conclusion

By mastering the req and res objects, you're becoming a faster and more efficient Fullstack engineer. Whether you're managing complex database logic or scaling a massive production application, these objects provide the tools and security you need to build the highest-quality and most professional applications.
