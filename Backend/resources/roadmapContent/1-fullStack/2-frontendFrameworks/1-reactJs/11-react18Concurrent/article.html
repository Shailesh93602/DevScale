React 18 Concurrent Features

React 18 introduced Concurrent Rendering, fundamentally changing how React operates behind the scenes. Instead of blocking the main thread while rendering deep component trees, React can now pause, yield, and resume rendering to keep the UI perfectly responsive.

Transitions (useTransition)

In standard React, all state updates are urgent. If you click a menu to open a heavy dashboard, the entire app freezes until the dashboard finishes rendering. useTransition lets you mark non-urgent updates, keeping the UI responsive while rendering happens in the background.

import { useState, useTransition } from 'react';

function FilterableList({ items }) {
  const [filter, setFilter] = useState('');
  // 1. We have a fast input state, but a heavy filtered list
  const [heavyFilter, setHeavyFilter] = useState('');
  const [isPending, startTransition] = useTransition();

  const onChange = (e) => {
    // Urgent update: The input must reflect keystrokes immediately
    setFilter(e.target.value);
    
    // Non-urgent update: The heavy list computation can lag behind
    startTransition(() => {
      setHeavyFilter(e.target.value);
    });
  };

  return (
    <div>
      <input value={filter} onChange={onChange} />
      {isPending && <span>Filtering...</span>}
      
      {/* Heavy component that takes hundreds of ms to render */}
      <HeavyList items={items} query={heavyFilter} />
    </div>
  );
}

useDeferredValue

useDeferredValue is similar to transitions, but instead of wrapping a state updating function, it wraps a value passed down from props or other state. It returns a deferred version of that value that "lags behind" during heavy renders.

import { useState, useDeferredValue, memo } from 'react';

function SearchPage() {
  const [query, setQuery] = useState('');
  
  // As the user types, 'query' updates instantly. 
  // 'deferredQuery' will update in the background.
  const deferredQuery = useDeferredValue(query);
  
  // Can check if stale to show visual feedback (opacity)
  const isStale = query !== deferredQuery;

  return (
    <div>
      <input value={query} onChange={e => setQuery(e.target.value)} />
      
      <div style={{ opacity: isStale ? 0.5 : 1 }}>
        <SlowSearchResults query={deferredQuery} />
      </div>
    </div>
  );
}

// IMPORTANT: Slow component MUST be wrapped in React.memo
// otherwise it will re-render anyway when the parent renders
const SlowSearchResults = memo(function Results({ query }) {
   // heavy rendering
});

Suspense for Data Fetching & Architecture

Suspense acts as a boundary that catches loading states from descendant components. When combined with Concurrent features or React Server Components, Suspense allows parts of the UI to stream in seamlessly.

import { Suspense, lazy } from 'react';

// The Dashboard component is lazy loaded to split the bundle
const DashboardPanel = lazy(() => import('./DashboardPanel'));

function App() {
  return (
    <div className="layout">
      <Sidebar />
      <main>
        {/* If DashboardPanel is loading, React will show the fallback but 
            the sidebar and main app remain fully interactive */}
        <Suspense fallback={<Spinner text="Loading Dashboard..." />}>
          <DashboardPanel />
        </Suspense>
      </main>
    </div>
  );
}

Automatic Batching

Prior to React 18, React only batched state updates that occurred inside React event handlers (like onClick). Updates in Promises, setTimeout, or native event handlers would trigger multiple re-renders. React 18 fixes this by automatically batching all state updates regardless of origin.

// In React 18, this only triggers ONE re-render!
fetch('/api/data').then(res => {
  setData(res.data);
  setLoading(false);
  setError(null);
  // React waits until the end of the microtask to flush these updates entirely
});

Best Practices

  • Use useTransition when you control the state update (e.g., inside an event handler).
  • Use useDeferredValue when receiving new values from above (as props) and you want a memoized child component to lag behind.
  • Always wrap heavy components consuming deferred values in React.memo() — without memo, the child re-renders aggressively anyway.
