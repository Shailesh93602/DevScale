React 19 & Modern Data Fetching

React 19 revolutionizes how developers handle asynchronous operations, forms, and data fetching on the client and server. By dropping boilerplate like useEffect for data loading in favor of new primitives, modern React is significantly simpler and faster.

The use Hook

The new use API is the only React hook that can be called conditionally. It allows you to read the value of a Promise or Context directly during render. When reading a Promise, React will suspend the component until the Promise resolves.

import { use, Suspense } from 'react';

// Fetching without useEffect!
function Comments({ commentsPromise }) {
  // `use` suspends this component until the promise resolves
  // The nearest <Suspense> boundary will show a fallback
  const comments = use(commentsPromise);
  
  return (
    <ul>
      {comments.map(c => <li key={c.id}>{c.text}</li>)}
    </ul>
  );
}

function PostPage() {
  const promise = fetch('/api/comments').then(r => r.json());
  
  return (
    <Suspense fallback={<p>Loading comments...</p>}>
      <Comments commentsPromise={promise} />
    </Suspense>
  );
}

Server Actions

Server Actions allow you to define asynchronous functions that run on the server, but can be called directly from Client Components (like from a form action or button onClick). They eliminate the need to manually build API endpoints for every mutation.

// actions.js (Server)
'use server';

export async function createPost(formData) {
  const title = formData.get('title');
  await db.post.insert({ title });
  // In Next.js, you might call revalidatePath('/posts') here
}

// PostForm.jsx (Client)
import { createPost } from './actions';

export function PostForm() {
  return (
    <form action={createPost}>
      <input name="title" />
      <button type="submit">Submit</button>
    </form>
  );
}

useActionState (formerly useFormState)

When you submit a form via an action, you usually need to display success messages or validation errors. useActionState connects the form to the Server Action, returning the current state of the response.

import { useActionState } from 'react';

async function addToCartAction(previousState, formData) {
  try {
    const id = formData.get('productId');
    await api.addToCart(id);
    return { success: 'Product added successfully!' };
  } catch (err) {
    return { error: 'Failed to add product' };
  }
}

function AddToCartForm() {
  // Returns: [state, dispatchFunction, isPending]
  const [state, formAction, isPending] = useActionState(addToCartAction, null);

  return (
    <form action={formAction}>
      <input type="hidden" name="productId" value="123" />
      
      {state?.error && <p className="error">{state.error}</p>}
      {state?.success && <p className="success">{state.success}</p>}
      
      <button disabled={isPending}>
        {isPending ? 'Adding...' : 'Add to Cart'}
      </button>
    </form>
  );
}

useFormStatus

useFormStatus is a hook designed for nested form components (like submit buttons deep in a form). It reads the status of the parent <form> without requiring props to be drilled down.

import { useFormStatus } from 'react-dom';

// This component MUST be rendered inside a <form action={...}>
function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button type="submit" disabled={pending}>
      {pending ? 'Submitting...' : 'Submit'}
    </button>
  );
}

useOptimistic

Optimistic updates make an app feel instantly responsive by updating the UI before the server confirms the mutation. useOptimistic temporarily overrides state during an async transition, rolling back automatically if it fails.

import { useOptimistic } from 'react';

function LikeButton({ likeCount, onLike }) {
  // During an action, optimisticCount takes over. Afterwards, it reverts to likeCount.
  const [optimisticCount, addOptimisticLike] = useOptimistic(
    likeCount,
    (currentCount, increment) => currentCount + increment
  );

  return (
    <form action={async () => {
      addOptimisticLike(1); // update UI instantly
      await onLike();       // send to server
    }}>
      <button type="submit">❤️ {optimisticCount}</button>
    </form>
  );
}

Best Practices

  • Replace simple useEffect data fetches with Server Components or the use(Promise) hook combined with Suspense.
  • Use Server Actions to eliminate boilerplate REST API routes for simple mutations.
  • Always use useOptimistic for interactions like upvoting, liking, or adding items to a cart to make the app feel instantly responsive.
