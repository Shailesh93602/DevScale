React Performance Optimization

React is fast by default, but as apps grow, unnecessary re-renders can slow things down. These tools let you surgically optimize performance.

When Does React Re-render?

A component re-renders when: (1) its state changes, (2) its parent re-renders, or (3) its context changes. The goal is to prevent #2 when the props haven't actually changed.

React.memo — Skip Re-renders for Same Props

// Without memo: re-renders every time parent re-renders
function ExpensiveList({ items }: { items: string[] }) {
  return <ul>{items.map(i => <li key={i}>{i}</li>)}</ul>;
}

// With memo: only re-renders if items actually changed
const ExpensiveList = React.memo(function({ items }) {
  return <ul>{items.map(i => <li key={i}>{i}</li>)}</ul>;
});

useMemo — Cache Expensive Calculations

function ProductList({ products, filter }: Props) {
  // Without useMemo: recalculates on EVERY render
  // With useMemo: recalculates only when products or filter changes
  const filteredProducts = useMemo(() =>
    products.filter(p => p.category === filter),
    [products, filter]
  );

  return <ul>{filteredProducts.map(p => <li key={p.id}>{p.name}</li>)}</ul>;
}

useCallback — Stable Function References

function Parent() {
  const [count, setCount] = useState(0);

  // Without useCallback: new function reference on every render
  // → breaks React.memo on child!
  const handleClick = useCallback(() => {
    console.log("clicked");
  }, []); // Stable reference unless deps change

  return (
    <>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <MemoizedChild onClick={handleClick} />
    </>
  );
}

Code Splitting & Lazy Loading

import { lazy, Suspense } from "react";

// Load component only when it's needed (not in initial bundle)
const Dashboard = lazy(() => import("./pages/Dashboard"));
const Analytics = lazy(() => import("./pages/Analytics"));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/analytics" element={<Analytics />} />
      </Routes>
    </Suspense>
  );
}

Virtualization — Render Large Lists Efficiently

// Install: npm install @tanstack/react-virtual
import { useVirtualizer } from "@tanstack/react-virtual";

function VirtualList({ items }: { items: string[] }) {
  const parentRef = useRef<HTMLDivElement>(null);
  const virtualizer = useVirtualizer({
    count: items.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 35,
  });

  return (
    <div ref={parentRef} style={{ height: "500px", overflow: "auto" }}>
      <div style={{ height: virtualizer.getTotalSize() }}>
        {virtualizer.getVirtualItems().map(vi => (
          <div key={vi.key} style={{ transform: `translateY(${vi.start}px)` }}>
            {items[vi.index]}
          </div>
        ))}
      </div>
    </div>
  );
}

Quick Wins Checklist

  • Profile first with React DevTools Profiler — don't optimize blindly.
  • Use key correctly in lists to avoid unnecessary unmounts.
  • Avoid creating objects/arrays inline in JSX (they create new references every render).
  • Move static data outside of components.
