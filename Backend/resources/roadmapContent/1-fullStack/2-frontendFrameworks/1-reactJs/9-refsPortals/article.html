Refs, forwardRef & Portals

While React's declarative state model is powerful, sometimes you need to step outside of it to interact directly with DOM elements or store mutable data without triggering re-renders. That's where useRef, forwardRef, and createPortal come in.

useRef for DOM Access

The most common use case for useRef is accessing an underlying DOM node directly, often to manage focus, text selection, or media playback.

import { useRef } from 'react';

function SearchInput() {
  const inputRef = useRef(null);

  const focusInput = () => {
    // Current points to the mounted DOM element
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} placeholder="Search..." />
      <button onClick={focusInput}>Focus Input</button>
    </div>
  );
}

useRef for Mutable Variables

Unlike useState, updating a ref does not trigger a re-render. Information that is mutated but doesn't affect the visual output should live in a ref, like timer IDs.

function Stopwatch() {
  const [seconds, setSeconds] = useState(0);
  const timerRef = useRef(null); // Stores the interval ID

  const start = () => {
    if (timerRef.current !== null) return;
    timerRef.current = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);
  };

  const stop = () => {
    clearInterval(timerRef.current);
    timerRef.current = null;
  };

  return (
    <div>
      <p>{seconds}s</p>
      <button onClick={start}>Start</button>
      <button onClick={stop}>Stop</button>
    </div>
  );
}

forwardRef: Exposing DOM Nodes from Custom Components

By default, you can't pass a ref to a custom component. forwardRef lets your component take a ref and pass it down ("forward" it) to a standard DOM element inside it.

import { forwardRef, useRef } from 'react';

// Custom, styled input component that can still be focused by a parent
const FancyInput = forwardRef((props, ref) => {
  return (
    <div className="fancy-wrapper">
      <span className="icon">🔍</span>
      <input className="fancy-input" ref={ref} {...props} />
    </div>
  );
});

function Parent() {
  const inputRef = useRef(null);
  
  return (
    <div>
      <FancyInput ref={inputRef} placeholder="Search..." />
      <button onClick={() => inputRef.current.focus()}>Focus</button>
    </div>
  );
}

useImperativeHandle: Customizing Exposed Refs

Usually, forwardRef exposes the raw DOM node. useImperativeHandle lets you expose a customized object with specific methods instead of the raw DOM node.

import { forwardRef, useRef, useImperativeHandle } from 'react';

const VideoPlayer = forwardRef((props, ref) => {
  const videoRef = useRef(null);

  // Parent only gets access to these two methods, not the whole video node
  useImperativeHandle(ref, () => {
    return {
      play() {
        videoRef.current.play();
      },
      pause() {
        videoRef.current.pause();
      }
    };
  }, []); // dependencies

  return <video ref={videoRef} src="video.mp4" />;
});

function RemoteControl() {
  const playerRef = useRef(null);
  return (
    <div>
      <VideoPlayer ref={playerRef} />
      <button onClick={() => playerRef.current.play()}>Play</button>
    </div>
  );
}

React Portals

Portals let you render a component's children into a different part of the DOM hierarchy than the parent component. This is essential for modals, tooltips, and dropdowns to escape overflow: hidden or z-index constraints.

import { createPortal } from 'react-dom';

function Modal({ children, isOpen }) {
  if (!isOpen) return null;

  // Renders the JSX directly into document.body, not where Modal is placed
  return createPortal(
    <div className="modal-backdrop">
      <div className="modal-content">
        {children}
      </div>
    </div>,
    document.body
  );
}

// Usage in App
// Even if App has overflow: hidden, the modal renders at the root level
function App() {
  const [open, setOpen] = useState(false);
  return (
    <div className="app-main" style={{ overflow: 'hidden' }}>
      <button onClick={() => setOpen(true)}>Open Modal</button>
      <Modal isOpen={open}>
        <h1>Hello from Portal!</h1>
      </Modal>
    </div>
  );
}

Best Practices

  • Don't overuse refs for DOM manipulation. If you can do it declaratively with state (e.g., toggling a class), don't use a ref.
  • Use refs for timers and intervals. They are perfect for storing mutable data that doesn't affect render output.
  • Always use portals for overlays. Modals rendered inline will eventually break due to CSS stacking contexts.
