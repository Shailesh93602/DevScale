Zustand & Jotai — Lightweight State Management

Redux Toolkit is powerful but heavy. For many applications, simpler alternatives like Zustand and Jotai provide excellent state management with minimal boilerplate.

Zustand — Simple Store-Based State

Zustand creates a single store using a hook. No providers, no reducers, no actions — just state and updater functions.

import { create } from 'zustand';
import { persist, devtools } from 'zustand/middleware';

interface CartState {
  items: CartItem[];
  total: number;
  addItem: (item: CartItem) => void;
  removeItem: (id: string) => void;
  clearCart: () => void;
}

const useCartStore = create<CartState>()(
  devtools(          // Redux DevTools integration
    persist(         // Persist to localStorage
      (set, get) => ({
        items: [],
        total: 0,

        addItem: (item) => set((state) => {
          const existing = state.items.find(i => i.id === item.id);
          if (existing) {
            return {
              items: state.items.map(i =>
                i.id === item.id ? { ...i, quantity: i.quantity + 1 } : i
              ),
              total: state.total + item.price
            };
          }
          return {
            items: [...state.items, { ...item, quantity: 1 }],
            total: state.total + item.price
          };
        }),

        removeItem: (id) => set((state) => ({
          items: state.items.filter(i => i.id !== id),
          total: state.total - (state.items.find(i => i.id === id)?.price ?? 0)
        })),

        clearCart: () => set({ items: [], total: 0 })
      }),
      { name: 'cart-storage' }  // localStorage key
    )
  )
);

Using Zustand in Components

// Subscribe to only the slice you need — prevents unnecessary re-renders
function CartIcon() {
  const itemCount = useCartStore(state => state.items.length);
  return <span>Cart ({itemCount})</span>;
}

function CartTotal() {
  const total = useCartStore(state => state.total);
  return <span>Total: ${total.toFixed(2)}</span>;
}

function ProductCard({ product }: { product: Product }) {
  const addItem = useCartStore(state => state.addItem);
  return (
    <div>
      <h3>{product.name}</h3>
      <button onClick={() => addItem(product)}>Add to Cart</button>
    </div>
  );
}

Zustand Store Slices (Modular Stores)

// Split a large store into slices
const createAuthSlice = (set) => ({
  user: null,
  login: (user) => set({ user }),
  logout: () => set({ user: null }),
});

const createUISlice = (set) => ({
  theme: 'light',
  toggleTheme: () => set(s => ({ theme: s.theme === 'light' ? 'dark' : 'light' })),
});

const useStore = create((set, get) => ({
  ...createAuthSlice(set),
  ...createUISlice(set),
}));

Jotai — Atomic State Management

Jotai models state as small, independent atoms. Components subscribe to specific atoms and only re-render when those atoms change.

import { atom, useAtom, useAtomValue, useSetAtom } from 'jotai';
import { atomWithStorage } from 'jotai/utils';

// Primitive atoms
const countAtom = atom(0);
const nameAtom = atom('Alice');

// Derived (read-only) atom
const doubleCountAtom = atom((get) => get(countAtom) * 2);

// Async atom
const userAtom = atom(async (get) => {
  const id = get(userIdAtom);
  const response = await fetch(`/api/users/${id}`);
  return response.json();
});

// Persistent atom
const themeAtom = atomWithStorage('theme', 'light');

Using Jotai in Components

function Counter() {
  const [count, setCount] = useAtom(countAtom);
  const doubleCount = useAtomValue(doubleCountAtom); // read-only

  return (
    <div>
      <p>Count: {count}</p>
      <p>Double: {doubleCount}</p>
      <button onClick={() => setCount(c => c + 1)}>+</button>
    </div>
  );
}

// Only writes, doesn't cause re-render on count change
function IncrementButton() {
  const increment = useSetAtom(countAtom);
  return <button onClick={() => increment(c => c + 1)}>+</button>;
}

Zustand vs. Jotai: When to Use Each

  • Zustand — Best for global app state where data is naturally grouped (cart, auth, UI config). Simple mental model, great for teams familiar with Redux.
  • Jotai — Best when state is naturally fine-grained and you want precise re-render control. Excellent for form state, derived computations, and async data.

Best Practices

  • Select only the slice of state you need from Zustand to avoid unnecessary re-renders.
  • Use devtools middleware in Zustand for Redux DevTools debugging.
  • Combine Jotai atomic state with React Query for server state + local state.
  • Avoid placing all state in one giant Zustand store — use slices for modularity.
