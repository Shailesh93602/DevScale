Form State Management with React Hook Form

Forms are one of the most complex UI challenges. React Hook Form (RHF) provides a performant, flexible, and easy-to-use solution with minimal re-renders and seamless TypeScript + Zod validation integration.

Why React Hook Form?

  • Performance — Uses uncontrolled inputs (no re-render on every keystroke).
  • Less boilerplate — No manual onChange/value wiring.
  • TypeScript-native — Full type inference from your schema.
  • Zod integration — Validates with the same schema as your backend.

Basic Form Setup with Zod

import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';

// 1. Define validation schema
const loginSchema = z.object({
  email: z.string().email('Invalid email address'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
  rememberMe: z.boolean().default(false),
});

type LoginFormData = z.infer<typeof loginSchema>;

// 2. Use the form
function LoginForm() {
  const {
    register,
    handleSubmit,
    formState: { errors, isSubmitting, isDirty },
    reset,
  } = useForm<LoginFormData>({
    resolver: zodResolver(loginSchema),
    defaultValues: { email: '', password: '', rememberMe: false },
  });

  const onSubmit = async (data: LoginFormData) => {
    try {
      await authService.login(data);
      reset(); // clear form on success
    } catch (error) {
      // Handle server errors
    }
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <div>
        <input {...register('email')} placeholder="Email" />
        {errors.email && <p className="error">{errors.email.message}</p>}
      </div>

      <div>
        <input {...register('password')} type="password" placeholder="Password" />
        {errors.password && <p className="error">{errors.password.message}</p>}
      </div>

      <label>
        <input {...register('rememberMe')} type="checkbox" />
        Remember me
      </label>

      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? 'Logging in...' : 'Login'}
      </button>
    </form>
  );
}

Dynamic Field Arrays

import { useFieldArray } from 'react-hook-form';

const schema = z.object({
  name: z.string().min(1),
  tags: z.array(z.object({ value: z.string().min(1) })).min(1, 'Add at least one tag'),
});

function TagsForm() {
  const { control, register, formState: { errors } } = useForm({
    resolver: zodResolver(schema),
    defaultValues: { tags: [{ value: '' }] }
  });

  const { fields, append, remove } = useFieldArray({
    control,
    name: 'tags'
  });

  return (
    <form>
      {fields.map((field, index) => (
        <div key={field.id}>
          <input {...register(`tags.${index}.value`)} />
          <button type="button" onClick={() => remove(index)}>Remove</button>
        </div>
      ))}
      <button type="button" onClick={() => append({ value: '' })}>Add Tag</button>
      {errors.tags?.root && <p>{errors.tags.root.message}</p>}
    </form>
  );
}

Watch — Conditional Fields

function SignupForm() {
  const { register, watch } = useForm();
  const accountType = watch('accountType');

  return (
    <form>
      <select {...register('accountType')}>
        <option value="personal">Personal</option>
        <option value="business">Business</option>
      </select>

      {/* Only show company name for business accounts */}
      {accountType === 'business' && (
        <input {...register('companyName')} placeholder="Company name" />
      )}
    </form>
  );
}

Best Practices

  • Always pair React Hook Form with Zod for end-to-end type safety from DB to UI.
  • Use useFieldArray for dynamic lists (tags, line items, team members).
  • Use watch sparingly — it causes re-renders on every change for watched fields.
  • Use setError to surface server-side validation errors back into the form.
  • Use isDirty to disable the submit button when nothing has changed.
