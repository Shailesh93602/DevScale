Node.js Fundamentals

Node.js is a JavaScript runtime built on Chrome's V8 engine. It lets you run JavaScript on the server, enabling full-stack development with one language.

How Node.js Works

  • Single-threaded — one execution thread handles all requests.
  • Non-blocking I/O — I/O operations (disk, network) are asynchronous; Node doesn't wait.
  • Event Loop — constantly polls for completed async operations and executes their callbacks.
  • This makes Node excellent for I/O-heavy apps (APIs, chat, streaming) but not CPU-intensive tasks.

ES Modules vs CommonJS

// CommonJS (traditional Node.js style)
const express = require("express");
module.exports = { myFunc };

// ES Modules (modern — in package.json: "type": "module")
import express from "express";
export { myFunc };
export default router;

Node.js Streams

import { createReadStream, createWriteStream } from "fs";

// Stream a large file instead of loading it all into memory
const readable = createReadStream("large-file.csv");
const writable = createWriteStream("output.csv");
readable.pipe(writable); // Pipe data through

// HTTP response is a writable stream
res.write("chunk1");
res.write("chunk2");
res.end();
