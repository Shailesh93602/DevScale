Performance, Scaling & PM2

Node.js runs your entire server logic on a single thread. While the Event Loop handles I/O efficiently, actual JavaScript execution is restricted to one CPU core. If you deploy a standard Node.js app to an expensive 8-core AWS server, 7 of those cores will sit at 0% utilization while the 1 core maxes out at 100%. Node must be scaled intentionally.

The Native Cluster Module

The cluster module allows you to easily create child processes (Workers) that share the same server port. The master process acts as a load balancer, distributing incoming connections across the CPU cores in a round-robin fashion.

const cluster = require('cluster');
const os = require('os');
const express = require('express');

// Count the physical CPU cores available
const numCPUs = os.cpus().length;

if (cluster.isPrimary) {
  console.log(`Primary Master ${process.pid} is running`);

  // Fork a worker for every CPU core
  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  // Self-Healing: If a worker crashes, instantly spin up a replacement
  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died. Restarting...`);
    cluster.fork();
  });
} else {
  // This code runs inside the child workers. 
  // All 8 workers will share port 3000!
  const app = express();
  app.get('/', (req, res) => {
    res.send(`Hello from Worker ${process.pid}`);
  });
  app.listen(3000, () => console.log(`Worker ${process.pid} started`));
}

Production Daemonization with PM2

Writing clustering code manually is tedious. In established production environments, the community standard is PM2. PM2 is an advanced Process Manager that handles clustering, automatic restarts on crashes, logging, and zero-downtime reloads automatically.

# Install PM2 globally
npm install pm2 -g

# Start the app and automatically scale to the maximum number of CPU cores (-i max)
pm2 start server.js -i max --name "my-api"

# View real-time performance, memory, and CPU metrics
pm2 monit

# Save the configuration so the server restarts on machine reboot
pm2 save
pm2 startup

If you use PM2, you do not need to use the native cluster module in your code. PM2 wraps your standard Express app and clusters it behind the scenes.

Reverse Proxy with NGINX

Node.js is Excellent at application logic, but it is mediocre at serving static files (images, CSS) and managing SSL certificates (HTTPS). Standard architecture places an NGINX reverse proxy in front of Node.js.

  1. The user's browser connects to Nginx on Port 80/443.
  2. Nginx decrypts the HTTPS traffic securely.
  3. If the user requests an image, Nginx serves it directly (blazing fast, bypassing Node entirely).
  4. If the user requests an API route, Nginx forwards (proxies) the request to the hidden Node.js server instances running on Port 3000.

Caching with Redis

The fastest way to scale a database is to stop querying it. High-traffic reads should be cached in memory. Because clustered Node instances cannot share RAM, an external in-memory database like Redis is required.

const redis = require('redis');
const client = redis.createClient({ url: 'redis://localhost:6379' });
await client.connect();

app.get('/api/heavy-data', async (req, res) => {
  // 1. Check Redis Cache
  const cachedData = await client.get('dashboard_stats');
  if (cachedData) {
    return res.json({ source: 'cache', data: JSON.parse(cachedData) });
  }

  // 2. Cache Miss: Calculate heavy data from PostgeSQL (takes 3 seconds)
  const heavyData = await db.calculateMassiveStatistics();
  
  // 3. Save to Redis with an Expiration (Time to Live) of 60 seconds
  await client.setEx('dashboard_stats', 60, JSON.stringify(heavyData));

  // 4. Return to user
  res.json({ source: 'database', data: heavyData });
});

Best Practices

  • Your Express app must remain completely Stateless. Do not store sessions or files in application memory (variables), because in a clustered environment, Worker 1 cannot access Worker 2's memory.
  • Always use PM2 for production process management to ensure your app stays alive after unexpected fatal crashes or unhandled promise rejections.
  • Offload HTTPS decryption and static file serving to a Reverse Proxy (Nginx, AWS ALB, Cloudflare).
