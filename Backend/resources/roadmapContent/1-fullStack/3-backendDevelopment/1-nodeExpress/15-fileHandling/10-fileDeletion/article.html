Deleting Files with Node.js

In any application that allows users to upload files, you must also provide a way to delete them. Whether it's removing an old profile picture or cleaning up temporary data, proper file deletion is essential for managing server storage and respecting user privacy.

The fs.unlink() Method

Node.js provides the fs.unlink() method specifically for deleting files. Like most fs methods, it has both an asynchronous and a synchronous version.

const fs = require('fs');

// Asynchronous deletion
fs.unlink('./uploads/old-file.txt', (err) => {
  if (err) {
    console.error("Error deleting file:", err);
    return;
  }
  console.log("File deleted successfully!");
});

// Promises-based deletion
const fsPromises = require('fs').promises;
await fsPromises.unlink('./uploads/old-file.txt');

Deleting Files from Cloud Storage (AWS S3)

If you're using cloud storage, you must use the provider's SDK to delete files. For AWS S3, you use the DeleteObjectCommand.

const { S3Client, DeleteObjectCommand } = require('@aws-sdk/client-s3');
const s3 = new S3Client({ region: 'us-east-1' });

const deleteParams = {
  Bucket: 'my-app-uploads',
  Key: 'user-123/avatar.png'
};

try {
  await s3.send(new DeleteObjectCommand(deleteParams));
  console.log("File deleted from S3.");
} catch (err) {
  console.error("Error deleting from S3:", err);
}

Syncing File Deletion with the Database

In a real application, you usually have a database record representing the file. Always ensure you delete both the file and the database record to maintain data integrity.

  1. Delete the File: Initiate the deletion on the file system or cloud storage.
  2. Verify Deletion: Ensure the file was actually removed.
  3. Delete the DB Record: Once the file is gone, remove its path or URL from your database.

Security and Best Practices

  • Permissions Check: Never allow a user to delete a file simply by providing a path as an input. Always verify that the authenticated user has permission to delete the specific file.
  • Graceful Failure: If a file deletion fails because the file doesn't exist, don't necessarily crash your app. Often, it's better to log the warning and proceed (idempotency).
  • Garbage Collection: Consider setting up a background task to find and delete "orphan" files—files that exist on disk or in the cloud but have no corresponding record in your database.
