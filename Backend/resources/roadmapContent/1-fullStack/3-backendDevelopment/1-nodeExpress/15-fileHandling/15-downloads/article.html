Handling File Downloads Correctly

Serving a file for download isn't just about sending its content. To ensure a professional and safe experience, you must set the correct HTTP headers to instruct the browser how to handle the file (e.g., whether to open it in a new tab or save it to disk).

Using res.download()

Express provides a convenient res.download() helper that sets the correct Content-Disposition header (which defaults to "attachment") and sends the file in a single call.

app.get('/download/invoice/:id', async (req, res) => {
  const filePath = path.join(__dirname, 'invoices', `invoice-${req.params.id}.pdf`);
  
  // Set the filename that the user will see in their download folder
  res.download(filePath, 'Your-Invoice.pdf', (err) => {
    if (err) {
      // Handle the error if the file cannot be found or sent
      res.status(404).send("Could not find the invoice.");
    }
  });
});

Setting the Content-Type Header

Always set the correct Content-Type (MIME type) so the user's computer knows which application to use for opening the file. Express's res.download() and res.sendFile() do this automatically based on the file extension, but you can also set it manually.

Secure Downloads: Avoiding IDOR

Never allow a user to download a file simply by providing a path as an input (e.g., /get-file?path=secret.env). This is a massive security hole. Instead, use internal, non-guessable IDs (like UUIDs) and always verify the user's permission in the database before serving the file.

Advanced Strategy: Serving from S3 via Redirect

Instead of downloading the file to your server and then sending it to the user, you can redirect the user directly to a Pre-signed URL from your cloud storage provider. This offloads the bandwidth cost and server load to the cloud provider.

app.get('/download/photo/:id', async (req, res) => {
  // 1. Authenticate user and verify permission
  // 2. Generate a temporary, private S3 URL for the specific photo
  const s3Url = await generatePresignedUrl(`photos/${req.params.id}.jpg`);
  
  // 3. Redirect the user's browser directly to S3
  res.redirect(s3Url);
});

Best Practices Summary

  1. Use Descriptive Names: Use the second argument of res.download() to provide a user-friendly name, rather than your internal system name (e.g., use Project-Brief.pdf instead of upload-234234.pdf).
  2. Rate Limit Downloads: To prevent bots from scraping all your files, implement rate limiting specifically on your download endpoints.
  3. Check for Permissions Every Time: Never cache a download link's permission. A user who could access a file yesterday shouldn't necessarily be able to today if their permissions have changed.
