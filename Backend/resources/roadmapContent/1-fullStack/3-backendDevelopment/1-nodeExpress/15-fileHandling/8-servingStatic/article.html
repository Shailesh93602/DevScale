Serving Static Files in Express

Static files are files that the server sends to the client exactly as they are stored on disk, without any dynamic processing. Common examples include images, CSS stylesheets, client-side JavaScript files, and HTML templates.

Using the express.static Middleware

Express provides a built-in middleware called express.static to serve files from a specific directory. When a request comes in, Express checks this directory for a matching file. If it finds one, it sends the file; if not, it passes the request to the next middleware.

const express = require('express');
const path = require('path');
const app = express();

// Serve files from the 'public' directory
app.use(express.static(path.join(__dirname, 'public')));

// Now you can access files like:
// http://localhost:3000/images/logo.png
// http://localhost:3000/css/styles.css

Virtual Path Prefixes

Sometimes you want to serve static files under a specific URL prefix that doesn't exist on your file system. You can do this by providing a mount path to app.use:

// Serve files from 'public' under the '/static' URL path
app.use('/static', express.static(path.join(__dirname, 'public')));

// Access via: http://localhost:3000/static/images/logo.png

Caching for Performance

Since static files don't change often, you should tell the browser to cache them. This significantly reduces server load and improves page transitions for your users. You can configure cache headers directly in express.static:

const options = {
  dotfiles: 'ignore', // Ignore files starting with a dot (like .env)
  etag: true,
  maxAge: '1d', // Cache for 1 day
  redirect: false,
  setHeaders: function (res, path, stat) {
    res.set('x-timestamp', Date.now())
  }
}

app.use(express.static('public', options));

Best Practices

  • Directory Isolation: Always keep your static files in a dedicated directory (like public or assets) that is separate from your application logic and source code.
  • Absolute Paths: Always use path.join(__dirname, 'public') to ensure your application can correctly find the directory regardless of where the Node.js process is started from.
  • Security: Be extremely careful not to serve directories that contain sensitive information like configuration files, environment variables, or private data.
  • Use a CDN in Production: For high-traffic applications, consider using a dedicated Content Delivery Network (CDN) to serve your static assets instead of your primary Node.js server.
