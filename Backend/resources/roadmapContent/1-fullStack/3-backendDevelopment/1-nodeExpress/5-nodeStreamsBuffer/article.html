Node Streams & Buffers

When dealing with large amounts of data, reading an entire file into memory before processing it is a recipe for crashing your server with an "Out of Memory" error. Node.js solves this with Buffers and Streams, allowing you to process data piece by piece (chunk by chunk).

What is a Buffer?

JavaScript originally had no mechanism for manipulating raw binary data. A Buffer is a globally available Node.js class that represents a fixed-size chunk of memory allocated outside V8's heap. Buffers store raw binary data (represented as hexadecimal arrays).

// Creating a buffer from a string
const buf = Buffer.from('Hello World', 'utf8');
console.log(buf); // <Buffer 48 65 6c 6c 6f 20 57 6f 72 6c 64> (Hex representations of characters)
console.log(buf.toString()); // 'Hello World'

// Allocating a specific size
const emptyBuf = Buffer.alloc(10); // 10 bytes initialized to zero

The Four Types of Streams

Streams are collections of data that might not be available all at once and don't have to fit in memory. They extend the EventEmitter class.

  1. Readable: Data can be read (e.g., fs.createReadStream(), req in Express).
  2. Writable: Data can be written to (e.g., fs.createWriteStream(), res in Express).
  3. Duplex: Both Readable and Writable (e.g., net.Socket).
  4. Transform: A type of Duplex stream where the output is calculated from the input (e.g., zlib.createGzip()).

The Memory Crash (Reading files synchronously)

Imagine reading a 2GB video file to serve to a client. fs.readFileSync() will attempt to load all 2GB into the server's RAM. If 5 users request the video, your server uses 10GB of RAM and crashes instantly.

// ❌ TERRIBLE for large files
app.get('/video', (req, res) => {
  const file = fs.readFileSync('movie.mp4'); 
  res.send(file); 
});

The Solution: Piping Streams

The pipe() method is the easiest way to consume streams. It takes data from a Readable stream and writes it to a Writable stream chunk by chunk automatically handling backpressure.

// ✅ EXCELLENT for large files
app.get('/video-stream', (req, res) => {
  // Sets correct header for video
  res.writeHead(200, { 'Content-Type': 'video/mp4' });
  
  // Creates a stream. Data is read in ~64kb chunks.
  const src = fs.createReadStream('movie.mp4');
  
  // As chunks are read, they are piped DIRECTLY to the response object 
  // (which is a Writable stream). Memory usage stays under 1MB!
  src.pipe(res);
});

Handling Backpressure

Backpressure occurs when the pipeline reads data from a source much faster than it can write it to the destination. Because network conditions vary, fs.createReadStream() can read a disk file at 100MB/s, but the user's internet can only receive 2MB/s.

The pipe() method handles backpressure automatically. If the writable stream's internal buffer fills up, .pipe() tells the readable stream to pause reading until the writer drains its buffer.

Event-Based Stream Processing

Sometimes you need to intercept the stream data to modify or log it before sending it. You can listen to the data and end events.

const src = fs.createReadStream('massive-log-file.txt');

let chunkCount = 0;

// Every time a new Buffer chunk is ready, this fires
src.on('data', (chunk) => {
  chunkCount++;
  console.log(`Received ${chunk.length} bytes of raw data`);
  // chunk is a Buffer. You can call chunk.toString() if it's text.
});

src.on('end', () => {
  console.log(`Finished reading. Received ${chunkCount} chunks.`);
});

src.on('error', (err) => {
  console.error('An error occurred during streaming', err);
});

Transform Streams (Zipping on the fly)

Transform streams are extremely powerful. You can pipe a file read stream into a gzip transform stream, and pipe the result directly to the HTTP response, compressing data on the fly.

const zlib = require('zlib');

app.get('/download-zipped', (req, res) => {
  res.writeHead(200, {
    'Content-Type': 'application/gzip',
    'Content-Disposition': 'attachment; filename="data.txt.gz"'
  });

  const rawFile = fs.createReadStream('data.txt');
  const zipper = zlib.createGzip(); // Transform Stream

  // Read File -> Zip It -> Send to Client
  rawFile.pipe(zipper).pipe(res);
});
