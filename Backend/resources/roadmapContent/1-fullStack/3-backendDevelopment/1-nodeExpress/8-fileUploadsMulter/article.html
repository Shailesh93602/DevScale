File Uploads with Multer

By default, Express cannot process files. When a user uploads a file, the browser sends the data using the multipart/form-data encoding type, rather than the standard application/json. To parse this format in Node.js, the community standard middleware is Multer.

Basic Multer Setup (Saving to Disk)

Multer intercepts the request before it hits your controller, parses the binary file data, and attaches a req.file (or req.files) object for you to use.

const express = require('express');
const multer  = require('multer');
const app = express();

// 1. Configure where and how to save the file
const storage = multer.diskStorage({
  destination: function (req, file, cb) {
    // Callback argument 2 is the destination folder
    cb(null, './uploads/') 
  },
  filename: function (req, file, cb) {
    // Highly recommended to rename the file to avoid collisions and directory traversal attacks
    const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
    cb(null, file.fieldname + '-' + uniqueSuffix + '-' + file.originalname)
  }
});

const upload = multer({ storage: storage });

// 2. Use the middleware in the route (upload.single expects one file under the 'avatar' field name)
app.post('/profile', upload.single('avatar'), (req, res) => {
  // req.file is the `avatar` file
  // req.body will hold any other text fields (if there were any)
  console.log(req.file);
  
  // File is already saved to disk by the time this runs!
  res.json({ message: "Successfully uploaded", path: req.file.path });
});

Security: Validating File Types and Sizes

Never trust the file extension provided by the user. A hacker can rename malware.exe to image.png. You must validate the MIME type, and restrict the upload size to prevent Denial of Service (DOS) attacks by filling up your hard drive.

const uploadFilter = multer({ 
  storage: storage,
  // Limit file size to 2 Megabytes
  limits: { fileSize: 2 * 1024 * 1024 },
  
  // Custom filter function
  fileFilter: (req, file, cb) => {
    // Check the MIME type
    if (file.mimetype === "image/png" || file.mimetype === "image/jpeg") {
      cb(null, true); // Accept file
    } else {
      cb(new Error("Only .png and .jpg format allowed!"), false); // Reject file
    }
  }
});

Handling Upload Errors

If a file is rejected by the filter or exceeds the size limit, Multer will throw an error that automatically skips your controller and hits your Global Error Handler. If you want to handle it directly in the route:

app.post('/upload', (req, res) => {
  // Execute the multer middleware manually
  uploadFilter.single('document')(req, res, function (err) {
    if (err instanceof multer.MulterError) {
      // A Multer error occurred when uploading (e.g. file too large).
      return res.status(400).json({ error: err.message });
    } else if (err) {
      // An unknown error occurred (like our custom mimetype error).
      return res.status(400).json({ error: err.message });
    }

    // Everything went fine
    res.json({ success: true });
  })
});

Uploading Directly to Cloud Storage (AWS S3)

Saving files to your own server's disk (diskStorage) is a bad practice in modern distributed architectures. If your server restarts or scales to multiple instances, the files will be lost or unsynced. Instead, stream the file directly from the user to Amazon S3 using multer-s3.

const multerS3 = require('multer-s3');
const { S3Client } = require('@aws-sdk/client-s3');

const s3 = new S3Client({ region: 'us-east-1', credentials: { /* ... */ } });

const uploadToS3 = multer({
  storage: multerS3({
    s3: s3,
    bucket: 'my-app-uploads-bucket',
    metadata: function (req, file, cb) {
      cb(null, { fieldName: file.fieldname });
    },
    key: function (req, file, cb) {
      // File name in S3
      cb(null, Date.now().toString() + '-' + file.originalname)
    }
  })
});

app.post('/cloud-upload', uploadToS3.single('video'), (req, res) => {
  // req.file.location contains the public URL to the S3 object!
  res.json({ s3Url: req.file.location });
});

Best Practices

  • Always enforce strict limits.fileSize in Multer to prevent memory/disk exhaustion attacks.
  • If accepting images, strongly consider piping the Multer memory buffer through a library like sharp to compress and strip EXIF data (GPS coordinates) before saving it.
  • Use Cloud Storage (S3, Cloudinary) rather than local disk storage to ensure your Node.js application remains stateless and horizontally scalable.
