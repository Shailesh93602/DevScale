API Security Best Practices

Security is not optional. A single vulnerability can compromise all your users' data. These are the must-implement protections for every production API.

1. Helmet.js — Security Headers

import helmet from "helmet";
app.use(helmet()); // Sets 11+ security HTTP headers automatically

// What helmet sets:
// X-Content-Type-Options: nosniff
// X-Frame-Options: DENY
// Content-Security-Policy: ...
// Strict-Transport-Security: max-age=15552000
// And more...

2. CORS — Control Who Can Call Your API

import cors from "cors";

app.use(cors({
  origin: ["https://myapp.com", "https://www.myapp.com"],
  methods: ["GET", "POST", "PUT", "PATCH", "DELETE"],
  allowedHeaders: ["Content-Type", "Authorization"],
  credentials: true, // Allow cookies
}));

3. Rate Limiting — Prevent Brute Force & DoS

import rateLimit from "express-rate-limit";

// General API limit
const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100,                  // 100 requests per window
  standardHeaders: true,
  message: { error: "Too many requests, try again later" },
});

// Stricter for auth endpoints
const authLimiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 5 });

app.use("/api", apiLimiter);
app.use("/api/auth/login", authLimiter);

4. Input Validation & Sanitization

import { z } from "zod"; // Zod schema validation

const createUserSchema = z.object({
  email: z.string().email().max(255),
  password: z.string().min(8).max(64),
  name: z.string().min(2).max(100).regex(/^[a-zA-Z\s]+$/),
});

router.post("/users", async (req, res) => {
  const result = createUserSchema.safeParse(req.body);
  if (!result.success) {
    return res.status(422).json({ errors: result.error.flatten() });
  }
  // result.data is now typed and validated
  await UserService.create(result.data);
});

5. SQL Injection Prevention

// ❌ NEVER do this — raw string interpolation
const user = await db.query(`SELECT * FROM users WHERE email = '${email}'`);

// ✅ Use parameterized queries (your ORM does this automatically)
const user = await prisma.user.findUnique({ where: { email } });
const user = await db.query("SELECT * FROM users WHERE email = $1", [email]);

6. OWASP Top 10 Quick Reference

  • Injection — Use parameterized queries and ORMs.
  • Broken Auth — Use bcrypt, short-lived JWTs, MFA.
  • Sensitive Data — HTTPS everywhere, never log passwords.
  • XSS — Sanitize output, use Content-Security-Policy.
  • CSRF — Use SameSite cookies, CSRF tokens.
  • Security Misconfiguration — Use Helmet, disable X-Powered-By.
  • Vulnerable Dependencies — Run npm audit regularly.
