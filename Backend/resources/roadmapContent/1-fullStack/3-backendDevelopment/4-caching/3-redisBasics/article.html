Understanding Redis Basics

Redis (Remote Dictionary Server) is an open-source, in-memory data structure store. It can be used as a database, a cache, and a message broker. Because it stores data in RAM rather than on a hard disk, it is incredibly fast, capable of handling millions of operations per second.

Core Data Structures

Unlike a traditional database that uses tables and rows, Redis uses simple key-value pairs where the value can be one of several data structures:

  • Strings: The simplest type (like SET name "Redis").
  • Lists: Ordered collections of strings (perfect for timelines or queues).
  • Sets: Unordered collections of unique strings (good for tracking user IDs).
  • Hashes: Maps between string fields and string values (like an object in JavaScript).
  • Sorted Sets: Similar to Sets but every member is associated with a score, allowing them to be ranked.

Key Features of Redis

  • In-Memory: Data is served directly from RAM, achieving microsecond latency.
  • Persistence: While primarily in-memory, Redis can periodically save data to disk to prevent loss on restart.
  • Atomic Operations: Operations on the same key are guaranteed to be atomic, preventing race conditions.
  • Pub/Sub: A built-in messaging system for real-time applications.

Redis in the Modern Stack

In a typical full-stack architecture, Redis sits between your application server (Express) and your primary database (Postgres/Mongo). Its primary role is to cache the results of expensive queries so the primary database isn't overwhelmed.

Best Practices Summary

  1. Use Descriptive Keys: Use a naming convention like user:123:profile to keep your data organized.
  2. Monitor Memory Usage: Because RAM is expensive and limited, monitor your Redis memory usage and configure an eviction policy (like allkeys-lru) to automatically delete old data when the cache is full.
  3. Don't Use it for Everything: While fast, Redis doesn't have the sophisticated query capabilities or strong relational guarantees of a database like PostgreSQL. Use it for performance, not primary data storage.
