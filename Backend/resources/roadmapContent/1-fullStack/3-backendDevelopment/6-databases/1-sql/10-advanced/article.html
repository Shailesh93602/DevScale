Mastering Advanced SQL Queries

Standard CRUD operations cover 90% of your daily work, but truly complex data analysis and application logic require mastering advanced SQL techniques. These allow you to perform powerful operations directly within the database engine, often with incredible efficiency.

1. Aggregate Functions and GROUP BY

Aggregate functions perform a calculation on a set of values and return a single value. When combined with GROUP BY, they provide powerful reporting capabilities.

  • COUNT(): Returns the number of rows.
  • SUM(): Returns the total sum of a numeric column.
  • AVG(): Returns the average value.
  • HAVING: Like a WHERE clause but for groups (used *after* GROUP BY).
-- Find categories with more than 10 products
SELECT category_id, COUNT(*) 
FROM products 
GROUP BY category_id 
HAVING COUNT(*) > 10;

2. Subqueries and CTEs (Common Table Expressions)

A Subquery is a query inside another query. A CTE (using the WITH clause) provides a cleaner, more readable way to define temporary result sets that can be used within a larger query.

-- Example CTE
WITH RecentSales AS (
    SELECT * FROM orders WHERE created_at > NOW() - INTERVAL '30 days'
)
SELECT u.name, rs.total 
FROM users u 
JOIN RecentSales rs ON u.id = rs.user_id;

3. Window Functions

Window functions perform a calculation across a set of table rows that are somehow related to the current row. This is incredibly powerful for tasks like calculating running totals or ranking items within categories.

-- Rank products within their category by price
SELECT name, category_id, price,
RANK() OVER (PARTITION BY category_id ORDER BY price DESC) as price_rank
FROM products;

4. UNION and INTERSECT

  • UNION: Combines the results of two SELECT statements into a single result set (removing duplicates).
  • UNION ALL: Like UNION but keeps all duplicates (faster).
  • INTERSECT: Returns only the rows that appear in both SELECT statements.

Best Practices Summary

  1. Prefer CTEs over Complex Subqueries: CTEs make your code much easier to read, maintain, and debug.
  2. Understand NULLs: Always be aware of how NULL values interact with aggregate functions (they are usually ignored) and WHERE clauses.
  3. Avoid Correlated Subqueries: A correlated subquery (one that runs for every row of the outer query) is incredibly slow. Try to rewrite it as a JOIN whenever possible.
