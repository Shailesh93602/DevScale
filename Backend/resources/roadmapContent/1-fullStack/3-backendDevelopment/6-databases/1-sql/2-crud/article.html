Mastering SQL CRUD Operations

CRUD (Create, Read, Update, Delete) are the four fundamental operations of persistent storage. In SQL, these map to the core statements: INSERT, SELECT, UPDATE, and DELETE. Understanding these in depth is essential for any backend developer.

1. CREATE (INSERT)

The INSERT INTO statement is used to add new rows of data to a table.

-- Basic Insert
INSERT INTO users (name, email, age) 
VALUES ('Jane Smith', 'jane@example.com', 25);

-- Inserting multiple rows at once (Bulk Insert)
INSERT INTO users (name, email) 
VALUES ('User A', 'userA@test.com'), ('User B', 'userB@test.com');

2. READ (SELECT)

The SELECT statement is the most powerful and complex part of SQL. It allows you to retrieve exactly the data you need using various clauses.

  • WHERE: Filters the results (e.g., WHERE age > 18).
  • ORDER BY: Sorts the results (e.g., ORDER BY created_at DESC).
  • LIMIT: Limits the number of rows returned (useful for pagination).
  • GROUP BY: Groups rows that have the same values into summary rows (used with aggregate functions like COUNT, SUM, AVG).
SELECT count(*), status FROM orders GROUP BY status;

3. UPDATE

The UPDATE statement modifies existing records in a table. CRITICAL: Always use a WHERE clause with UPDATE, or you will update every single row in the table!

UPDATE users 
SET age = 26, is_verified = true 
WHERE id = 456;

4. DELETE

The DELETE statement removes rows from a table. Just like UPDATE, always use WHERE to avoid deleting your entire database.

DELETE FROM users WHERE last_login < '2023-01-01';

Note: In many professional apps, we use a "Soft Delete" (setting a deleted_at timestamp) instead of actually removing the row from the database.

Best Practices Summary

  1. Always use WHERE: When performing UPDATE or DELETE, double-check your WHERE clause to prevent accidental data loss.
  2. Use Transactions: For operations that involve multiple related changes (e.g., transferring money between two accounts), wrap them in a BEGIN TRANSACTION and COMMIT to ensure they either all succeed or all fail together.
  3. Sanitize Inputs: Never concatenate user input directly into your SQL queries. Use Parameterized Queries (prepared statements) to prevent SQL Injection attacks.
