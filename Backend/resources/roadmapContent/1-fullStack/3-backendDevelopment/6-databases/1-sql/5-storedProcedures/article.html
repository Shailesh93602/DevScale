Advanced SQL: Stored Procedures

A Stored Procedure is a prepared SQL code that you can save so that the code can be reused over and over again. Instead of sending long and complex queries from your application server, you just call the procedure by its name.

What is a Stored Procedure?

Think of a stored procedure like a "function" for your database. It can accept parameters, perform complex internal logic (using IF...ELSE or loops), and return multiple data sets or individual values.

Benefits of Stored Procedures

  • Reduced Network Traffic: Only the procedure call is sent over the network, rather than a massive query.
  • Improved Performance: Many databases "pre-compile" and optimize the execution plan for stored procedures.
  • Security: You can grant users permission to execute a procedure without giving them direct access to the underlying tables. This helps prevent SQL Injection.
  • Consistency: Centralizing business logic in the database ensures that every application (e.g., website, mobile app, internal tool) uses the exact same logic.

Creating a Simple Stored Procedure

-- Creating a procedure to fetch user activity
CREATE PROCEDURE GetUserActivity(IN userId INT)
BEGIN
    SELECT * FROM logins WHERE user_id = userId;
    SELECT * FROM posts WHERE author_id = userId;
END;

-- Calling the procedure from SQL
CALL GetUserActivity(123);

When to Use Stored Procedures?

Stored procedures are excellent for complex, multi-step operations that must be performed as a single transaction. However, modern developers often prefer to keep their "business logic" in their application code (Node.js) rather than the database for easier version control and testing.

Best Practices Summary

  1. Avoid Overusing: Don't put *all* your logic in the database. Use procedures sparingly for performance-critical or security-sensitive operations.
  2. Keep it Generic: Use parameters effectively to make your procedures reusable.
  3. Document Carefully: Since stored procedures aren't stored in your application's Git repository, ensure you have a clear way to track and document their changes.
