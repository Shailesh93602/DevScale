Mastering NoSQL Indexing

Indexing is the single most important tool for ensuring your NoSQL database remains fast as your data grows. Without an index, the database must perform a "Collection Scan" (reading every single document in a collection) for every query. With an index, the database can jump directly to the relevant records.

1. Single Field Index

The simplest type of index. You index a single field (like email or username) to make searches for that exact field blazing fast.

db.users.createIndex({ email: 1 });

2. Compound Index

If you frequently query by multiple fields together (e.g., finding "completed orders for a specific user"), a Compound Index is much more efficient than two separate single-field indexes.

db.orders.createIndex({ user_id: 1, status: 1 });

3. Multikey Index (Indexing Arrays)

NoSQL databases like MongoDB allow you to index fields that contain arrays. This allows you to efficiently search for documents that contain a specific element in an array (e.g., finding all posts with the tag "Javascript").

db.posts.createIndex({ tags: 1 });

4. TTL (Time To Live) Index

A specialized index that automatically deletes documents from a collection after a certain period of time. This is perfect for logs, sessions, and temporary event data.

db.sessions.createIndex({ "createdAt": 1 }, { expireAfterSeconds: 3600 });

5. Sparse and Partial Indexes

  • Sparse Index: Only includes documents that actually contain the indexed field. This saves massive amounts of storage if the field only exists in 1% of your documents.
  • Partial Index: Only indexes documents that meet a specific filter criteria (e.g., only index "active" users).

Best Practices Summary

  1. Index your Queries, Not your Data: Only create an index for fields your application actually uses for filtering or sorting. Every index consumes memory and slows down write operations.
  2. Rule of Thumb: If your query takes longer than 50ms locally, it almost certainly needs an index.
  3. Use Compound Indexes Strategically: The order of fields in a compound index matters! Put the most frequently filtered fields first.
  4. Monitor Index Usage: Use your database's profiling tools (like $indexStats in MongoDB) to find "unused" indexes and delete them to reclaim resources.
