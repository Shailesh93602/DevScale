MongoDB — Document Database

MongoDB is a NoSQL document database that stores data as flexible JSON-like documents (BSON). Unlike relational DBs, there's no fixed schema — documents in the same collection can have different fields.

Core Concepts

  • Database → contains collections
  • Collection → like a SQL table, holds documents
  • Document → a JSON object with key-value pairs
  • _id → auto-generated unique identifier for each document

CRUD Operations (MongoDB Shell)

// Insert
db.users.insertOne({ name: "Alice", age: 25, role: "admin" });
db.users.insertMany([{ name: "Bob" }, { name: "Carol" }]);

// Read
db.users.findOne({ name: "Alice" });
db.users.find({ age: { $gte: 18 } }).sort({ name: 1 }).limit(10);

// Update
db.users.updateOne({ name: "Alice" }, { $set: { age: 26 } });
db.users.updateMany({ role: "guest" }, { $set: { active: false } });

// Delete
db.users.deleteOne({ name: "Bob" });
db.users.deleteMany({ active: false });

Query Operators

$eq, $ne         // equal, not equal
$gt, $gte        // greater than (or equal)
$lt, $lte        // less than (or equal)
$in, $nin        // in array / not in array
$and, $or, $not  // logical operators
$exists          // field exists check

Aggregation Pipeline

Use pipeline stages to transform documents:

db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$userId", total: { $sum: "$amount" } } },
  { $sort: { total: -1 } },
  { $limit: 5 }
]);

Indexing for Performance

db.users.createIndex({ email: 1 }, { unique: true });
db.users.createIndex({ name: "text" }); // full-text search

When to Use MongoDB

  • ✅ Schema flexibility needed (user-generated content, CMS)
  • ✅ Nested/hierarchical data that doesn't fit rows/columns
  • ✅ Horizontal scaling across many servers
  • ❌ Complex multi-table transactions (use PostgreSQL instead)
