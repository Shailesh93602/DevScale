Git Workflows & Team Collaboration

Git is the industry-standard version control system. Mastering Git workflows is essential for collaborating in professional teams.

Core Git Commands

git init                      # Initialise new repository
git clone <url>               # Clone remote repository

git status                    # See changed files
git diff                      # See exact changes

git add .                     # Stage all changes
git add src/app.ts            # Stage specific file

git commit -m "feat: add login endpoint"
git commit --amend            # Edit last commit message

git log --oneline --graph     # Visual branch history
git show HEAD                 # Inspect latest commit

Branching

git branch                    # List branches
git checkout -b feature/auth  # Create + switch to new branch
git switch main               # Switch branch (modern syntax)

git merge feature/auth        # Merge into current branch
git rebase main               # Reapply commits on top of main (cleaner history)

git branch -d feature/auth    # Delete branch after merge

Professional Git Workflow (GitFlow)

# Branches:
main      — Production-ready code only
develop   — Integration branch for features
feature/* — Individual feature branches
hotfix/*  — Emergency production fixes
release/* — Release preparation

# Standard workflow:
git checkout develop
git checkout -b feature/user-auth

# ... make commits ...

git push origin feature/user-auth
# → Open pull request on GitHub/GitLab
# → Code review by team
# → Merge into develop

Resolving Merge Conflicts

# When git can't auto-merge:
git merge feature/auth
# CONFLICT: Merge conflict in src/routes/users.ts

# Conflict markers in file:
<<<<<<< HEAD (current branch)
  const users = await getUsers();
=======
  const users = await fetchAllUsers();
>>>>>>> feature/auth

# Fix: choose the right version, remove markers, then:
git add src/routes/users.ts
git commit -m "resolve merge conflict"

Commit Message Convention

# Conventional Commits — parseable by tools
feat:     new feature
fix:      bug fix
docs:     documentation changes
style:    formatting, no logic change
refactor: code restructuring, no behavior change
test:     adding/updating tests
chore:    build process, tooling changes

# Examples:
feat: add JWT refresh token endpoint
fix: resolve null pointer in user repository
feat(auth): add Google OAuth login
fix(api)!: breaking change — rename userId to user_id
