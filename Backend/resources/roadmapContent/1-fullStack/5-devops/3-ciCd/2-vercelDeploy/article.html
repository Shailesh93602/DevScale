Vercel Deployment

Vercel is a cloud platform optimized for frontend and full-stack deployments. It provides zero-config deployments, automatic preview URLs for every git branch, serverless functions, and global CDN — making it the go-to platform for Next.js apps.

Why Vercel?

  • ✅ Zero config — auto-detects framework (Next.js, Vite, etc.)
  • ✅ Preview deployments — every PR gets its own URL
  • ✅ Edge Network — files served from CDN close to users globally
  • ✅ Serverless Functions — API routes run as serverless lambdas
  • ✅ Analytics & Logs — built-in performance and error monitoring

Deploying via Vercel CLI

# Install CLI globally
npm i -g vercel

# Login
vercel login

# Deploy from project root
vercel            # preview deployment
vercel --prod    # production deployment

Connecting GitHub for Auto-Deploy

  1. Go to vercel.com → Import Project
  2. Choose your GitHub repository
  3. Vercel auto-deploys on every push to main
  4. Pull requests get unique preview URLs automatically

Environment Variables

# vercel.json — project config
{
  "env": {
    "DATABASE_URL": "@database-url"
  },
  "build": {
    "env": {
      "NEXT_PUBLIC_API_URL": "https://api.myapp.com"
    }
  }
}

Set secrets via Vercel Dashboard → Settings → Environment Variables, or via CLI:

vercel env add DATABASE_URL production

Vercel Serverless Functions

Any file in /api (or Next.js app/api/route.ts) becomes a serverless endpoint:

// app/api/hello/route.ts
export async function GET(request: Request) {
  return Response.json({ message: 'Hello from Vercel!' });
}

Custom Domains

# Add domain via CLI
vercel domains add myapp.com

# Or via Dashboard → Project → Settings → Domains

vercel.json Configuration

{
  "rewrites": [
    { "source": "/api/:path*", "destination": "https://backend.myapp.com/api/:path*" }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [{ "key": "X-Content-Type-Options", "value": "nosniff" }]
    }
  ]
}

Best Practices

  • Use NEXT_PUBLIC_ prefix for env vars accessible in the browser.
  • Keep serverless functions under 50MB and execution under 10s.
  • Use Vercel Edge Middleware for auth checks without cold starts.
  • Enable Vercel Speed Insights to track Core Web Vitals per page.
