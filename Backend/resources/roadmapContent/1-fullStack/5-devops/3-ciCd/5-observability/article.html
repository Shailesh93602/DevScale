Observability & APM

When you are building an app locally on your laptop, and your code crashes, you immediately see the stack trace in your VSCode terminal window. You fix the bug in 5 seconds.

When your app is deployed to a server in Virginia handling 10,000 users an hour, and a single user's payment fails at 3:00 AM, how do you know it happened? How do you see the exact error message? You cannot physically stare at the server logs 24/7. Your application must be Observable.

1. Error Tracking (Sentry)

If an unhandled exception occurs in production, it is unacceptable for the user to just see a blank white screen while you sleep. The industry standard tool for instantly capturing these bugs is Sentry.

You install the Sentry SDK into your Node.js or React application:

import * as Sentry from "@sentry/node";

Sentry.init({ dsn: "https://your-unique-sentry-url.com" });

app.get("/checkout", async (req, res) => {
  try {
    await processPayment();
  } catch (error) {
    // 1. Instantly transmit the error mathematically to Sentry's dashboard!
    Sentry.captureException(error);
    
    // 2. Return a generic safe error to the user gracefully.
    res.status(500).send("Payment failed. Our engineers have been securely notified.");
  }
});

When the error fires, Sentry instantly aggressively records the Exact Line Number, the physical Browser the user was using natively, their Operating System, and sends a critical Slack notification to the engineering team.

2. Structured Logging (Winston)

You must stop using `console.log()` in production entirely. It is synchronous, blocking, unsearchable, and completely unstructured.

Instead, Enterprise backend systems utilize libraries like Winston or Pino. These log precisely formatted JSON objects, allowing massive centralized aggregators (like ELK stack or Datadog) to instantly filter millions of logs.

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.json(),
  transports: [
    new winston.transports.File({ filename: 'combined.log' }),
  ],
});

// Logs structured JSON: {"level":"error","message":"Database timeout","timestamp":"2023-11-04T12:00:00Z"}
logger.error('Database timeout');
