Structural Design Patterns

Structural patterns explain how to assemble objects and classes into larger, more complex structures, while keeping these structures completely flexible and efficient.

1. The Adapter Pattern

The Adapter pattern allows objects with incompatible interfaces to successfully collaborate. It acts as a wrapper between an existing class and a newly required interface.

For example, imagine you have a legacy system that expects data in XML format, but your modern API only provides data in JSON.

// The modern API providing JSON data
class ModernApi {
  public getJsonData(): string {
    return '{"user": "John", "age": 30}';
  }
}

// The legacy system that explicitly requires XML
interface LegacySystem {
  processXmlData(xml: string): void;
}

// The Adapter wrapper smoothly translates between the two incompatible systems
class JsonToXmlAdapter implements LegacySystem {
  private api: ModernApi;

  constructor(api: ModernApi) {
    this.api = api;
  }

  public processXmlData(): void {
    const json = this.api.getJsonData();
    // Complex mathematical conversion logic happens here...
    const xml = `<user>John</user><age>30</age>`;
    
    console.log("Safely processing adapted XML data:", xml);
  }
}

const api = new ModernApi();
const adapter = new JsonToXmlAdapter(api);
adapter.processXmlData(); // Successfully executes dynamically!

2. The Decorator Pattern

The Decorator pattern lets you mathematically attach new behaviors or responsibilities to individual objects dynamically at runtime, without explicitly altering their actual code or completely rewriting the entire class hierarchy.

interface Notifier { send(message: string): void; }

// The Concrete Component
class BasicEmailNotifier implements Notifier {
  send(message: string) { console.log(`Sending email: ${message}`); }
}

// The Base Decorator safely holding a reference to the wrapped object
class NotifierDecorator implements Notifier {
  protected wrappee: Notifier;
  constructor(notifier: Notifier) { this.wrappee = notifier; }
  send(message: string) { this.wrappee.send(message); }
}

// A Concrete Decorator successfully adding SMS functionality dynamically
class SmsDecorator extends NotifierDecorator {
  send(message: string) {
    super.send(message);
    console.log(`Sending SMS fallback safely: ${message}`);
  }
}

// Usage: Dynamically layering behaviors
let notifier = new BasicEmailNotifier();
notifier = new SmsDecorator(notifier);

// This cleanly securely triggers BOTH the Email and the SMS perfectly
notifier.send("Server down!");

3. The Facade Pattern

The Facade Pattern completely shields client code from the terrifying complexities of a massive, heavily nested subsystem by providing a single, simplified, high-level interface.

Instead of manually calling highly complex classes for Video Formatting, Rendering, Codec Extraction, and Audio Syncing, you strictly build a simplified `VideoConverterFacade` wrapper class that cleanly completely hides all those complex methods.

You then simply call `facade.convertFile("myvideo.mp4")`, and the Facade expertly coordinates the vast underlying complexity smoothly securely on your behalf.
