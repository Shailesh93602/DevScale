Airflow Operators

In Apache Airflow, an operator defines a single unit of work within a DAG. Each operator creates a Task when instantiated. Airflow ships with hundreds of built-in operators and lets you build custom ones.

Common Built-in Operators

from airflow.operators.python import PythonOperator
from airflow.operators.bash import BashOperator
from airflow.operators.empty import EmptyOperator
from airflow.providers.postgres.operators.postgres import PostgresOperator

# BashOperator
run_script = BashOperator(
    task_id='run_script',
    bash_command='python /scripts/transform.py',
    dag=dag,
)

# PythonOperator
def process_data(**context):
    print(context['ds'])  # execution date

run_python = PythonOperator(
    task_id='process_data',
    python_callable=process_data,
    dag=dag,
)

# PostgresOperator
load_data = PostgresOperator(
    task_id='load',
    postgres_conn_id='my_postgres',
    sql='INSERT INTO summary SELECT ...;',
    dag=dag,
)

TaskFlow API (Modern Airflow 2.x+)

The TaskFlow API uses Python decorators — cleaner and more Pythonic:

from airflow.decorators import dag, task
from datetime import datetime

@dag(start_date=datetime(2024, 1, 1), schedule='@daily')
def etl_pipeline():

    @task
    def extract() -> dict:
        return {"rows": 1000, "source": "api"}

    @task
    def transform(data: dict) -> dict:
        data["processed"] = True
        return data

    @task
    def load(data: dict):
        print(f"Loading {data['rows']} rows...")

    raw = extract()
    transformed = transform(raw)
    load(transformed)

etl_pipeline()

Sensors — Wait for External Conditions

from airflow.sensors.filesystem import FileSensor
from airflow.sensors.external_task import ExternalTaskSensor

# Wait for a file to appear
wait_for_file = FileSensor(
    task_id='wait_for_file',
    filepath='/data/input.csv',
    poke_interval=60,   # check every 60s
    timeout=3600,       # fail after 1 hour
    dag=dag,
)

# Wait for another DAG to complete
wait_for_upstream = ExternalTaskSensor(
    task_id='wait_upstream',
    external_dag_id='upstream_pipeline',
    external_task_id='final_task',
    dag=dag,
)

XCom — Passing Data Between Tasks

def push_data(**context):
    context['ti'].xcom_push(key='row_count', value=500)

def pull_data(**context):
    count = context['ti'].xcom_pull(task_ids='push_task', key='row_count')
    print(f"Processing {count} rows...")

Custom Operator

from airflow.models import BaseOperator

class SlackAlertOperator(BaseOperator):
    def __init__(self, message: str, **kwargs):
        super().__init__(**kwargs)
        self.message = message

    def execute(self, context):
        send_slack(self.message.format(**context))

Best Practices

  • Use TaskFlow API for new DAGs — it handles XComs automatically.
  • Keep tasks small and single-purpose (idempotent).
  • Avoid heavy computation inside operators — call external services instead.
  • Use sensors sparingly — set reasonable timeout to avoid zombie tasks.
