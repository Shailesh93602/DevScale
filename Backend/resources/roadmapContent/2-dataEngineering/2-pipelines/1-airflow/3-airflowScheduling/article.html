Airflow Scheduling & Monitoring

Airflow's scheduler is the heart of the platform — it determines when DAGs run. Understanding schedule intervals, SLAs, backfilling, and the Airflow UI is critical for production pipelines.

Schedule Intervals

# Cron expressions
@dag(schedule='0 6 * * *')   # daily at 6 AM UTC
@dag(schedule='0 */4 * * *') # every 4 hours
@dag(schedule='@hourly')
@dag(schedule='@daily')
@dag(schedule='@weekly')
@dag(schedule='@monthly')
@dag(schedule=None)          # manual trigger only

# Also accepts timedelta
from datetime import timedelta
@dag(schedule=timedelta(hours=6))

Key DAG Parameters

@dag(
    dag_id='etl_pipeline',
    start_date=datetime(2024, 1, 1),
    schedule='@daily',
    catchup=False,         # don't backfill historical runs
    max_active_runs=1,     # prevent parallel runs
    tags=['production', 'etl'],
    default_args={
        'retries': 3,
        'retry_delay': timedelta(minutes=5),
        'email_on_failure': True,
        'email': ['alerts@company.com'],
    }
)

Backfilling

Run a DAG for historical dates (e.g., after deploying a new pipeline):

# Via CLI
airflow dags backfill \
  --start-date 2024-01-01 \
  --end-date 2024-03-01 \
  etl_pipeline

Set catchup=False to prevent automatic backfilling on startup.

SLAs (Service Level Agreements)

from datetime import timedelta

@task(sla=timedelta(hours=2))  # alert if task takes > 2h
def long_running_task():
    ...

# DAG-level SLA callback
def sla_miss_callback(dag, task_list, blocking_task_list, slas, blocking_tis):
    send_slack_alert(f"SLA missed for tasks: {task_list}")

@dag(sla_miss_callback=sla_miss_callback)

Airflow UI Key Features

  • Graph View — visualise DAG task dependencies
  • Grid View — per-run task status history
  • Gantt Chart — identify bottleneck tasks
  • Task Logs — full stdout/stderr per task run
  • Connections — manage DB/API credentials
  • Variables — key-value config store

# airflow.dag.*.duration         → DAG run duration
# airflow.task_instance.failures → task failure count

Best Practices

  • Set catchup=False unless backfilling is intentional.
  • Use max_active_runs=1 for pipelines that must not overlap.
  • Add SLA callbacks for critical pipelines to alert on-call.
  • Keep DAG files lightweight — no heavy imports at module level.
