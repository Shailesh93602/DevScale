Apache Spark — Distributed Data Processing

Apache Spark is the industry standard for processing large-scale data that doesn't fit on a single machine. It distributes work across a cluster and is 100x faster than MapReduce for in-memory workloads.

Spark Architecture

  • Driver — Your application code; orchestrates the job.
  • Executor — Workers on cluster nodes that run tasks.
  • Partition — A chunk of data; parallelism = number of partitions.
  • Stage — A set of tasks with no shuffle boundary.
  • Shuffle — Redistributing data across partitions (expensive!).

Creating a SparkSession

from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("SalesAnalysis") \
    .master("local[4]")   # 4 local cores; or "spark://master:7077" for cluster
    .config("spark.sql.shuffle.partitions", "16") \
    .getOrCreate()

DataFrames — Preferred API

from pyspark.sql import functions as F
from pyspark.sql.types import *

# Read data
df = spark.read.parquet("s3://bucket/data/sales/")
df = spark.read.csv("data.csv", header=True, inferSchema=True)

# Transformations (lazy — nothing runs until an action)
result = df \
    .filter(F.col("status") == "completed") \
    .withColumn("revenue", F.col("quantity") * F.col("unit_price")) \
    .groupBy("region", "product_id") \
    .agg(
        F.sum("revenue").alias("total_revenue"),
        F.count("id").alias("order_count"),
        F.avg("revenue").alias("avg_order_value"),
    ) \
    .orderBy(F.desc("total_revenue"))

# Actions — these trigger execution
result.show(20)
result.count()
result.write.parquet("s3://bucket/output/sales_summary/", mode="overwrite")

Spark SQL

df.createOrReplaceTempView("sales")

result = spark.sql("""
  SELECT
    region,
    DATE_TRUNC('month', order_date) AS month,
    SUM(quantity * unit_price) AS revenue
  FROM sales
  WHERE status = 'completed'
  GROUP BY region, month
  ORDER BY month, revenue DESC
""")

Performance Optimisation

# Partitioning — avoid too many small files or one huge file
df.repartition(16)          # Expand partitions (needs shuffle)
df.coalesce(4)              # Reduce partitions (no shuffle)
df.repartition("region")    # Partition by column value

# Caching — reuse stored partitions
df.cache()
df.persist(StorageLevel.MEMORY_AND_DISK)

# Broadcast join — for small lookup tables
from pyspark.sql.functions import broadcast
