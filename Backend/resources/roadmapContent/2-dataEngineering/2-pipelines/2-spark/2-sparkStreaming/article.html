Spark Streaming

Spark Structured Streaming is a scalable, fault-tolerant stream processing engine built on Spark SQL. It treats a live data stream as a continuously appended table, letting you write the same DataFrame/SQL code you use for batch processing.

Core Concepts

  • Source — where data comes in (Kafka, files, socket)
  • DataFrame/Dataset — unbounded table of streaming records
  • Output Modes — how results are written: append, complete, update
  • Trigger — how often Spark processes new data
  • Watermark — how long to wait for late-arriving data
  • Sink — where output goes (Kafka, DB, files, console)

Reading from Kafka

from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("kafka-streaming") \
    .getOrCreate()

df = spark.readStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("subscribe", "orders") \
    .load()

# Kafka gives binary value — cast and parse
from pyspark.sql.functions import col, from_json
from pyspark.sql.types import StructType, StringType, DoubleType

schema = StructType() \
    .add("order_id", StringType()) \
    .add("amount", DoubleType()) \
    .add("user_id", StringType())

orders = df.select(
    from_json(col("value").cast("string"), schema).alias("data")
).select("data.*")

Transformations on Streams

from pyspark.sql.functions import window, sum as spark_sum

# Windowed aggregation (5-min windows, sliding every 1 min)
result = orders \
    .withWatermark("event_time", "10 minutes") \
    .groupBy(window("event_time", "5 minutes", "1 minute"), "user_id") \
    .agg(spark_sum("amount").alias("total"))

Writing Output

# Write to console (for debugging)
query = result.writeStream \
    .outputMode("update") \
    .format("console") \
    .trigger(processingTime="10 seconds") \
    .start()

# Write to Kafka
query = result.writeStream \
    .format("kafka") \
    .option("kafka.bootstrap.servers", "localhost:9092") \
    .option("topic", "order_summaries") \
    .option("checkpointLocation", "/checkpoints/orders") \
    .start()

query.awaitTermination()

Checkpointing for Fault Tolerance

Always set a checkpoint location — Spark saves offsets here so it can recover from failures:

.option("checkpointLocation", "s3://bucket/checkpoints/stream-name")

Output Modes

  • append — only new rows (no aggregations, or final results only)
  • update — only changed rows (ideal for live aggregations)
  • complete — all result rows every trigger (small result tables only)

Best Practices

  • Always set a watermark to bound state size for time-based aggregations.
  • Use processingTime triggers (e.g., every 10 seconds) for near-real-time.
  • Monitor backlog via Spark UI → Structured Streaming tab.
  • Process Kafka from earliest offset in development, latest in production.
