dbt Deployment & CI/CD

Deploying a dbt project requires managing environments, scheduling pipeline runs, ensuring artifacts like manifest.json are tracked, and validating changes on pull requests using CI/CD methodologies.

Managing Environments

Most dbt setups use at least two environments (development and production) mapped to separate profiles in profiles.yml.

my_project:
  target: dev
  outputs:
    dev:
      type: bigquery
      dataset: dbt_alice # Personal dev dataset
      ...
    prod:
      type: bigquery
      dataset: analytics # Prod dataset
      ...

Scheduling dbt Runs

You can run dbt in production via the CLI (dbt Core) on a scheduled Airflow task or GitHub Action, or use dbt Cloud to manage the orchestration for you.

Airflow Integration

The BashOperator is often used to execute dbt commands:

from airflow.operators.bash import BashOperator

dbt_build = BashOperator(
    task_id='dbt_build',
    bash_command='dbt build --target prod',
    dag=dag
)

Alternatively, packages like astronomer-cosmos can parse the manifest.json to turn individual dbt models into separate Airflow tasks, showing the full DAG line-by-line in the Airflow UI.

Continuous Integration (CI) and Slim CI

When someone opens a Pull Request, running dbt build over the entire warehouse can be expensive and slow. dbt provides state comparison (Slim CI) to build only what changed.

# Download the manifest.json from the latest prod run
# Compare current state to prod state, run only modified models and their downstream dependencies

dbt build --select state:modified+ --defer --state path/to/prod/artifacts

The --defer flag tells dbt to resolve unchanged upstream models using the production tables, saving money and time.

dbt Cloud Features

dbt Cloud provides a managed environment that simplifies deployment:

  • Native GitHub/GitLab integration for automatic PR builds (Slim CI built-in).
  • Job scheduler with cron integration to run the pipelines.
  • Web-based IDE for development without local setup.
  • Hosted documentation and model lineage graph.

Generating & Hosting Documentation

dbt generates a static website documenting your DAG, models, and columns descriptions.

# Resolves descriptions and generates catalog.json
dbt docs generate

# Serves the static site locally
dbt docs serve

In production, you can deploy the generated target/ folder (specifically index.html, manifest.json, and catalog.json) to an S3 bucket or Vercel to host the internal company data dictionary.
