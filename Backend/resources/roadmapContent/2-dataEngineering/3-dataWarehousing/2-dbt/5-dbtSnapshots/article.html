Slowly Changing Dimensions (Snapshots)

In standard data pipelines, databases often overwrite historical data. If a user named "Bob" currently lives in "New York", and he moves to "California", the application database simply executes an UPDATE statement. The row now says "California", and the historical fact that Bob ever lived in New York is permanently deleted.

The Problem with Overwriting History

From an operational standpoint, overwriting is fine. But for Data Analysts, deleted history is catastrophic. If the Marketing team wants to know "How many users lived in New York during the 2022 Christmas Sale?", the calculation will be inherently broken because Bob's record in the data warehouse was mutated in 2024 to "California."

To fundamentally solve this, Data Engineers implement exactly what is known as Type 2 Slowly Changing Dimensions (SCD Type 2).

Enter dbt Snapshots

Building a custom SCD Type 2 pipeline manually in SQL is incredibly complex and prone to massive architectural errors. dbt Snapshots automate this entire process flawlessly.

How Snapshots Work Physically

When you execute dbt snapshot, dbt inspects the raw source database and compares it to its internal Snapshot table.

  1. The first time you run it, dbt inserts Bob's "New York" record with dbt_valid_from = 2022-01-01 and dbt_valid_to = NULL (meaning this is his current, active location).
  2. When Bob updates his profile to "California" in 2024, the raw database simply overwrites the old row.
  3. The next time you run dbt snapshot, dbt notices that the last_updated_date on Bob's row has structurally changed!

Instead of overwriting the Data Warehouse, dbt magically executes an elegant transformation. It updates Bob's "New York" row, explicitly marking dbt_valid_to = 2024-05-01 (closing the historical chapter). It then inserts a brand new row for "California" with dbt_valid_from = 2024-05-01 and dbt_valid_to = NULL.

Now, when Analysts query the Data Warehouse, they can explicitly filter by the dbt_valid_from timestamps to perfectly recreate the exact state of the universe on any given historical date safely.
