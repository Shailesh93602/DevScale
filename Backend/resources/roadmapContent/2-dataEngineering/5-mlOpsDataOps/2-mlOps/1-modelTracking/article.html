Model Tracking & MLflow

In traditional software engineering, tracking code changes via Git is standard. In Machine Learning, the performance of a model depends on three axes: Code, Data, and Hyperparameters.

If you run an experiment today that achieves 95% accuracy, but a month from now you can't recreate it because you forgot which learning rate you used or what the dataset looked like, you have a major problem. Model Tracking solves this.

What is MLflow?

MLflow is an open-source platform tailored strictly for managing the machine learning lifecycle. It is framework agnostic, supporting Scikit-Learn, PyTorch, TensorFlow, XGBoost, and more.

Core Components of MLflow

  • MLflow Tracking: An API and UI to log parameters, code versions, metrics, and output files when running your machine learning code.
  • MLflow Projects: A standard format for packaging reusable data science code. You define the dependencies (like a conda environment.yaml) so that another developer can run your project perfectly on their machine.
  • MLflow Models: A standard format for packaging an ML model (like a ZIP file containing the weights and a generic predict() wrapper function) so it can be easily deployed to a generic REST API server or Apache Spark for batch scoring.
  • MLflow Model Registry: A centralized store to collaboratively manage the full lifecycle of an MLflow Model. It provides chronological model lineage, model versioning, stage transitions (Staging → Production), and annotations.

Tracking Experiments in Python

Adding MLflow to a training script is straightforward:

import mlflow
import mlflow.sklearn
from sklearn.ensemble import RandomForestRegressor

# Wrap training inside an mlflow run block
with mlflow.start_run():
    # 1. Log Hyperparameters
    n_estimators = 100
    mlflow.log_param("n_estimators", n_estimators)

    model = RandomForestRegressor(n_estimators=n_estimators)
    model.fit(X_train, y_train)
    predictions = model.predict(X_test)

    # 2. Log Metrics (Accuracy, MSE, F1)
    rmse = mean_squared_error(y_test, predictions, squared=False)
    mlflow.log_metric("rmse", rmse)

    # 3. Log the Model Artifact itself
    mlflow.sklearn.log_model(model, "rf_model")

The Autologging Feature

MLflow provides autolog() integration for major frameworks. It automatically logs parameters and metrics without needing explicit mlflow.log_param() calls.

import mlflow.xgboost

# One line handles everything!
mlflow.xgboost.autolog()

model = xgb.train(params, dtrain)

Why Is This Part of Data Engineering?

Machine Learning Engineers build the models, but Data Engineers build the pipelines that constantly retrain these models with fresh batches of structured data. Understanding how to pull the latest "Production" model from the MLflow Registry via an Airflow DAG to perform batch predictions on a Spark cluster is a core Data Engineering responsibility.
