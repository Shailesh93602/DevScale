Shell Scripting

As a DevOps engineer, you will repeatedly perform tasks like server backups, log rotation, or environment setup. Shell Scripting automates these manual commands into a single executable file.

The Shebang

Every bash script should start with a "shebang" (#!) indicating which interpreter should run the file.

#!/bin/bash
echo "Hello, Server!"

Variables and Input

#!/bin/bash

# Assigning variables (no spaces around '=')
BACKUP_DIR="/var/backups"
DB_NAME="production_db"

# Reading user input
read -p "Enter backup name: " USER_INPUT

# Using Positional Parameters ($1, $2)
# Run script as: ./backup.sh my_database
TARGET=$1
echo "Backing up ${TARGET} to ${BACKUP_DIR}"

Conditionals (If Statements)

Bash uses [ ] or [[ ]] for test conditions, and requires spaces inside the brackets.

#!/bin/bash
FILE="/etc/nginx/nginx.conf"

if [[ -f "$FILE" ]]; then
    echo "$FILE exists. Proceeding with reload..."
    systemctl reload nginx
else
    echo "Error: $FILE not found!"
    exit 1
fi

Common flags: -f (is file), -d (is directory), -z (string is empty).

Loops

Loops are essential for running the same command against multiple files or servers.

#!/bin/bash

# For Loop iterating over a list
for SERVER in web1 web2 db1; do
    echo "Pinging $SERVER..."
    ping -c 1 $SERVER
done

# While Loop reading a file line-by-line
while read -r LINE; do
    echo "Processing user: $LINE"
done < users.txt

Exit Codes and Error Handling

Every Linux command returns an exit code (0 for success, 1-255 for errors). You can access the last exit code using $?.

#!/bin/bash
mkdir /root/secured_folder

if [[ $? -ne 0 ]]; then
    echo "Failed to create directory. Are you root?"
    exit 1
fi

Pro-Tip: Put set -e at the top of your scripts. It tells bash to exit immediately if any command fails, preventing destructive cascading errors.
