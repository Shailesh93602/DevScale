Process Management

Servers run dozens of programs simultaneously. Understanding how to track, manage, and kill these processes is critical when an application hangs or consumes too much memory.

Viewing Processes

The most common tools to view running processes are ps and top (or htop).

# Snapshot of all running processes with full details
ps aux

# Search for a specific process (e.g., node.js)
ps aux | grep node

# Interactive real-time process viewer
top
htop  # (More user-friendly alternative)

Foreground vs Background Jobs

When you run a command, it blocks your terminal until it finishes. This is a foreground process.

# Run a script in the background using '&'
./long_script.sh &

# List background jobs
jobs

# Bring a job back to the foreground
fg %1

If you close your SSH session, background jobs are killed. To keep them running indefinitely, use nohup or a multiplexer like tmux.

nohup ./long_script.sh &

Killing Processes

If an application freezes, you can terminate it by sending a Signal to its Process ID (PID).

# Graceful termination (SIGTERM = 15). Allows app to clean up.
kill 1234

# Forceful kill (SIGKILL = 9). The app instantly dies. Use as last resort.
kill -9 1234

# Kill all processes by name
killall node

Systemd: The Daemon Manager

Modern Linux distributions use systemd to launch processes exactly when the server boots and restart them if they crash. These are called services.

# Start a service
sudo systemctl start nginx

# Stop a service
sudo systemctl stop nginx

# View logs generated by the service
sudo journalctl -u nginx --tail 50 -f

Diagnostic Tools

  • free -m — View available and used RAM.
  • df -h — View disk space usage.
  • lsof -i :80 — Find which process is occupying port 80.
