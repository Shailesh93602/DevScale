Networking Fundamentals for Engineers

Understanding networking is essential for debugging production issues, configuring cloud infrastructure, and understanding security architecture.

OSI Model — The Stack

  • Layer 7 — Application: HTTP, HTTPS, DNS, SSH, FTP
  • Layer 4 — Transport: TCP (reliable), UDP (fast, no guarantee)
  • Layer 3 — Network: IP addresses, routing
  • Layer 2 — Data Link: MAC addresses, switches

TCP vs UDP

TCP (Transmission Control Protocol)
✅ Reliable — guarantees delivery, ordering, error correction
✅ Used for: HTTP/S, SSH, databases, file transfers
❌ Higher overhead — connection setup (3-way handshake), ACKs

UDP (User Datagram Protocol)
✅ Fast — no connection setup, no acknowledgment
✅ Used for: video streaming, DNS, gaming, VoIP
❌ No delivery guarantee — packets may be lost or arrive out of order

DNS — Domain Name System

# How DNS resolves api.myapp.com:
1. Check /etc/hosts and local cache
2. Query recursive resolver (ISP or 8.8.8.8)
3. Recursive resolver queries root nameservers (.com → myapp.com NS)
4. Query authoritative nameserver → returns IP 203.0.113.10
5. Response cached for TTL duration

# Common DNS record types:
A         api.myapp.com → 203.0.113.10        (IPv4 address)
CNAME     www.myapp.com → myapp.com           (alias)
MX        myapp.com → mail.myapp.com          (mail server)
TXT       myapp.com → "v=spf1 include:..."   (SPF, verification)

# Useful commands
nslookup api.myapp.com     # DNS lookup
dig +short api.myapp.com   # DNS lookup with details
curl -I https://myapp.com  # Check HTTP headers

Load Balancers

# Round Robin — distribute requests evenly
# Least Connections — route to server with fewest active connections
# IP Hash — same client always hits same server (session stickiness)

# Health checks
upstream backend {
  server app1:3000 weight=3;  # Gets 3x more traffic
  server app2:3000;
  server app3:3000 backup;    # Only used if others fail
}

VPC — Virtual Private Cloud

# AWS VPC structure:
VPC: 10.0.0.0/16 (65,536 IPs)
  Public Subnet:  10.0.1.0/24  → Has route to Internet Gateway → accessible from internet
  Private Subnet: 10.0.2.0/24  → No internet route → databases, app servers
  NAT Gateway in public subnet → allows private subnet to access internet outbound
