TCP/IP & Subnetting

Every piece of data that travels across a network — from a web page request to a database query — follows the TCP/IP protocol suite. Understanding how this model works is fundamental for any DevOps engineer, SRE, or cloud architect diagnosing network issues or designing VPC architectures.

The OSI Model (Simplified)

The OSI (Open Systems Interconnection) model describes how data moves from an application on one machine to an application on another, passing through 7 logical layers. In practice, the TCP/IP model collapses this into 4 layers:

  • Application Layer (Layer 7): HTTP, DNS, SMTP, SSH — protocols applications use directly
  • Transport Layer (Layer 4): TCP and UDP — provides end-to-end communication between processes
  • Internet Layer (Layer 3): IP — handles logical addressing and routing packets across networks
  • Network Access Layer (Layer 2/1): Ethernet, Wi-Fi — handles physical transmission of bits

TCP vs UDP

  • TCP (Transmission Control Protocol): Connection-oriented. Before data is sent, a 3-way handshake establishes a reliable connection (SYN → SYN-ACK → ACK). TCP guarantees that all packets arrive in order and retransmits lost packets. Used by HTTP, HTTPS, SSH, PostgreSQL. The reliability comes at a cost: higher latency.
  • UDP (User Datagram Protocol): Connectionless. Packets are fired off with no handshake and no guarantee of delivery or ordering. Used for DNS queries, VoIP, video streaming, and gaming — applications where speed trumps guaranteed delivery.

IPv4 Addressing

An IPv4 address is 32 bits, written as 4 octets in decimal: 192.168.1.100. Every device on a network needs a unique IP address.

Private IP ranges (used internally in VPCs, home networks, not routable on the internet):

  • 10.0.0.0/8 — used by AWS VPCs, large corporate networks
  • 172.16.0.0/12 — used by Docker by default
  • 192.168.0.0/16 — used by home routers

CIDR Notation & Subnetting

CIDR (Classless Inter-Domain Routing) notation like 10.0.1.0/24 defines a range of IP addresses. The number after the slash indicates how many bits are the "network prefix" (fixed), while the remaining bits identify individual hosts.

10.0.1.0/24 breakdown:
- Network bits: 24 (the first 3 octets are fixed: 10.0.1)
- Host bits: 32 - 24 = 8 bits remaining
- Total addresses: 2^8 = 256 (10.0.1.0 — 10.0.1.255)
- Usable hosts: 254 (first = network address, last = broadcast)

10.0.0.0/16 → 65,536 addresses (entire 10.0.x.x range)
10.0.0.0/32 → 1 address (exactly one specific IP — used for single-host rules)
0.0.0.0/0   → All addresses (used in routing tables as "default route")

In AWS: Designing VPC Subnets

VPC CIDR: 10.0.0.0/16 (65,536 IPs total)
├─ Public Subnet (us-east-1a):  10.0.1.0/24  (254 hosts, internet-facing)
├─ Public Subnet (us-east-1b):  10.0.2.0/24  (254 hosts, internet-facing)
├─ Private Subnet (us-east-1a): 10.0.10.0/24 (254 hosts, no direct internet)
└─ Private Subnet (us-east-1b): 10.0.11.0/24 (254 hosts, no direct internet)

Splitting the larger /16 into smaller /24 subnets is called subnetting. Public subnets have an Internet Gateway attached and are used for load balancers. Private subnets house your databases and application servers with no direct internet access.
