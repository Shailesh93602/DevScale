Firewalls, VPNs & Network Security

In a cloud environment, every layer of your infrastructure can be exposed to the internet. A single misconfigured firewall rule can expose your entire database cluster to the world. Understanding the tools and models for controlling network access is one of the most critical skills in cloud architecture and DevOps.

Stateful vs Stateless Firewalls

  • Stateless Firewalls: Evaluate each packet in complete isolation against a rule list. If you allow inbound TCP on port 443, a stateless firewall also separately requires an explicit rule allowing the return traffic (the response) to get back to the client. Used for Network ACLs (NACLs) in AWS.
  • Stateful Firewalls: Track active connection state. If an internal server initiates an outbound HTTPS request, the firewall automatically allows the inbound response packets back — no separate return-traffic rule needed. This is how AWS Security Groups work and is why Security Groups are usually preferred over NACLs for instance-level control.

AWS Security Groups vs Network ACLs

Security Groups (Instance-level):
✅ Stateful — return traffic automatically allowed
✅ Only ALLOW rules (default: deny all inbound, allow all outbound)
✅ Applied to: EC2 instances, RDS, Lambda, etc.
✅ Rules evaluated together (all rules checked)

Network ACLs (Subnet-level):
⚠️  Stateless — must explicitly allow both inbound AND return traffic
⚠️  ALLOW and DENY rules both supported
⚠️  Applied to: entire subnets
⚠️  Rules evaluated in order (lowest number wins)

The recommended pattern: Use Security Groups as your primary access control (fine-grained, per-instance). Use NACLs as a secondary defense (subnet-level, e.g., block an entire IP range known for attacks).

VPN: Connecting Private Networks

A VPN (Virtual Private Network) creates an encrypted tunnel between two networks or between a user and a network, making private resources accessible as if they were on the local LAN:

  • AWS Site-to-Site VPN: Connects your on-premise data center to your AWS VPC. All traffic between the two travels through an encrypted IPSec tunnel over the public internet, with no traffic visible to ISPs.
  • AWS Client VPN: Allows individual remote developers to connect to private AWS resources using OpenVPN or WireGuard clients from their laptops.
  • AWS Direct Connect: A dedicated physical network connection from your data center to AWS — not a VPN, but private. Used when VPN latency or bandwidth is insufficient for high-throughput workloads.

Zero Trust Networking

Traditional network security operated on a "castle and moat" model: trust everything inside the perimeter, block everything outside. Zero Trust replaces this with: "Never trust, always verify."

In a Zero Trust model:

  • Every request — even from inside the corporate network — must be authenticated and authorized
  • Access is granted with the least privilege necessary (specific resources, specific time windows)
  • All traffic is encrypted, even between internal microservices (using mutual TLS — mTLS)
  • Continuous verification: identity is re-evaluated on each request, not just at login

Tools like HashiCorp Consul (service mesh), Cloudflare Access, and AWS Verified Access implement Zero Trust models at scale for modern cloud-native architectures.
