AWS IAM (Identity & Access Management)

IAM is the absolute perimeter of your AWS cloud. A misconfigured IAM policy can instantly expose your entire infrastructure and customer data to the public internet.

The Principle of Least Privilege

The golden rule of IAM: Give a user or system exactly the minimum permissions it needs to perform its job, and absolutely nothing more. Avoid using the managed `AdministratorAccess` policy for anything other than your initial root setup.

Core Components

  • Users: A human (or a service account) that logs into the AWS console or uses an Access Key to make API calls.
  • Groups: A collection of Users. (e.g., You attach a "Developer" policy to the "Developers" group. Any user added to the group inherits those permissions).
  • Roles: Intended to be assumable by things, not humans. An EC2 server assumes a Role to get permission to read from an S3 bucket without you having to hardcode an Access Key onto the server.
  • Policies: JSON documents attached to Users, Groups, or Roles. They define the exact permissions.

Anatomy of an IAM Policy

A policy consists of an Effect, an Action, and a Resource (and sometimes a Condition). By default, everything is Denied.

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::my-company-bucket/*"
    }
  ]
}

If you attach this policy to a user, they can read and write objects to my-company-bucket, but they cannot delete the bucket itself (s3:DeleteBucket), nor can they access any other bucket.

Best Practices

  • Enable MFA (Multi-Factor Authentication) on all user accounts, especially the root account.
  • Never generate Access Keys for the root account.
  • Do not hardcode Access Keys in your application code. Use IAM Roles attached to your compute resources (EC2, ECS, Lambda).
  • Use IAM Identity Center (formerly AWS SSO) instead of creating individual IAM Users if you have an organization with multiple AWS accounts.
