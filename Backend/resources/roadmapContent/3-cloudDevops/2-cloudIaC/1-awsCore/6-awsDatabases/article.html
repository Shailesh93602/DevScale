AWS Databases: RDS, DynamoDB & ElastiCache

AWS offers a purpose-built database for virtually every workload. Choosing the right database service dramatically impacts your application's performance, cost, and operational complexity. The three most important managed database services are RDS, DynamoDB, and ElastiCache.

Amazon RDS: Managed Relational Databases

Amazon RDS (Relational Database Service) provides managed instances of PostgreSQL, MySQL, MariaDB, Oracle, and SQL Server. AWS handles automated backups, OS patching, minor version upgrades, and failure detection.

Multi-AZ vs Read Replicas

Multi-AZ Deployment:
├─ PRIMARY instance (us-east-1a) — handles all reads AND writes
└─ STANDBY instance (us-east-1b) — synchronous replica, NO read traffic served
   └─ Automatic failover if primary fails (typically 60-120 seconds)
   └─ Purpose: HIGH AVAILABILITY, not performance

Read Replicas:
├─ PRIMARY instance — handles all writes
├─ Read Replica 1 (us-east-1a) — serves SELECT queries, asynchronous replication
├─ Read Replica 2 (us-east-1b) — serves SELECT queries
└─ Read Replica 3 (us-east-1c) — serves SELECT queries
   └─ Purpose: READ SCALABILITY, not failover

Production best practice: Combine both. Use Multi-AZ for HA, and add Read Replicas to offload analytics queries from the primary instance.

Amazon DynamoDB: Managed NoSQL at Scale

Amazon DynamoDB is AWS's fully managed, serverless NoSQL key-value and document database. It delivers consistent single-digit millisecond latency at any scale, from 1 item to 1 billion items, with no tuning required.

Single-Table Design

DynamoDB is optimized for specific access patterns defined upfront. The most powerful DynamoDB pattern is single-table design — storing multiple entity types in one table:

Table: e-commerce

PK (Partition Key)       SK (Sort Key)            Attributes
─────────────────────────────────────────────────────────────
USER#usr_123             PROFILE                  {email, name, created}
USER#usr_123             ORDER#ord_001            {total, status, date}
USER#usr_123             ORDER#ord_002            {total, status, date}
PRODUCT#prod_456         DETAILS                  {name, price, stock}
PRODUCT#prod_456         REVIEW#rev_001           {rating, text, user}

A single DynamoDB query with PK = USER#usr_123 returns the user's profile AND all their orders in one round trip — no JOINs needed.

DynamoDB Streams & Global Tables

DynamoDB Streams emit a real-time record of every change (INSERT, MODIFY, DELETE) to a table — enabling event-driven Lambda triggers. Global Tables replicate a DynamoDB table across multiple AWS regions in under 1 second for ultra-low latency global applications.

Amazon ElastiCache: In-Memory Caching

Amazon ElastiCache provides managed Redis and Memcached clusters. Used as a caching layer in front of RDS to absorb repetitive read traffic:

cache = redis.Redis(host='my-redis.cache.amazonaws.com', port=6379)
