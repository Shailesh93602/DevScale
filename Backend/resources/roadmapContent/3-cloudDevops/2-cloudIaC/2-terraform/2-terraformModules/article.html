Terraform Modules

As your infrastructure grows, writing thousands of lines of code in a single main.tf file becomes unmanageable. If you need to spin up 5 identical web servers across 3 environments (Dev, Staging, Prod), copying and pasting the Terraform resource blocks will lead to catastrophic typos.

Terraform Modules solve this by allowing you to package and reuse infrastructure configurations.

What is a Module?

A module is simply a directory containing Terraform configuration files (.tf). In fact, the directory where you run terraform apply is technically the "Root Module". You can call other modules (Child Modules) from the Root Module.

Creating a Custom Module

Imagine you have a directory modules/vpc with a standard company-compliant VPC setup.

# Inside main.tf (Calling the module)
module "my_custom_vpc" {
  # The path to your module directory
  source = "./modules/vpc"

  # Passing input variables expected by the module
  vpc_name    = "production-vpc"
  cidr_block  = "10.0.0.0/16"
}

The Terraform Public Registry

You don't always have to write your own modules from scratch. The Terraform Registry (registry.terraform.io) contains thousands of verified, production-ready modules written by AWS, Google, and the HashiCorp community.

# Calling a public registry module
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.0.0"  # ALWAYS pin the version!

  name = "my-vpc"
  cidr = "10.0.0.0/16"

  azs             = ["eu-west-1a", "eu-west-1b", "eu-west-1c"]
  private_subnets = ["10.0.1.0/24", "10.0.2.0/24", "10.0.3.0/24"]
  public_subnets  = ["10.0.101.0/24", "10.0.102.0/24", "10.0.103.0/24"]

  enable_nat_gateway = true
}

Module Best Practices

  • Pin Versions: Always use the version argument when pulling external modules. If an author releases a major update with breaking changes, an unpinned module will automatically pull the newest state and destroy your servers on the next apply.
  • Use Standard Structure: Ensure your inner module always contains main.tf, variables.tf, and outputs.tf.
  • Parameterise Everything: Hardcode as little as possible inside the module so it remains fully reusable across totally different environments.
