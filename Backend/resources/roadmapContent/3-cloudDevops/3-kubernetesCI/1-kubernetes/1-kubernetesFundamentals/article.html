Kubernetes Fundamentals

Kubernetes (K8s) is the industry-standard platform for deploying, scaling, and managing containerised applications.

Core Architecture

  • Control Plane — API server, scheduler, etcd (cluster state)
  • Node — Worker machine running pods (kubelet + container runtime)
  • Pod — Smallest deployable unit; one or more containers
  • Deployment — Manages replicas of a pod; handles rolling updates
  • Service — Stable network endpoint to access pods
  • Ingress — HTTP routing / load balancer / TLS termination

Deployment Manifest

apiVersion: apps/v1
kind: Deployment
metadata:
  name: api-server
  labels:
    app: api
spec:
  replicas: 3                          # 3 pods running
  selector:
    matchLabels: { app: api }
  strategy:
    type: RollingUpdate
    rollingUpdate: { maxUnavailable: 1, maxSurge: 1 }
  template:
    metadata:
      labels: { app: api }
    spec:
      containers:
      - name: api
        image: myorg/api:v1.2.3        # Always use specific tags!
        ports:
        - containerPort: 3000
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: url
        resources:
          requests: { cpu: "100m", memory: "128Mi" }
          limits:   { cpu: "500m", memory: "512Mi" }
        livenessProbe:
          httpGet: { path: /health, port: 3000 }
          initialDelaySeconds: 30
        readinessProbe:
          httpGet: { path: /ready, port: 3000 }

Service & Ingress

kind: Service
spec:
  selector: { app: api }
  type: ClusterIP           # Internal only; use LoadBalancer for external
  ports:
  - port: 80, targetPort: 3000

---
kind: Ingress
spec:
  rules:
  - host: api.myapp.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service: { name: api-server, port: { number: 80 } }

Essential kubectl Commands

kubectl get pods               # List pods
kubectl get pods -o wide       # With node info
kubectl describe pod api-xyz   # Full pod details
kubectl logs api-xyz -f        # Follow logs
kubectl exec -it api-xyz -- sh # Shell into pod
kubectl apply -f deployment.yaml
kubectl rollout status deployment/api-server
kubectl rollout undo deployment/api-server   # Rollback!
kubectl scale deployment/api-server --replicas=5

ConfigMaps & Secrets

kubectl create configmap app-config --from-file=config.yaml
kubectl create secret generic db-secret --from-literal=url='postgresql://...'
