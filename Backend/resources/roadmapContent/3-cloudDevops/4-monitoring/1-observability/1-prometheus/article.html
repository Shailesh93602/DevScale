Prometheus

In a cloud environment, you need to know if your servers are running out of CPU, or if your API is suddenly taking 5 seconds to respond instead of 50 milliseconds. Prometheus is the industry standard tool for collecting these metrics.

Time-Series Database

Prometheus is fundamentally a Time-Series Database (TSDB). It stores metrics as streams of numeric values over time. Every data point consists of three things:

  1. Metric Name: (e.g., http_requests_total)
  2. Labels: Key-value pairs for filtering (e.g., {method="GET", status="500"})
  3. Value & Timestamp: (e.g., 42 @ 1629837192)

The Pull Model

Unlike traditional monitoring tools where servers push their metrics to a central database, Prometheus uses a Pull Model.

Your application exposes an HTTP endpoint (usually /metrics). Every 15 seconds, the Prometheus server makes an HTTP GET request to that endpoint on every single server in your cluster, scraping the data and saving it to its database.

# Example payload exposed at /metrics
# HELP http_requests_total The total number of HTTP requests.
# TYPE http_requests_total counter
http_requests_total{method="post",code="200"} 1027
http_requests_total{method="post",code="400"} 3
http_requests_total{method="get",code="200"} 4539

Metric Types

  • Counter: A number that only ever goes UP (e.g., total number of requests, total errors). Useful for calculating rates (e.g., requests per second).
  • Gauge: A number that can go up and down (e.g., current RAM usage, current active database connections).
  • Histogram: Samples observations into configurable buckets. (e.g., tracking API response times: 10 requests took <100ms, 5 requests took <500ms). Crucial for calculating Apdex scores and percentiles.

PromQL (Prometheus Query Language)

To alert on metrics, you query Prometheus using PromQL. It is specifically designed for time-series arithmetic.

# Get the per-second rate of 500 errors over the last 5 minutes
rate(http_requests_total{status="500"}[5m])

# Alert if the 5-minute error rate exceeds 5 errors per second
rate(http_requests_total{status="500"}[5m]) > 5

Alertmanager

Prometheus itself just collects data and evaluates PromQL rules. When a rule is violated, it fires an alert to the Alertmanager. The Alertmanager handles actual notification routing (sending a message to a Slack channel, triggering a PagerDuty incident, or sending an email) and handles deduplication (if 50 servers crash, it sends 1 consolidated Slack message, not 50).
