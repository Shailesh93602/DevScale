Statistical Inference for Machine Learning

Statistical inference is the process of drawing conclusions about a population from sample data — and quantifying your confidence in those conclusions. In machine learning, it underpins hypothesis testing for model comparison, confidence intervals for predictions, understanding when regularization is needed, and detecting data drift in production systems.

Population vs. Sample

import numpy as np
from scipy import stats

# ─── In ML, you never have the full "population" ────────────────────────────
# - Population: ALL 1 million customers' purchase behavior (unknowable)
# - Sample:     The 10,000 customers in your training dataset
#
# Statistical inference lets you reason: "Given our sample, what can we
# confidently say about the true population parameter?"

np.random.seed(42)

# Simulate: True population mean purchase value = $47.50 (unknown to us)
population_mean = 47.50

# We observe a sample of n=200 customers
sample = np.random.normal(loc=population_mean, scale=15, size=200)
print(f"Sample mean: ${np.mean(sample):.2f}")          # ≈ $47.50 (varies)
print(f"Sample std:  ${np.std(sample, ddof=1):.2f}")   # Use ddof=1 for sample std
print(f"Standard Error (SE) = std/√n = {np.std(sample, ddof=1) / np.sqrt(200):.2f}")

Confidence Intervals

A 95% confidence interval means: "If we repeated this sampling process 100 times, approximately 95 of the resulting intervals would contain the true population mean." It is not a probability statement about one specific interval:

# Method 1: Using scipy.stats.t (Student's t-distribution — correct for small samples)
confidence_level = 0.95
ci_low, ci_high = stats.t.interval(
    confidence_level,
    df=len(sample) - 1,           # Degrees of freedom
    loc=np.mean(sample),          # Sample mean
    scale=stats.sem(sample)       # Standard Error of the Mean
)
print(f"95% CI for population mean: (${ci_low:.2f}, ${ci_high:.2f})")
# → (${45.83}, ${49.27}) — likely contains true $47.50

# For large samples (n > 30), normal distribution ≈ t-distribution
ci_normal = stats.norm.interval(0.95, loc=np.mean(sample), scale=stats.sem(sample))
print(f"95% CI (normal approx): (${ci_normal[0]:.2f}, ${ci_normal[1]:.2f})")

Hypothesis Testing

Hypothesis tests answer: "Is the difference between groups real, or just random noise from our sample?" The p-value is the probability of observing a result as extreme as what we found IF the null hypothesis were true.

One-Sample t-Test: Is my model's accuracy better than baseline?

# Null Hypothesis (H₀): Model accuracy = 0.75 (baseline random forest)
# Alt Hypothesis (H₁): Model accuracy ≠ 0.75 (two-tailed test)

# 10-fold cross-validation accuracy scores from your new model
cv_scores = np.array([0.782, 0.791, 0.805, 0.778, 0.799, 0.812, 0.783, 0.797, 0.801, 0.788])

baseline = 0.75
t_stat, p_value = stats.ttest_1samp(cv_scores, popmean=baseline)
print(f"t-statistic: {t_stat:.4f}")
print(f"p-value:     {p_value:.6f}")
print(f"Mean CV accuracy: {cv_scores.mean():.4f}")

alpha = 0.05  # Significance level (5% false positive rate)
if p_value < alpha:
    print(f"✅ Reject H₀: Model improvement is statistically significant (p={p_value:.4f} < α={alpha})")
else:
    print(f"❌ Fail to reject H₀: Not enough evidence of improvement")

Two-Sample t-Test: Is Model A significantly better than Model B?

# Cross-val scores for two competing models
model_a_scores = np.array([0.812, 0.835, 0.821, 0.819, 0.828])
model_b_scores = np.array([0.798, 0.791, 0.803, 0.795, 0.789])

# Use Welch's t-test (equal_var=False) — safe when variances may differ
t_stat, p_value = stats.ttest_ind(model_a_scores, model_b_scores, equal_var=False)
print(f"Model A mean: {model_a_scores.mean():.4f}")
print(f"Model B mean: {model_b_scores.mean():.4f}")
print(f"p-value: {p_value:.4f}")
# p < 0.05 → the difference is statistically significant, not due to chance

Chi-Square Test: Are categorical features independent?

# Is gender independent of churn? (for feature selection)
# Observed frequencies (from training data)
#              Churned  Not Churned
# Male:          200       800
# Female:        150       850

observed = np.array([[200, 800],
                     [150, 850]])

chi2, p_value, dof, expected = stats.chi2_contingency(observed)
print(f"Chi-square statistic: {chi2:.4f}")
print(f"p-value: {p_value:.4f}")
print(f"Expected frequencies (under independence):\n{expected.astype(int)}")
# If p < 0.05: gender IS associated with churn → keep as a feature
# If p > 0.05: gender is independent of churn → may not be useful

Multiple Testing Problem (Critical in ML)

If you run 100 feature importance tests at α=0.05, you'd expect 5 false positives by chance alone. This is the multiple comparisons problem. Apply corrections:

from statsmodels.stats.multitest import multipletests

# 100 p-values from testing 100 features against the target variable
np.random.seed(42)
# Simulate: 90 null (p ~ Uniform), 10 truly significant (p ~ small)
p_values = np.concatenate([
    np.random.uniform(0.01, 0.99, 90),           # Noise features
    np.random.uniform(0.0001, 0.005, 10)          # Truly important features
])

# Bonferroni correction: α_adjusted = α / n_tests (conservative)
reject_bonf, p_bonf, _, _ = multipletests(p_values, alpha=0.05, method='bonferroni')

# Benjamini-Hochberg (FDR) correction: controls False Discovery Rate (less conservative)
reject_fdr, p_fdr, _, _ = multipletests(p_values, alpha=0.05, method='fdr_bh')

print(f"Significant features (uncorrected α=0.05): {(p_values < 0.05).sum()}")
print(f"Significant features (Bonferroni):          {reject_bonf.sum()}")
print(f"Significant features (FDR/BH):              {reject_fdr.sum()}")
# FDR correction is preferred in ML — less likely to discard true signals

A/B Testing for ML Models

# Evaluate two recommendation algorithms on 1000 users each
# Metric: click-through rate (CTR)

np.random.seed(42)
n_users = 1000
control_ctr = 0.12   # Current algorithm: 12% CTR
treatment_ctr = 0.14 # New algorithm: 14% CTR

# Simulate observed clicks (binomial distribution)
control_clicks = stats.binom.rvs(1, control_ctr, size=n_users)
treatment_clicks = stats.binom.rvs(1, treatment_ctr, size=n_users)

# Proportions test (more appropriate for binary outcomes than t-test)
from statsmodels.stats.proportion import proportions_ztest

count = np.array([treatment_clicks.sum(), control_clicks.sum()])
nobs = np.array([n_users, n_users])

z_stat, p_value = proportions_ztest(count, nobs)
print(f"Control CTR:   {control_clicks.mean():.3f}")
print(f"Treatment CTR: {treatment_clicks.mean():.3f}")
print(f"p-value: {p_value:.4f}")
# p < 0.05 → ship the new algorithm!

Best Practices

  • Pre-register your hypothesis and significance level before running a test. "HARKing" (Hypothesizing After Results are Known) inflates false positives.
  • Always report effect size alongside p-values — a statistically significant improvement of 0.1% accuracy is rarely worth deploying a more complex model.
  • Use Bootstrap resampling for model comparison when sample sizes are small — it makes no distributional assumptions.
  • Monitor production models for distribution shift: use Kolmogorov-Smirnov tests (scipy.stats.ks_2samp) to detect when incoming data distribution differs significantly from training data.
