Information Theory for ML

Information theory, developed by Claude Shannon in 1948, quantifies the concept of "surprise" or "uncertainty" in information. Its concepts form the mathematical foundation of machine learning loss functions, decision tree splitting, and model evaluation — even if practitioners rarely realize they're using information theory every day.

Shannon Entropy: Measuring Uncertainty

Entropy measures the average uncertainty in a random variable. A fair coin has maximum entropy (maximum uncertainty). A coin that always lands heads has zero entropy (no uncertainty):

import numpy as np

def entropy(probabilities):
    """H(X) = -sum(p * log2(p)) — measured in bits"""
    probs = np.array(probabilities)
    return -np.sum(probs * np.log2(probs + 1e-10))

# Examples:
print(entropy([0.5, 0.5]))   # 1.0 bit — maximum uncertainty (fair coin)
print(entropy([0.9, 0.1]))   # 0.47 bits — low uncertainty (biased coin)
print(entropy([1.0, 0.0]))   # 0.0 bits — no uncertainty (deterministic)
print(entropy([0.25]*4))     # 2.0 bits — max uncertainty for 4 outcomes

In decision trees, the splitting criterion is to choose the feature that maximizes information gain — the reduction in entropy after splitting on that feature. This is why decision trees naturally select the most discriminative features first.

Cross-Entropy: The Classification Loss Function

Cross-entropy measures how well a predicted probability distribution approximates the true distribution. It's the standard loss for classification because minimizing cross-entropy is mathematically equivalent to maximizing the likelihood of correct predictions:

import torch
import torch.nn.functional as F

# Binary Cross-Entropy Loss (for sigmoid output — binary classification)
# H(y, p) = -y*log(p) - (1-y)*log(1-p)
predictions = torch.tensor([0.9, 0.2, 0.8, 0.1])  # Model confidence
targets = torch.tensor([1.0, 0.0, 1.0, 0.0])       # True labels
bce_loss = F.binary_cross_entropy(predictions, targets)

# Categorical Cross-Entropy (for softmax output — multi-class classification)
# H(y, p) = -sum(y_i * log(p_i))
logits = torch.tensor([[2.0, 1.0, 0.1],   # Prediction for sample 1
                        [0.1, 0.5, 2.5]])  # Prediction for sample 2
labels = torch.tensor([0, 2])              # True class indices
ce_loss = F.cross_entropy(logits, labels)  # = NLL(softmax(logits), labels)

KL Divergence: Measuring Distribution Difference

def kl_divergence(p, q):
    """KL(P||Q) = sum(P * log(P/Q))
    Measures how different Q is from the 'true' distribution P.
    KL=0 means identical distributions. Not symmetric: KL(P||Q) ≠ KL(Q||P)
    """
    p, q = np.array(p), np.array(q)
    return np.sum(p * np.log(p / (q + 1e-10) + 1e-10))

# Used in:
# - Variational Autoencoders (VAE): KL term forces latent space to be Gaussian
# - Knowledge Distillation: minimize KL between teacher and student distributions
# - Comparing topic model distributions

Mutual Information for Feature Selection

Mutual Information (MI) measures how much knowing one variable reduces uncertainty about another — a non-linear measure of feature relevance superior to correlation for detecting non-linear relationships:

from sklearn.feature_selection import mutual_info_classif

# Rank features by mutual information with the target variable
mi_scores = mutual_info_classif(X_train, y_train)
feature_importance = pd.Series(mi_scores, index=feature_names).sort_values(ascending=False)
print(feature_importance.head(10))
# MI=0 means feature is completely independent of target (useless for prediction)
# Higher MI = more information the feature provides about the target
