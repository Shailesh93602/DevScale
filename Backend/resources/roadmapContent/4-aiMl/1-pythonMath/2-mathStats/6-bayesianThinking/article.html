Bayesian Thinking & Probabilistic Models

Most engineers think in terms of frequencies — "the model is 87% accurate." Bayesian thinking is different: it treats probability as a measure of belief that updates as new evidence arrives. This framework leads to more honest models that quantify what they don't know, crucial for high-stakes ML applications in healthcare, finance, and autonomous systems.

Bayes' Theorem

Bayes' theorem describes how to update our belief in a hypothesis H after observing evidence E:

P(H|E) = P(E|H) × P(H) / P(E)

Posterior  = Likelihood × Prior / Evidence

Where:
  P(H)    = Prior: your belief BEFORE seeing evidence
  P(E|H)  = Likelihood: probability of seeing this evidence IF H is true
  P(E)    = Marginal: total probability of seeing this evidence (normalizing constant)
  P(H|E)  = Posterior: your updated belief AFTER seeing evidence

# Medical test example: Bayesian reasoning in practice
# Disease prevalence: 1% of population (prior)
# Test sensitivity: 99% (P(positive | disease) = 0.99)
# Test specificity: 95% (P(negative | no disease) = 0.95)

p_disease = 0.01       # Prior: P(Disease)
p_no_disease = 0.99    # P(No Disease)
p_pos_given_disease = 0.99     # Likelihood: P(Positive | Disease)
p_pos_given_no_disease = 0.05  # P(Positive | No Disease) — false positive rate

# P(Positive) — total probability of a positive test
p_positive = (p_pos_given_disease * p_disease + 
              p_pos_given_no_disease * p_no_disease)

# Posterior: P(Disease | Positive test)
p_disease_given_positive = (p_pos_given_disease * p_disease) / p_positive
print(f"Probability of disease given positive test: {p_disease_given_positive:.1%}")
# Output: 16.7% — even with a 99% accurate test, most positives are false positives!
# This counter-intuitive result shows why base rate (prior) matters enormously.

Bayesian Hyperparameter Optimization

Grid search exhaustively tests all combinations — exponentially expensive. Random search tests random combinations — better but blind. Bayesian optimization learns from each experiment to intelligently choose the next hyperparameters to test:

from skopt import BayesSearchCV
from skopt.space import Real, Integer, Categorical
from sklearn.ensemble import GradientBoostingClassifier

# Define the hyperparameter search space
search_spaces = {
    'n_estimators': Integer(100, 1000),
    'max_depth': Integer(3, 10),
    'learning_rate': Real(0.001, 0.3, prior='log-uniform'),  # Log scale
    'subsample': Real(0.6, 1.0),
    'max_features': Categorical(['sqrt', 'log2', None])
}

# Bayesian optimization — each trial informs the next choice of hyperparameters
# Builds a surrogate model (Gaussian Process) of the loss landscape
bayes_search = BayesSearchCV(
    GradientBoostingClassifier(),
    search_spaces,
    n_iter=50,     # Only 50 trials needed vs 1000+ for grid search
    cv=5,
    scoring='roc_auc',
    n_jobs=-1
)
bayes_search.fit(X_train, y_train)
print(f"Best params: {bayes_search.best_params_}")
print(f"Best ROC-AUC: {bayes_search.best_score_:.4f}")

Uncertainty Quantification

A Bayesian model produces a distribution over predictions, not just a single value — expressing what it knows and doesn't know:

from sklearn.calibration import CalibratedClassifierCV

# Standard classifier: probability estimates are often overconfident
# Bayesian/calibrated model: probabilities reflect true uncertainty

# Temperature scaling — a simple post-hoc calibration
