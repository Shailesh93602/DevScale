Regression & Classification Algorithms Deep Dive

Before gradient boosting and neural networks, linear models built the foundation of machine learning. Understanding them deeply reveals why regularization works, the mathematical nature of the bias-variance tradeoff, and when simpler models remain the superior choice.

Linear Regression with Regularization

from sklearn.linear_model import Ridge, Lasso, ElasticNet
from sklearn.model_selection import cross_val_score
import numpy as np

# Unregularized Linear Regression: minimizes sum of squared errors (SSE)
# Problem: with many features, overfits and coefficients become unstable

# Ridge (L2): penalizes sum of squared coefficients
# Effect: shrinks all coefficients toward (but not to) zero
# Best when: all features have some predictive value
ridge = Ridge(alpha=1.0)  # alpha controls regularization strength — tune this!

# Lasso (L1): penalizes sum of |coefficients|
# Effect: drives unimportant coefficients exactly to zero (automatic feature selection)
# Best when: many features are irrelevant
lasso = Lasso(alpha=0.01)

# ElasticNet: combines L1 + L2
# Best when: many correlated features exist (Lasso picks one arbitrarily; ElasticNet groups them)
elastic = ElasticNet(alpha=0.01, l1_ratio=0.5)  # 50% L1, 50% L2

The Bias-Variance Tradeoff

Total Error = Bias² + Variance + Irreducible Noise

High Bias (Underfitting):
  - Model is too simple to capture true patterns
  - Train error = high, Test error = high
  - Fix: more complex model, more features

High Variance (Overfitting):
  - Model memorizes training data including noise
  - Train error = low, Test error = high
  - Fix: more data, regularization, simpler model, feature selection

Algorithm Bias-Variance Spectrum:
Low Variance ←────────────────────────→ High Variance
High Bias                               Low Bias
Linear Regression → Random Forest → Full Decision Tree
Logistic Regression → Gradient Boosting → Neural Network (overfit)
