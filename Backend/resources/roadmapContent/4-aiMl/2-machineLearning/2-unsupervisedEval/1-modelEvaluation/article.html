Model Evaluation & Feature Engineering

A model is only as good as how well you evaluate it and prepare its features. This topic covers the essential techniques for both.

Classification Metrics

from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    roc_auc_score, confusion_matrix, ConfusionMatrixDisplay
)

# Accuracy — % correct (misleading for imbalanced datasets!)
accuracy = accuracy_score(y_test, y_pred)

# Precision — of predicted positives, how many are actually positive?
precision = precision_score(y_test, y_pred)

# Recall (Sensitivity) — of actual positives, how many did we find?
recall = recall_score(y_test, y_pred)

# F1-Score — harmonic mean of precision and recall
f1 = f1_score(y_test, y_pred)

# ROC-AUC — area under the ROC curve (threshold-independent, best overall metric)
auc = roc_auc_score(y_test, y_prob)

Bias-Variance Tradeoff

# Underfitting (High Bias):
# - Model too simple (e.g., linear model for non-linear data)
# - High training AND test error
# Fix: more complex model, more features, more data

# Overfitting (High Variance):
# - Model memorises training data (e.g., deep decision tree, no regularisation)
# - Low training error but HIGH test error
# Fix: regularisation, more training data, dropout, cross-validation
