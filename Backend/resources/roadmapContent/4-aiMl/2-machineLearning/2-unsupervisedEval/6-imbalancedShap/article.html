Imbalanced Learning & SHAP Explainability

Real-world ML problems are rarely perfectly balanced. Fraud detection might have 0.1% positive cases. Medical diagnosis might have 2% positive disease prevalence. A model that predicts "not fraud" for every transaction achieves 99.9% accuracy — but is completely useless. Handling imbalance correctly and explaining model decisions are two of the most practically important skills in production ML.

Handling Imbalanced Datasets

from imblearn.over_sampling import SMOTE, ADASYN
from imblearn.under_sampling import RandomUnderSampler

# Strategy 1: Class weights — cheapest, often best starting point
# Tells the model to penalize mistakes on the minority class more heavily
model = RandomForestClassifier(class_weight='balanced', n_estimators=200)
# LogisticRegression(class_weight='balanced') — same parameter

# Strategy 2: SMOTE — Synthetic Minority Oversampling Technique
# Creates synthetic minority samples by interpolating between existing samples
# IMPORTANT: SMOTE must be applied INSIDE the cross-validation loop (not before)
smote = SMOTE(sampling_strategy=0.3, k_neighbors=5, random_state=42)
# sampling_strategy=0.3 means: upsample minority to 30% of majority count

# Create a proper imbalanced-learn pipeline
pipeline = ImbPipeline([
    ('smote', SMOTE(random_state=42)),
    ('classifier', RandomForestClassifier(n_estimators=200))

SHAP Values: Understanding Model Predictions

SHAP (SHapley Additive exPlanations) is the gold standard for ML model explainability. Based on game theory, SHAP assigns each feature a contribution value for each individual prediction:

import shap

explainer = shap.TreeExplainer(xgb_model)
shap_values = explainer.shap_values(X_test)

# Global Explainability: which features matter most overall?
shap.summary_plot(shap_values, X_test, feature_names=feature_names)
# Shows: each dot = one sample, position = SHAP value (impact on prediction)
# High feature value + positive SHAP = feature increases P(churn)

# Individual explanation: why did the model predict churn for THIS customer?
customer_idx = 42
shap.waterfall_plot(shap.Explanation(
    values=shap_values[customer_idx],
    base_values=explainer.expected_value,
    data=X_test.iloc[customer_idx],
    feature_names=feature_names
))
# Output might show:
# Base value (average): +0.1
# support_calls=8: +0.35  ← This customer called 8 times (major churn indicator)
# tenure=2:        +0.22  ← Very new customer (high churn risk)
# contract=monthly:-0.05  ← On month contract (slightly more likely to churn)
# Final prediction: 0.82 churn probability

# Interaction analysis: which pairs of features interact?
shap_interaction = explainer.shap_interaction_values(X_test[:100])
