LLMs, RAG & Prompt Engineering

Large Language Models have transformed AI. Understanding how to use them effectively — through prompt engineering, RAG, and APIs — is a critical skill for modern developers.

How LLMs Work (High Level)

  • LLMs are transformer-based neural networks trained on billions of documents.
  • They predict the next token (word piece) given all previous tokens.
  • GPT-4, Claude, Gemini, Llama — all follow this paradigm.
  • Key concepts: context window (how much text they can "see"), tokens (not words — ~4 chars), temperature (randomness of output).

Using OpenAI API

from openai import OpenAI

client = OpenAI(api_key="sk-...")

# Chat completion
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "You are an expert Python tutor. Explain concepts clearly with examples."},
        {"role": "user",   "content": "Explain list comprehensions in Python"}
    ],
    temperature=0.3,       # 0=deterministic, 1=creative
    max_tokens=500,
)
print(response.choices[0].message.content)

GOOD: "Write a Python function that sorts a list of dictionaries by the 'date' key (ISO 8601 format) in descending order. Include type hints and docstring."

RAG — Retrieval-Augmented Generation

# Problem: LLMs have outdated knowledge and can hallucinate
# RAG solution: Retrieve relevant context → inject into prompt → generate

# Step 1: Embed documents into a vector database
from openai import OpenAI
import chromadb

client = OpenAI()

def create_embedding(text: str) -> list[float]:
    return client.embeddings.create(model="text-embedding-3-small", input=text).data[0].embedding

# Step 2: At query time — find relevant documents
def rag_query(question: str, db: chromadb.Collection) -> str:
    q_embedding = create_embedding(question)

    # Semantic search — find top 3 most relevant chunks
    results = db.query(query_embeddings=[q_embedding], n_results=3)
    context = "\n---\n".join(results["documents"][0])

    # Step 3: Inject context into prompt
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": f"Answer based on this context:\n{context}"},
            {"role": "user", "content": question}
        ]
    )
    return response.choices[0].message.content

Fine-tuning vs RAG

  • RAG — Update knowledge via retrieval. No training needed. Good for dynamic/frequently updated data.
  • Fine-tuning — Train the model to behave differently (tone, format, task). Good for style/behaviour changes.
  • Few-shot prompting — Cheapest option; provide examples in the prompt. Start here.
