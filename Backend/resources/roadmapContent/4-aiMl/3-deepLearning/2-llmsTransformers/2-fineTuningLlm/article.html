Fine-Tuning Large Language Models (LoRA & PEFT)

Pre-trained LLMs (GPT-4, Llama, Mistral) are general-purpose text predictors. Fine-tuning adapts them to a specific domain, task, or style using your own labeled dataset. The challenge: fine-tuning a 7-billion parameter model requires 80GB+ of GPU memory — far beyond typical hardware. Parameter-Efficient Fine-Tuning (PEFT) methods like LoRA solve this by training only a tiny fraction of the model's parameters.

When to Fine-Tune vs. Prompt Engineering

ApproachBest ForCost
Prompt EngineeringGeneral tasks, prototypingAPI costs only
RAGKnowledge from documents/DBEmbedding + retrieval costs
Fine-tuningSpecific style/format, domain jargon, consistent behaviorGPU time + labeled data

LoRA (Low-Rank Adaptation)

A transformer's attention and FFN layers have weight matrices (e.g., W is 4096×4096 = 16M parameters). LoRA doesn't update W directly — instead it adds two small matrices A (4096×r) and B (r×4096) where r is the rank (typically 8–64). The effective update is W' = W + B·A. Instead of 16M trainable parameters, you train only 2×4096×r (e.g., 65,536 for r=8) — a 250× reduction.

from transformers import AutoTokenizer, AutoModelForCausalLM, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType, prepare_model_for_kbit_training
from trl import SFTTrainer, SFTConfig
from datasets import load_dataset
import torch

# ─── Step 1: Load base model in 4-bit quantization (QLoRA) ──────────────────
# This reduces memory from ~28GB (7B model FP16) to ~5GB (7B model 4-bit)!
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",      # NormalFloat4 — preserves more information
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True  # Double quantization saves additional memory
)

model_name = "meta-llama/Llama-3.1-8B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_name)
tokenizer.pad_token = tokenizer.eos_token

model = AutoModelForCausalLM.from_pretrained(
    model_name,
    quantization_config=bnb_config,
    device_map="auto",            # Automatically distributes across available GPUs
    torch_dtype=torch.bfloat16
)
model = prepare_model_for_kbit_training(model)  # Required for QLoRA
# ─── Step 2: Configure LoRA ─────────────────────────────────────────────────
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,                      # Rank: higher = more capacity, more memory (8-64 typical)
    lora_alpha=32,             # Scaling factor (usually 2× or equal to r)
    lora_dropout=0.05,
    target_modules=[           # Which layers to apply LoRA to
        "q_proj", "k_proj",    # Query and Key attention projections
        "v_proj", "o_proj",    # Value and Output projections
        "gate_proj", "up_proj", "down_proj"  # FFN layers
    ],
    bias="none"
)
peft_model = get_peft_model(model, lora_config)
peft_model.print_trainable_parameters()
# Trainable: 20,971,520 || All: 8,051,306,496 || Trainable%: 0.26%
# Only 0.26% of parameters are updated during training!
# ─── Step 3: Prepare dataset ─────────────────────────────────────────────────
# Format: conversational instruction tuning (Alpaca-style)
dataset = load_dataset("alpaca", split="train[:5000]")  # Small subset for demo

def format_prompt(example):
    """Format each example into an instruction following prompt"""
    if example["input"]:
        return f"### Instruction:\n{example['instruction']}\n\n### Input:\n{example['input']}\n\n### Response:\n{example['output']}"
    return f"### Instruction:\n{example['instruction']}\n\n### Response:\n{example['output']}"

dataset = dataset.map(lambda x: {"text": format_prompt(x)})

# ─── Step 4: Train with SFTTrainer (Supervised Fine-Tuning) ──────────────────
trainer = SFTTrainer(
    model=peft_model,
    train_dataset=dataset,
    args=SFTConfig(
        output_dir="./output/lora-llama",
        num_train_epochs=3,
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,  # Effective batch size = 4×4 = 16
        gradient_checkpointing=True,    # Trade compute for memory (~30% less VRAM)
        learning_rate=2e-4,
        lr_scheduler_type="cosine",
        warmup_ratio=0.05,
        logging_steps=25,
        save_steps=200,
        fp16=False,
        bf16=True,                      # bfloat16 is more stable than fp16 for LLMs
        report_to="wandb",
        dataset_text_field="text",
        max_seq_length=2048,
        packing=True,                   # Pack multiple short examples into one sequence
    )
)
trainer.train()
# ─── Step 5: Save & Merge LoRA Weights ──────────────────────────────────────
# Save just the LoRA adapter weights (~100MB vs 16GB for full model)
peft_model.save_pretrained("./output/lora-adapter")
tokenizer.save_pretrained("./output/lora-adapter")

# For inference, merge LoRA into base model (optional — needed for GGUF/ONNX export)
from peft import PeftModel
base_model = AutoModelForCausalLM.from_pretrained(model_name, torch_dtype=torch.float16)
merged_model = PeftModel.from_pretrained(base_model, "./output/lora-adapter")
merged_model = merged_model.merge_and_unload()  # Fuses W + B·A into single matrix
merged_model.save_pretrained("./output/merged-model")

Inference with the Fine-Tuned Model

from transformers import pipeline

# Load with the adapter for inference (memory-efficient)
pipe = pipeline(
    "text-generation",
    model=peft_model,
    tokenizer=tokenizer,
    torch_dtype=torch.bfloat16
)

prompt = "### Instruction:\nExplain gradient descent in simple terms.\n\n### Response:\n"
result = pipe(
    prompt,
    max_new_tokens=256,
    temperature=0.7,     # Lower = more deterministic, higher = more creative
    top_p=0.9,
    do_sample=True,
    repetition_penalty=1.1
)
print(result[0]['generated_text'])

Evaluating Fine-Tuned Models

from evaluate import load

# ROUGE: measures n-gram overlap between generated and reference text
rouge = load("rouge")
predictions = ["The model generates concise summaries."]
references = ["The fine-tuned model generates accurate and concise summaries."]
results = rouge.compute(predictions=predictions, references=references)
print(results)  # {'rouge1': 0.75, 'rouge2': 0.60, 'rougeL': 0.70}

Best Practices

  • Start with QLoRA (4-bit quantization + LoRA) — achieves 90%+ of full fine-tuning quality with 6× less GPU memory.
  • Use 1,000–10,000 high-quality examples rather than 100,000 mediocre ones — data quality is far more impactful than quantity for instruction fine-tuning.
  • Apply chat templates correctly for instruction-tuned models (tokenizer.apply_chat_template()) — wrong formatting prevents the model from following instructions properly.
  • Always evaluate on a held-out test set before deploying. A model that scores higher on training loss but lower on your test prompts has overfit to your fine-tuning data.
