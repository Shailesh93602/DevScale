Vector Databases: Pinecone & ChromaDB

Traditional databases store structured data and retrieve it with exact-match queries: WHERE email = 'user@example.com'. But how do you query "find the 10 most semantically similar products to this one" or "find the passages from these 10,000 documents most relevant to this user's question"? You can't do that with SQL. Vector databases store high-dimensional numerical representations of data (embeddings) and retrieve them by similarity using algorithms like Approximate Nearest Neighbour (ANN). They are the core infrastructure behind semantic search, RAG (Retrieval-Augmented Generation) systems, and recommendation engines.

What Are Embeddings?

An embedding is a list of floating-point numbers (e.g., 1536 numbers for OpenAI's text-embedding-3-small) that encodes the semantic meaning of text, images, or audio. Two items with similar meaning have embeddings that are close together in this high-dimensional space — measurable with cosine similarity:

from openai import OpenAI

client = OpenAI()  # Uses OPENAI_API_KEY env variable

def get_embedding(text: str, model: str = "text-embedding-3-small") -> list[float]:
    text = text.replace("\n", " ")
    response = client.embeddings.create(input=[text], model=model)
    return response.data[0].embedding  # List of 1536 floats

# Example:
query_embedding = get_embedding("How do I reset my password?")
doc_embedding   = get_embedding("Steps to recover your account access")
# These will have HIGH cosine similarity (~0.92)
# Despite using completely different words

Pinecone — Managed Vector Database

Pinecone is a fully managed, cloud-native vector database optimized for production-scale similarity search. No infrastructure to manage — just an API:

from pinecone import Pinecone, ServerlessSpec

pc = Pinecone(api_key=os.environ["PINECONE_API_KEY"])

# Create an index (do this once, not on every run)
pc.create_index(
    name="product-catalog",
    dimension=1536,          # Must match your embedding model's output size
    metric="cosine",         # Options: cosine, euclidean, dotproduct
    spec=ServerlessSpec(cloud="aws", region="us-east-1")
)

index = pc.Index("product-catalog")

# ─── Upsert (insert or update) product embeddings ────────────────────────
def index_products(products: list[dict]):
    vectors = []
    for product in products:
        embedding = get_embedding(f"{product['name']} {product['description']}")
        vectors.append({
            "id": str(product["id"]),
            "values": embedding,
            "metadata": {               # Store filterable metadata alongside the vector
                "name": product["name"],
                "category": product["category"],
                "price": product["price"],
                "in_stock": product["in_stock"]
            }
        })
    index.upsert(vectors=vectors, namespace="products")

# ─── Semantic search ──────────────────────────────────────────────────────
def semantic_search(query: str, category_filter: str = None, top_k: int = 10):
    query_embedding = get_embedding(query)
    
    filter_dict = {}
    if category_filter:
        filter_dict = {"category": {"$eq": category_filter}}
    
    results = index.query(
        vector=query_embedding,
        top_k=top_k,
        include_metadata=True,
        filter=filter_dict if filter_dict else None,
        namespace="products"
    )
    
    return [
        {
            "id": match["id"],
            "score": match["score"],    # Cosine similarity: 1.0 = identical
            "product": match["metadata"]
        }
        for match in results["matches"]
        if match["score"] > 0.75  # Filter by minimum similarity threshold
    ]

# Usage:
results = semantic_search("waterproof hiking boots", category_filter="footwear")
print(results[0])  # → {"id": "42", "score": 0.94, "product": {...}}

ChromaDB — Local/Self-Hosted Vector Database

ChromaDB is the leading open-source vector database, perfect for development, local RAG pipelines, and self-hosted deployments. Zero infrastructure needed for local development:

import chromadb
from chromadb.utils import embedding_functions

# Local persistent storage (also supports HTTP server mode for production)
client = chromadb.PersistentClient(path="./chroma_data")

# Use OpenAI's embedding function directly (Chroma calls the API automatically)
openai_ef = embedding_functions.OpenAIEmbeddingFunction(
    api_key=os.environ["OPENAI_API_KEY"],
    model_name="text-embedding-3-small"
)

collection = client.get_or_create_collection(
    name="documentation",
    embedding_function=openai_ef,
    metadata={"hnsw:space": "cosine"}  # Cosine similarity metric
)

# ─── Add documents (Chroma generates embeddings automatically) ────────────
collection.add(
    documents=[
        "To reset your password, click 'Forgot Password' on the login page.",
        "Your order ships within 2-3 business days after payment confirmation.",
        "To cancel an order, contact support within 24 hours of purchase.",
    ],
    metadatas=[
        {"source": "help-center", "category": "account"},
        {"source": "help-center", "category": "shipping"},
        {"source": "help-center", "category": "orders"},
    ],
    ids=["doc_1", "doc_2", "doc_3"]
)

# ─── Query ────────────────────────────────────────────────────────────────
results = collection.query(
    query_texts=["How long does delivery take?"],  # Chroma embeds this automatically
    n_results=3,
    where={"category": "shipping"}  # Metadata filter
)

print(results["documents"][0])   # Top 3 most relevant passages
print(results["distances"][0])   # Cosine distances (lower = more similar)

Building a Simple RAG Pipeline

RAG (Retrieval-Augmented Generation) combines a vector database with an LLM: retrieve the most relevant context, then have the LLM generate an answer grounded in that context:

def answer_question(question: str) -> str:
    # 1. Retrieve relevant context from ChromaDB
    results = collection.query(query_texts=[question], n_results=3)
    context = "\n\n".join(results["documents"][0])
    
    # 2. Generate answer with LLM, grounded in retrieved context
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": "Answer questions using ONLY the provided context. If unsure, say so."},
            {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {question}"}
        ]
    )
    return response.choices[0].message.content

answer = answer_question("How long does shipping take?")
# → "Your order ships within 2-3 business days after payment confirmation."

Best Practices

  • Always chunk large documents before embedding — embeddings of 8,000-word articles lose semantic precision. Chunk to 500–1000 tokens with 100-token overlap between chunks.
  • Store original document IDs in metadata so you can trace back retrieved chunks to their source document.
  • Use Pinecone for production and ChromaDB for local development — they have nearly identical APIs, making migration straightforward.
  • Set a minimum similarity threshold (e.g., 0.75 cosine similarity) — returning low-confidence matches to an LLM produces hallucinated answers.
