React Native Core Components

Unlike React for the web, React Native does not use HTML tags like <div>, <span>, or <img>. Instead, it provides a set of Core Components that map directly to the native UI building blocks of iOS (UIKit/SwiftUI) and Android (View/Jetpack Compose).

View and Text: The Foundation

The <View> is the fundamental container in React Native, analogous to the web's <div>. It supports layout with Flexbox, styling, and basic touch handling. The <Text> component is the only way to display text — text strings cannot be bare inside a <View>.

import { View, Text, StyleSheet } from 'react-native';

export default function BasicLayout() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Welcome to React Native</Text>
      <Text style={styles.subtitle}>Built with native performance.</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,                // Take up all available space
    padding: 20,
    backgroundColor: '#fff',
    justifyContent: 'center', // Vertically center (default flex direction is column)
    alignItems: 'center',     // Horizontally center
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 8,
    color: '#111827',
  },
  subtitle: {
    fontSize: 16,
    color: '#6b7280',
  }
});

Image & ImageBackground

The <Image> component handles both local assets and remote URLs. A critical quirk of React Native: remote images must have explicit width and height, or they will render at 0x0 size and remain invisible.

import { Image, ImageBackground, View, Text } from 'react-native';

// Local asset (size is automatically inferred)
const localLogo = require('./assets/logo.png');

export default function ImageDemo() {
  return (
    <View>
      <Image source={localLogo} style={{ width: 100, height: 100 }} />
      
      {/* Remote image — MUST have dimensions */}
      <Image 
        source={{ uri: 'https://example.com/profile.jpg' }} 
        style={{ width: 150, height: 150, borderRadius: 75 }} 
        resizeMode="cover" 
      />

      {/* Background image holding text */}
      <ImageBackground 
        source={{ uri: 'https://example.com/pattern.jpg' }} 
        style={{ width: '100%', height: 200, justifyContent: 'center' }}
      >
        <Text style={{ color: 'white', textAlign: 'center' }}>Hero Text</Text>
      </ImageBackground>
    </View>
  );
}

TextInput & Interactions

<TextInput> is the foundational component for inputting text via the keyboard. Handling touches requires specialized components called Touchables or the newer <Pressable> API.

import { TextInput, TouchableOpacity, View, Text, Alert } from 'react-native';
import { useState } from 'react';

export default function InputDemo() {
  const [text, setText] = useState('');

  return (
    <View style={{ padding: 20 }}>
      <TextInput
        style={{ height: 40, borderColor: 'gray', borderWidth: 1, padding: 10, borderRadius: 8 }}
        onChangeText={setText}
        value={text}
        placeholder="Enter your email"
        keyboardType="email-address"
        autoCapitalize="none"
        secureTextEntry={false}  // True for passwords
      />

      {/* TouchableOpacity decreases opacity slightly when pressed (standard iOS feel) */}
      <TouchableOpacity 
        style={{ marginTop: 20, backgroundColor: '#6366f1', padding: 15, borderRadius: 8 }}
        onPress={() => Alert.alert('Submitted', `You typed: ${text}`)}
      >
        <Text style={{ color: 'white', textAlign: 'center', fontWeight: 'bold' }}>Submit</Text>
      </TouchableOpacity>
    </View>
  );
}

Handling Lists: ScrollView vs. FlatList

The standard <ScrollView> renders all its children instantly. This is fine for a settings screen, but fatal for memory performance if you render a list of 1,000 items. For long lists, always use <FlatList>, which relies on "virtualization" (only rendering items currently visible on screen).

import { FlatList, View, Text, StyleSheet } from 'react-native';

const DATA = Array.from({ length: 1000 }).map((_, i) => ({
  id: `item-${i}`,
  title: `List Item ${i}`
}));

export default function ListDemo() {
  // Render function extracted for performance
  const renderItem = ({ item }) => (
    <View style={styles.item}>
      <Text style={styles.title}>{item.title}</Text>
    </View>
  );

  return (
    <FlatList
      data={DATA}
      renderItem={renderItem}
      keyExtractor={item => item.id}
      ItemSeparatorComponent={() => <View style={{ height: 1, backgroundColor: '#e5e7eb' }} />}
      // Optional: Pull-to-refresh
      refreshing={false}
      onRefresh={() => console.log('Refreshing...')}
    />
  );
}

const styles = StyleSheet.create({
  item: { padding: 20 },
  title: { fontSize: 16 }
});

Best Practices

  • Never put text outside of a <Text> component. It will crash natively.
  • Use <Pressable> or <TouchableOpacity> instead of <Button>. The core <Button> component looks totally different on iOS vs Android and is impossible to style identically. Custom components are always required for branded buttons.
  • Use <KeyboardAvoidingView>. Unlike the web, the mobile virtual keyboard pops up over the UI, hiding your input fields. Wrap your screen forms in KeyboardAvoidingView to automatically push the UI up.
  • Avoid deep nesting of Views. React Native translates every <View> to an actual platform view grouping. Deeply nested views bloat the mobile layout tree and degrade rendering speed.
