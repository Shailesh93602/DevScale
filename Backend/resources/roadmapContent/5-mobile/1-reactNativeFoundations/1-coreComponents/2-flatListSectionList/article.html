FlatList, SectionList & ScrollView Deep Dive

FlatList is the workhorse of React Native UI — almost every production app uses it for feeds, contact lists, product catalogs, and message threads. Understanding its internals and optimization techniques is essential for keeping your UI smooth at 60fps even with thousands of items.

FlatList: Windowed Rendering

FlatList renders only the items currently visible on screen plus a small buffer — it doesn't render all 10,000 list items at once. Items outside the view are unmounted to save memory:

import React, { useCallback } from 'react';
import { FlatList, View, Text, StyleSheet, RefreshControl } from 'react-native';

const ProductList = ({ products, onRefresh, isRefreshing, onEndReached }) => {
  // keyExtractor: stable, unique key per item — must NOT use array index
  // Using index causes unnecessary re-renders when items are added/removed
  const keyExtractor = useCallback((item) => item.id.toString(), []);

  // renderItem: wrap in useCallback to prevent recreation on every parent render
  const renderItem = useCallback(({ item, index }) => (
    <ProductCard product={item} index={index} />
  ), []);

  // getItemLayout: if all items have FIXED height, provide this for massive perf gain
  // Allows FlatList to calculate scroll position without measuring each item
  const getItemLayout = useCallback((_, index) => ({
    length: ITEM_HEIGHT,       // Fixed item height in pixels
    offset: ITEM_HEIGHT * index, // Distance from top of list
    index,
  }), []);

  return (
    <FlatList
      data={products}
      keyExtractor={keyExtractor}
      renderItem={renderItem}
      getItemLayout={getItemLayout}     // Remove if items have variable height!

      // Pull-to-refresh
      refreshControl={
        <RefreshControl refreshing={isRefreshing} onRefresh={onRefresh} />
      }

      // Infinite scroll — fire when 50% from the bottom
      onEndReached={onEndReached}
      onEndReachedThreshold={0.5}

      // Performance tuning
      initialNumToRender={10}           // Render 10 items on first paint
      maxToRenderPerBatch={10}          // Render up to 10 items per JS batch
      windowSize={5}                    // Render 5 screen-heights of items
      removeClippedSubviews={true}      // Unmount views outside viewport (Android)

      // Separator between items
      ItemSeparatorComponent={() => <View style={styles.separator} />}
      ListEmptyComponent={<EmptyState />}
      ListFooterComponent={isRefreshing ? <LoadingSpinner /> : null}
    />
  );
};

SectionList: Grouped Data with Sticky Headers

import { SectionList } from 'react-native';

// SectionList is like FlatList but supports sections with headers
// Perfect for: contacts (A-Z), order history (by date), settings screens

const sections = [
  { title: 'Today',     data: todayOrders },
  { title: 'This Week', data: weekOrders },
  { title: 'Older',     data: olderOrders },
];

<SectionList
  sections={sections}
  keyExtractor={(item) => item.id.toString()}
  renderItem={({ item }) => <OrderRow order={item} />}
  renderSectionHeader={({ section: { title } }) => (
    <View style={styles.sectionHeader}>
      <Text style={styles.sectionTitle}>{title}</Text>
    </View>
  )}
  stickySectionHeadersEnabled={true}  // Headers stick at top while scrolling through section
/>

FlashList: The FlatList Replacement

import { FlashList } from '@shopify/flash-list';

// FlashList (by Shopify) recycles native cell views instead of JS components
// Result: 10x less memory usage, eliminates blank flickering on fast scroll

<FlashList
  data={products}
  renderItem={({ item }) => <ProductCard product={item} />}
  estimatedItemSize={120}    // REQUIRED: estimated height for virtualization
  keyExtractor={(item) => item.id}
/>

// When to use FlashList over FlatList:
// - Lists with 100+ items that users scroll quickly
// - Blank flash/flicker issues in FlatList despite optimization
// - Memory pressure causing crashes on low-end Android devices
