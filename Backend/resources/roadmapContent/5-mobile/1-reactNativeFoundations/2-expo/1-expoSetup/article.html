Expo Setup & Build Tools

Setting up and configuring the React Native development environment is often the most frustrating step for newcomers. Expo eliminates most of that friction by providing a managed workflow, pre-configured native modules, and cloud build services — letting you focus on building your app, not configuring Xcode or Android Studio.

Expo Managed Workflow vs. Bare Workflow

Managed WorkflowBare Workflow
SetupZero native configurationStandard React Native setup (Xcode + Android Studio)
Native codeExpo handles all native codeFull access to iOS/Android source
Custom native modulesOnly Expo-compatible packagesAny npm package including custom native bridges
EAS Build✅ Fully supported✅ Fully supported
Best for85% of projectsProjects needing deep native customization

Creating a New Expo Project

# Create a new project using Expo's TypeScript template (recommended)
npx create-expo-app@latest MyApp --template blank-typescript

# Navigate into it
cd MyApp

# Start the Metro bundler and Expo Go development server
npx expo start

Scan the QR code with the Expo Go app on your phone. Your app loads instantly on your physical device over Wi-Fi — no USB cable, no simulators needed. Every time you save a file, Expo instantly hot-reloads the changes on your device via Fast Refresh.

Project Structure

MyApp/
├── app/                # File-based routing (Expo Router)
│   ├── _layout.tsx    # Root navigator configuration
│   ├── index.tsx      # Home screen (maps to "/" route)
│   └── profile.tsx    # Profile screen (maps to "/profile" route)
├── assets/            # Static files: images, fonts, icons
├── components/        # Reusable UI components
├── app.json           # Expo configuration: app name, icon, splash screen
├── eas.json           # EAS Build configuration (production, preview profiles)
└── package.json

app.json — The Expo Configuration File

app.json is the single source of truth for your app's identity and native settings. You never touch Info.plist (iOS) or AndroidManifest.xml (Android) directly in the managed workflow:

{
  "expo": {
    "name": "MyApp",
    "slug": "my-app",
    "version": "1.0.0",
    "icon": "./assets/icon.png",        // 1024x1024 PNG, no transparency
    "splash": {
      "image": "./assets/splash.png",  // Shown while app loads
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "ios": {
      "bundleIdentifier": "com.yourcompany.myapp",  // Unique Apple ID
      "supportsTablet": true
    },
    "android": {
      "package": "com.yourcompany.myapp",            // Unique Google ID
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      }
    },
    "plugins": [
      // Config plugins automatically inject native permissions
      ["expo-camera", { "cameraPermission": "Allow MyApp to access your camera." }],
      ["expo-location", { "locationAlwaysAndWhenInUsePermission": "..." }]
    ]
  }
}

Installing Expo-Compatible Packages

Use npx expo install instead of npm install — it automatically selects the version of the package that is compatible with your current Expo SDK version:

# Always use this for Expo packages:
npx expo install expo-image-picker expo-location expo-camera

# For third-party packages without a native layer, npm/yarn is fine:
npm install lodash date-fns

Expo Router — File-Based Navigation

Expo Router brings Next.js-style file-based routing to mobile. Create a file → it becomes a screen:

// app/(tabs)/index.tsx — Home tab
export default function HomeScreen() {
  return Home;
}

// app/(tabs)/profile.tsx — Profile tab (automatically gets a tab item)
export default function ProfileScreen() {
  return Profile;
}

// app/[id].tsx — Dynamic route (like /post/123)
import { useLocalSearchParams } from 'expo-router';
export default function PostScreen() {
  const { id } = useLocalSearchParams();
  return Post: {id};
}

Development Builds vs. Expo Go

Expo Go is a sandbox — it works for most development, but cannot run packages with custom native code (like react-native-vision-camera). For those, use a Development Build:

# Install expo-dev-client
npx expo install expo-dev-client

# Build a dev client for your physical iOS device
eas build --profile development --platform ios

# Then start the dev server (your custom client app connects to it)
npx expo start --dev-client

Expo SDK Updates

Expo releases major SDK versions every ~3 months. To upgrade:

npx expo install expo@latest
npx expo-doctor  # Diagnoses version mismatches in your dependencies

Best Practices

  • Keep your Expo SDK version consistent — mixing SDK 50 and SDK 51 packages causes hard-to-debug native crashes.
  • Use npx expo-doctor weekly during active development to catch incompatible package versions early.
  • Commit eas.json to Git, but never commit your .env file — use EAS Secrets for production environment variables.
  • For complex animations or custom native features, consider ejecting to the bare workflow rather than fighting Expo's constraints.
