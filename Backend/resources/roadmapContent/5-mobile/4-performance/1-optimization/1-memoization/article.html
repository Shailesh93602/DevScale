Memoization in React Native

React Native apps re-render components automatically whenever state or props change. While this is the right default, it becomes a performance bottleneck when complex components or expensive calculations are triggered far more often than needed. Memoization is the technique of caching the output of a function so that if the same inputs are provided again, the cached result is returned immediately instead of recomputing.

Understanding Why Components Re-render

In React Native, a component re-renders when:

  1. Its own state changes (via useState or useReducer).
  2. Its parent re-renders (props may or may not have changed).
  3. A context value it subscribes to changes.

The key insight: when a parent re-renders, all child components re-render by default — even if the child's props are identical. Memoization breaks this chain for expensive children.

React.memo — Memoizing Components

React.memo is a Higher Order Component that wraps a component and does a shallow comparison of its props before re-rendering. If props haven't changed, the previous render output is reused.

import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

interface ProductCardProps {
  name: string;
  price: number;
  onAddToCart: () => void;
}

// Without React.memo: re-renders every time the parent renders
// With React.memo: only re-renders when name, price, or onAddToCart changes
const ProductCard = React.memo({ name, price, onAddToCart }: ProductCardProps) => {
  console.log(`Rendering ProductCard: ${name}`);  // Use this to verify memoization works

  return (
    <View style={styles.card}>
      <Text style={styles.name}>{name}</Text>
      <Text style={styles.price}>${price.toFixed(2)}</Text>
      <Text onPress={onAddToCart} style={styles.button}>Add to Cart</Text>
    </View>
  );
});

// Optional: custom comparison function for deep comparison
const MemoizedCard = React.memo(ProductCard, (prevProps, nextProps) => {
  return prevProps.name === nextProps.name && prevProps.price === nextProps.price;
  // Return true = props are equal = skip re-render
});

useMemo — Memoizing Expensive Calculations

useMemo caches the result of a computation between renders. It only recomputes when one of its declared dependencies changes.

import { useMemo } from 'react';

const ProductListScreen = ({ products, searchQuery }: Props) => {
  // ❌ Without useMemo: filters and sorts on EVERY render
  // const filtered = products.filter(...).sort(...);

  // ✅ With useMemo: only recomputes when products or searchQuery changes
  const filteredAndSorted = useMemo(() => {
    console.log('Filtering products...');  // Should only log when deps change
    return products
      .filter(p => p.name.toLowerCase().includes(searchQuery.toLowerCase()))
      .sort((a, b) => a.price - b.price);
  }, [products, searchQuery]);

  return <FlatList data={filteredAndSorted} renderItem={...} />;
};

useCallback — Memoizing Functions

Functions defined inside a component get a new reference on every render. When you pass a function as a prop to a React.memo child, a new function reference causes the child to re-render even if the logic is identical. useCallback solves this:

import { useCallback, useState } from 'react';

const ShoppingCart = ({ products }: Props) => {
  const [cart, setCart] = useState([]);

  // ❌ New function reference on every render → breaks React.memo on ProductCard
  // const handleAddToCart = (product) => setCart(prev => [...prev, product]);

  // ✅ Stable function reference — only changes when setCart changes (never)
  const handleAddToCart = useCallback((product: Product) => {
    setCart(prev => [...prev, product]);
  }, []); // Empty deps = never changes

  return (
    <>
      {products.map(product => (
        <ProductCard   // This is the React.memo'd component from above
          key={product.id}
          name={product.name}
          price={product.price}
          onAddToCart={() => handleAddToCart(product)}
        />
      ))}
    </>
  );
};

The Inline Object Problem

A common mistake that silently breaks React.memo:

// ❌ WRONG: New style object created on every render → ProductCard always re-renders
<ProductCard style={{ margin: 10, padding: 5 }} ... />

// ✅ CORRECT: Define styles outside component or use useMemo
const styles = StyleSheet.create({ card: { margin: 10, padding: 5 } });
<ProductCard style={styles.card} ... />

When NOT to Memoize

Memoization has a cost — it adds memory overhead and the comparison computation itself. Don't use it for:

  • Simple components that render fast (just a Text or View) — the comparison cost exceeds the render cost.
  • Components whose props change on nearly every render — memoization never kicks in, just adds overhead.
  • Components with complex props (large arrays) where shallow comparison is insufficient.

Best Practices

  • Use React.DevTools Profiler or console.log in render to verify memoization is actually working before adding it everywhere.
  • Apply memoization from the bottom up — memoize leaf components first, then work upward only if profiling shows a bottleneck.
  • Always move static objects and arrays outside the component body; useMemo for dynamic ones.
