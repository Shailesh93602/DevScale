FlatList Optimization in React Native

FlatList is React Native's primary component for rendering scrollable lists. It is far more performant than using a ScrollView with .map() because it uses windowed rendering — it only renders the items currently visible on screen (plus a small buffer), recycling component instances as the user scrolls. However, FlatList can still become slow if not configured correctly, especially on mobile devices with limited RAM.

The Core Problem: Unnecessary Re-renders

The most common FlatList performance issue is every item re-rendering when any state in the parent changes. Here's the solution chain:

import React, { useCallback, memo } from 'react';
import { FlatList, View, Text, StyleSheet } from 'react-native';

// ─── Step 1: Memoize the render item component ────────────────────────────
const ProductItem = memo(({ item, onPress }) => {
  return (
    <View style={styles.item}>
      <Text style={styles.name}>{item.name}</Text>
      <Text style={styles.price}>${item.price}</Text>
      <Text style={styles.btn} onPress={() => onPress(item.id)}>View</Text>
    </View>
  );
});

// ─── Step 2: Memoize renderItem to prevent new function refs ─────────────
const ProductList = ({ products }) => {
  const [selectedId, setSelectedId] = React.useState(null);

  // useCallback keeps renderItem reference stable → prevents all items re-rendering
  const renderItem = useCallback(({ item }) => (
    <ProductItem item={item} onPress={setSelectedId} />
  ), []); // No deps = never changes

  // ─── Step 3: Provide a stable key extractor ──────────────────────────
  const keyExtractor = useCallback((item) => item.id.toString(), []);

  return (
    <FlatList
      data={products}
      renderItem={renderItem}
      keyExtractor={keyExtractor}

      // ─── Step 4: Performance props ──────────────────────────────────
      // Pre-render items 10 screens ahead beyond the visible window
      windowSize={10}

      // Number of items to render in the initial batch
      initialNumToRender={15}

      // Max items to update per JS frame (throttles updates)
      maxToRenderPerBatch={10}

      // Remove off-screen items from memory (trade CPU for RAM)
      removeClippedSubviews={true}

      // Avoids re-renders when data reference changes but items are equal
      // Only use if your item objects are immutable
      extraData={selectedId}  // Re-render list when selectedId changes
    />
  );
};

getItemLayout — The Single Biggest Win for Fixed-Height Lists

By default, FlatList must measure each item's height dynamically before it can calculate scroll positions and jump to any index. If your items have a fixed height, getItemLayout pre-computes all positions, dramatically improving scroll performance:

const ITEM_HEIGHT = 80; // Fixed height per item in dp

<FlatList
  data={products}
  renderItem={renderItem}
  keyExtractor={keyExtractor}
  getItemLayout={(data, index) => ({
    length: ITEM_HEIGHT,
    offset: ITEM_HEIGHT * index,
    index,
  })}
  // Now scrollToIndex() and initialScrollIndex work instantly
  initialScrollIndex={50}  // Jump directly to item 50 without measuring 0-49
/>

Handling Images in Lists

Images are the most common FlatList performance killer. Every image decode is a CPU-intensive operation:

// ❌ Slow: Plain Image re-loads on every render
<Image source={{ uri: item.imageUrl }} style={{ width: 80, height: 80 }} />

// ✅ Fast: Use react-native-fast-image for caching and priority loading
import FastImage from 'react-native-fast-image';

<FastImage
  style={{ width: 80, height: 80 }}
  source={{ uri: item.imageUrl, priority: FastImage.priority.normal }}
  resizeMode={FastImage.resizeMode.cover}
  // FastImage caches to disk – subsequent renders load from disk, not network
/>

Pagination — Loading Data Incrementally

const ProductList = () => {
  const [products, setProducts] = React.useState([]);
  const [page, setPage] = React.useState(1);
  const [loading, setLoading] = React.useState(false);

  const loadMore = useCallback(async () => {
    if (loading) return;
    setLoading(true);
    const newProducts = await fetchProducts(page);
    setProducts(prev => [...prev, ...newProducts]);
    setPage(prev => prev + 1);
    setLoading(false);
  }, [loading, page]);

  return (
    <FlatList
      data={products}
      renderItem={renderItem}
      keyExtractor={keyExtractor}
      onEndReached={loadMore}          // Triggers when user scrolls near bottom
      onEndReachedThreshold={0.5}      // At 50% of the last item
      ListFooterComponent={loading ? <ActivityIndicator /> : null}
    />
  );
};

Best Practices Summary

  • Always wrap your renderItem component in React.memo and pass a useCallback-wrapped renderItem function.
  • Use getItemLayout whenever items have a fixed height — it's the single highest-impact optimization for long lists.
  • Never use ScrollView + .map() for lists longer than ~20 items — it renders all items at once and will cause memory issues.
  • Replace React Native's Image with react-native-fast-image for list images — it caches to disk and uses better memory strategies.
