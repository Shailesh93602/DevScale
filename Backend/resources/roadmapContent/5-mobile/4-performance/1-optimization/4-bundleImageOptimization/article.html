Bundle Optimization & Image Performance

A bloated JavaScript bundle and unoptimized images are the two biggest causes of slow app startup on mobile. The goal: get the Time-to-Interactive under 2 seconds on a mid-range Android device, where most of your users will be.

// Tree-shaking: import specific functions, not entire libraries
// BAD — imports entire lodash (500KB!)
import _ from 'lodash';
const result = _.debounce(fn, 300);

// GOOD — imports only debounce (2KB)
import debounce from 'lodash/debounce';

// BAD — imports entire moment.js (300KB!)
import moment from 'moment';

// GOOD — use date-fns (modular, tree-shakable, 5-30KB)
import { format, formatDistanceToNow } from 'date-fns';

Lazy Loading Screens

import { createNativeStackNavigator } from '@react-navigation/native-stack';
import React, { lazy, Suspense } from 'react';
import { ActivityIndicator } from 'react-native';

// Lazy load screens that aren't needed on startup
// The JS bundle for these screens is only downloaded when the user navigates to them
const ProfileScreen = lazy(() => import('./screens/ProfileScreen'));
const SettingsScreen = lazy(() => import('./screens/SettingsScreen'));
const OrderHistoryScreen = lazy(() => import('./screens/OrderHistoryScreen'));

const Stack = createNativeStackNavigator();

function AppNavigator() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home" component={HomeScreen} />  {/* Eager loaded */}
      <Stack.Screen name="Profile">
        {(props) => (
          <Suspense fallback={<ActivityIndicator />}>
            <ProfileScreen {...props} />
          </Suspense>
        )}
      </Stack.Screen>
    </Stack.Navigator>
  );
}

Image Performance with expo-image

import { Image } from 'expo-image';

// expo-image: dramatically faster than React Native's built-in Image
// - Aggressive disk and memory caching
// - Supports WebP (30-40% smaller than JPEG at same quality)
// - Blurhash placeholder while loading (prevents layout shifts)
// - Progressive loading
// - Prefetching

// Production-ready image component
<Image
  source={{ uri: 'https://cdn.myapp.com/products/img.webp' }}
  style={{ width: 300, height: 200 }}
  contentFit="cover"              // object-fit equivalent
  placeholder="L6PZfSi_.AyE_3t7t7R**0o#DgR4"  // Blurhash placeholder
  transition={200}                // 200ms fade-in after load
  cachePolicy="memory-disk"       // Cache in both memory and disk
  recyclingKey={item.id}          // Important for FlatList — reuse image slots
/>

// Prefetch images before user navigates (e.g., next page images)
async function prefetchNextPageImages(imageUrls) {
  await Image.prefetch(imageUrls);
}
