End-to-End Testing with Detox

Unit tests verify individual components in isolation, but they can't catch integration failures — bugs that only appear when multiple components interact and real user flows run end-to-end. Detox is the leading end-to-end (E2E) testing framework for React Native, developed by Wix. It controls a real or simulated device — automatically tapping buttons, entering text, swiping, and verifying UI state — exactly as a real user would.

Jest vs Detox: Complementary, Not Competing

AspectJest (Unit/Integration)Detox (E2E)
Runs onNode.js (no device needed)iOS Simulator / Android Emulator / Real device
SpeedMilliseconds per testSeconds to minutes per test
TestsIndividual components, hooks, utilitiesFull user flows across multiple screens
When to writeFor every function and componentFor critical business flows (login, checkout, onboarding)

Configuration

// .detoxrc.js
/** @type {Detox.DetoxConfig} */
module.exports = {
  testRunner: {
    args: { '$0': 'jest', config: 'e2e/jest.config.js' },
    jest: { setupTimeout: 120000 }
  },
  apps: {
    'ios.debug': {
      type: 'ios.app',
      binaryPath: 'ios/build/Build/Products/Debug-iphonesimulator/YourApp.app',
      build: 'xcodebuild -workspace ios/YourApp.xcworkspace -scheme YourApp -configuration Debug -sdk iphonesimulator -derivedDataPath ios/build'
    },
    'android.debug': {
      type: 'android.apk',
      binaryPath: 'android/app/build/outputs/apk/debug/app-debug.apk',
      build: 'cd android && ./gradlew assembleDebug assembleAndroidTest -DtestBuildType=debug',
    }
  },
  devices: {
    simulator: { type: 'ios.simulator', device: { type: 'iPhone 15' } },
    emulator: { type: 'android.emulator', device: { avdName: 'Pixel_6_API_33' } }
  },
  configurations: {
    'ios.sim.debug': { device: 'simulator', app: 'ios.debug' },
    'android.emu.debug': { device: 'emulator', app: 'android.debug' }
  }
};

Writing E2E Tests

// e2e/login.test.ts
describe('Login Flow', () => {
  beforeAll(async () => {
    await device.launchApp({ newInstance: true });
  });

  beforeEach(async () => {
    await device.reloadReactNative();
  });

  it('should complete login and reach home screen', async () => {
    // Verify we're on the login screen
    await expect(element(by.text('Welcome Back'))).toBeVisible();

    // Enter credentials using testID (add testID to your TextInput components)
    await element(by.id('email-input')).typeText('user@example.com');
    await element(by.id('password-input')).typeText('Password123');
    
    // Dismiss keyboard before tapping the button
    await element(by.id('password-input')).tapReturnKey();
    
    // Tap the login button
    await element(by.text('Sign In')).tap();

    // Assert we navigated to the home screen
    await waitFor(element(by.text('Your Dashboard')))
      .toBeVisible()
      .withTimeout(5000); // Wait up to 5s for navigation to complete
  });

  it('should show error for invalid credentials', async () => {
    await element(by.id('email-input')).typeText('wrong@email.com');
    await element(by.id('password-input')).typeText('WrongPassword');
    await element(by.text('Sign In')).tap();

    await expect(element(by.text('Invalid email or password'))).toBeVisible();
  });
});

Common Matchers and Actions

// Matchers — finding elements
by.id('testID')          // by testID prop (most reliable)
by.text('Submit')        // by visible text content
by.type('RCTTextInput')  // by native component type
by.label('Username')     // by accessibility label

// Actions — interacting
await element(...).tap()
await element(...).longPress()
await element(...).typeText('hello')
await element(...).clearText()
await element(...).scroll(200, 'down')
await element(...).swipe('left', 'fast')

// Expectations — assertions
await expect(element(...)).toBeVisible()
await expect(element(...)).toExist()
await expect(element(...)).not.toBeVisible()
await expect(element(...)).toHaveText('Submit')

Running Detox Tests


# Build the app for testing (do this once, or when native code changes)
detox build --configuration ios.sim.debug

# Run E2E tests
detox test --configuration ios.sim.debug

# Run a specific test file
detox test --configuration ios.sim.debug e2e/login.test.ts

# Run on Android emulator
detox test --configuration android.emu.debug

Best Practices

  • Add testID props to all interactive elements in your components — this is the most reliable selector across platform differences.
  • Use waitFor(...).toBeVisible().withTimeout(5000) for any assertion involving navigation or async operations — never assume instant rendering.
  • Reset app state between tests with device.reloadReactNative() or device.launchApp({ newInstance: true }) to ensure test isolation.
  • Run Detox tests in CI on iOS Simulator and Android Emulator — GitHub Actions supports both with the right runner configuration.
