REST API Integration in React Native

Almost every modern mobile app communicates with a backend server — fetching user data, submitting forms, loading content feeds. In React Native, HTTP communication uses the browser-standard fetch API (available globally) or the popular Axios library. Building a robust API layer means handling loading states, errors, caching, and retry logic — patterns that separate professional apps from hobby projects.

The Fetch API Basics

// Basic GET request
const fetchProducts = async (): Promise => {
  const response = await fetch('https://api.yourapp.com/v1/products');
  
  if (!response.ok) {
    throw new Error(`HTTP Error: ${response.status} ${response.statusText}`);
  }
  
  const data = await response.json();
  return data.products;
};

// POST request with JSON body and auth header
const createOrder = async (orderData: CreateOrderDto): Promise => {
  const token = await AsyncStorage.getItem('auth_token');
  
  const response = await fetch('https://api.yourapp.com/v1/orders', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
    },
    body: JSON.stringify(orderData),
  });
  
  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message || 'Failed to create order');
  }
  
  return response.json();
};

Building a Reusable API Client

Copying the base URL and auth headers into every fetch call is error-prone. Build a centralized API client:

// src/api/client.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const BASE_URL = process.env.EXPO_PUBLIC_API_URL ?? 'https://api.yourapp.com';

class ApiClient {
  private async getHeaders(): Promise {
    const token = await AsyncStorage.getItem('auth_token');
    return {
      'Content-Type': 'application/json',
      ...(token ? { Authorization: `Bearer ${token}` } : {}),
    };
  }

  async get(endpoint: string): Promise {
    const response = await fetch(`${BASE_URL}${endpoint}`, {
      headers: await this.getHeaders(),
    });
    return this.handleResponse(response);
  }

  async post(endpoint: string, body: unknown): Promise {
    const response = await fetch(`${BASE_URL}${endpoint}`, {
      method: 'POST',
      headers: await this.getHeaders(),
      body: JSON.stringify(body),
    });
    return this.handleResponse(response);
  }

  private async handleResponse(response: Response): Promise {
    if (response.status === 401) {
      // Token expired — clear it and redirect to login
      await AsyncStorage.removeItem('auth_token');
      throw new Error('UNAUTHORIZED');
    }
    if (!response.ok) {
      const error = await response.json().catch(() => ({}));
      throw new Error(error.message ?? `Error ${response.status}`);
    }
    return response.json();
  }
}

export const api = new ApiClient();

Data Fetching with TanStack Query (React Query)

React Query is the industry standard for managing server state in React Native. It handles loading/error states, caching, background refetching, and deduplication automatically:

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { api } from '../api/client';

// ─── Fetching data ────────────────────────────────────────────────────
const useProducts = (category: string) => {
  return useQuery({
    queryKey: ['products', category],  // Cache key — unique per category
    queryFn: () => api.get(`/products?category=${category}`),
    staleTime: 5 * 60 * 1000,  // Consider data fresh for 5 minutes
    retry: 3,                   // Retry failed requests 3 times
  });
};

// In component:
const { data: products, isLoading, isError, error, refetch } = useProducts('electronics');

// ─── Mutations (POST/PUT/DELETE) ──────────────────────────────────────
const queryClient = useQueryClient();

const addToCart = useMutation({
  mutationFn: (item: CartItem) => api.post('/cart/items', item),
  onSuccess: () => {
    // Invalidate cart cache so it refetches fresh data
    queryClient.invalidateQueries({ queryKey: ['cart'] });
  },
  onError: (error) => {
    Alert.alert('Failed to add item', error.message);
  }
});

File Uploads (Images)

import * as ImagePicker from 'expo-image-picker';

const uploadProfilePhoto = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    quality: 0.7,
    allowsEditing: true,
    aspect: [1, 1]
  });
  
  if (!result.canceled && result.assets[0]) {
    const asset = result.assets[0];
    
    // FormData — not JSON — for file uploads
    const formData = new FormData();
    formData.append('photo', {
      uri: asset.uri,
      type: asset.mimeType ?? 'image/jpeg',
      name: 'profile.jpg'
    } as any);

    const response = await fetch(`${BASE_URL}/users/photo`, {
      method: 'POST',
      headers: { Authorization: `Bearer ${token}` },
      // Note: No 'Content-Type' header — fetch sets it automatically with correct boundary
      body: formData
    });
  }
};

Best Practices

  • Store your API base URL in environment variables (EXPO_PUBLIC_API_URL) — never hardcode in source code.
  • Always handle the 401 Unauthorized case at the API client level to redirect users to the login screen when tokens expire.
  • Use React Query for all server state — it eliminates the need to manually manage loading/error state with useState for every API call.
  • Never block the UI thread with synchronous storage reads — use async/await for all AsyncStorage calls.
