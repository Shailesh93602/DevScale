GraphQL in React Native

REST APIs return fixed data structures — a GET /users/:id endpoint returns all user fields, whether you need them or not. GraphQL solves this by letting the client specify exactly what data it needs in a single request. For mobile apps running on potentially slow connections, this "ask for exactly what you need" approach reduces payload sizes, combines multiple resource requests into one, and dramatically simplifies data fetching logic.

GraphQL vs REST for Mobile

AspectRESTGraphQL
Data fetchingMultiple endpoints, over-fetch or under-fetchSingle endpoint, ask for exactly what you need
Multiple resourcesN requests for N resource typesOne request for all related resources
Versioningv1, v2, v3 endpointsSchema evolution with deprecations
BandwidthFixed response shape (over-fetches on mobile)Minimal — only requested fields are returned

Apollo Client Setup

// src/api/apolloClient.ts
import { ApolloClient, InMemoryCache, createHttpLink, ApolloLink } from '@apollo/client';
import { setContext } from '@apollo/client/link/context';
import AsyncStorage from '@react-native-async-storage/async-storage';

const httpLink = createHttpLink({
  uri: 'https://api.yourapp.com/graphql',
});

// Attach auth token to every request
const authLink = setContext(async (_, { headers }) => {
  const token = await AsyncStorage.getItem('auth_token');
  return {
    headers: {
      ...headers,
      authorization: token ? `Bearer ${token}` : '',
    },
  };
});

export const client = new ApolloClient({
  link: authLink.concat(httpLink),
  cache: new InMemoryCache(),
});

Querying Data with useQuery

import { View, Text, FlatList, ActivityIndicator } from 'react-native';

// Define the query — request ONLY the fields you need
const GET_USER_PROFILE = gql`
  query GetUserProfile($userId: ID!) {
    user(id: $userId) {
      id
      firstName
      lastName
      avatar
      # Friends, orders, etc. NOT requested — Apollo sends nothing extra
      recentOrders(limit: 5) {
        id
        total
        status
        createdAt
      }
    }
  }
`;

Best Practices

  • Co-locate your GraphQL queries with the components that use them — this makes it immediately clear what data each component needs.
  • Use fetchPolicy: 'cache-and-network' for screens that need fresh data but should also render immediately from cache.
  • Use optimistic responses for mutations that update simple fields (name, bio, avatar) — it makes the app feel instant.
  • Use GraphQL Code Generator to auto-generate TypeScript types from your schema — eliminates hand-written types for every query.
