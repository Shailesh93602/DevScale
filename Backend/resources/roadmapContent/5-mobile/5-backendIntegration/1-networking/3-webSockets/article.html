WebSockets in React Native

HTTP is a request-response protocol — the client asks, the server answers, connection closes. This is fine for loading a product list, but it fundamentally cannot support real-time features like live chat, collaborative editing, live sports scores, or real-time trading dashboards. For these cases, you need WebSockets — a persistent, bidirectional TCP connection where both client and server can send messages to each other at any time.

WebSocket vs HTTP

PropertyHTTPWebSocket
ConnectionNew connection per requestPersistent connection
DirectionClient → Server (request only)Bidirectional (both directions)
LatencyHigh (handshake per request)Very low (connection already open)
Use caseFetching data, CRUD operationsLive chat, notifications, real-time data
OverheadFull HTTP headers per requestSmall frame headers (2 bytes minimum)

Using the Native WebSocket API

React Native includes WebSocket support natively — no library needed for basic usage:

import React, { useEffect, useRef, useState, useCallback } from 'react';
import { View, Text, TextInput, FlatList, Pressable } from 'react-native';

interface Message {
  id: string;
  text: string;
  sender: string;
  timestamp: number;
}

const ChatRoom = ({ roomId, userId }: { roomId: string; userId: string }) => {
  const [messages, setMessages] = useState<Message[]>([]);
  const [inputText, setInputText] = useState('');
  const [isConnected, setIsConnected] = useState(false);
  const ws = useRef<WebSocket | null>(null);

  const connect = useCallback(() => {
    const socket = new WebSocket(
      `wss://api.yourapp.com/ws/chat/${roomId}?userId=${userId}`
    );

    socket.onopen = () => {
      console.log('WebSocket connected');
      setIsConnected(true);
      socket.send(JSON.stringify({ type: 'join', roomId, userId }));
    };

    socket.onmessage = (event) => {
      const data = JSON.parse(event.data);

      switch (data.type) {
        case 'message':
          setMessages(prev => [...prev, data.payload]);
          break;
        case 'user_joined':
          console.log(`${data.username} joined the room`);
          break;
      }
    };

    socket.onerror = (error) => {
      console.error('WebSocket error:', error);
    };

    socket.onclose = (event) => {
      setIsConnected(false);
      console.log(`Disconnected: ${event.code} ${event.reason}`);
      // Auto-reconnect after 3 seconds (unless intentional close)
      if (event.code !== 1000) {
        setTimeout(connect, 3000);
      }
    };

    ws.current = socket;
  }, [roomId, userId]);

  useEffect(() => {
    connect();
    return () => {
      ws.current?.close(1000, 'Component unmounted');
    };
  }, [connect]);

  const sendMessage = useCallback(() => {
    if (!inputText.trim() || !ws.current || ws.current.readyState !== WebSocket.OPEN) return;

    ws.current.send(JSON.stringify({
      type: 'message',
      payload: { text: inputText.trim(), sender: userId }
    }));
    setInputText('');
  }, [inputText, userId]);

  return (
    <View style={{ flex: 1 }}>
      <Text>{isConnected ? '🟢 Connected' : '🔴 Disconnected'}</Text>
      <FlatList
        data={messages}
        keyExtractor={item => item.id}
        renderItem={({ item }) => (
          <View>
            <Text style={{ fontWeight: 'bold' }}>{item.sender}</Text>
            <Text>{item.text}</Text>
          </View>
        )}
      />
      <View style={{ flexDirection: 'row' }}>
        <TextInput value={inputText} onChangeText={setInputText} style={{ flex: 1 }} />
        <Pressable onPress={sendMessage}><Text>Send</Text></Pressable>
      </View>
    </View>
  );
};

Using Socket.io for Advanced Features

The native WebSocket API is low-level. Socket.io adds automatic reconnection, rooms, namespaces, acknowledgements (guaranteed delivery), and fallback to HTTP long-polling — making it much simpler for production use:

import { io, Socket } from 'socket.io-client';

const socket: Socket = io('https://api.yourapp.com', {
  auth: { token: authToken },
  transports: ['websocket'],  // Skip HTTP polling, use WebSocket directly
  reconnection: true,
  reconnectionAttempts: 5,
  reconnectionDelay: 2000,
});

// Listen for events
socket.on('connect', () => console.log('Connected:', socket.id));
socket.on('disconnect', (reason) => console.warn('Disconnected:', reason));
socket.on('new_message', (message: Message) => {
  setMessages(prev => [...prev, message]);
});

// Emit with acknowledgement (guaranteed delivery confirmation)
socket.emit('send_message', { text: 'Hello!', room: 'general' }, (ack) => {
  if (ack.success) console.log('Message delivered');
});

// Join a room (server-side filtering — only receive messages for this room)
socket.emit('join_room', 'general');

// Cleanup on unmount
useEffect(() => {
  return () => { socket.disconnect(); };
}, []);

Handling Disconnections and App State

import { AppState, AppStateStatus } from 'react-native';

useEffect(() => {
  const subscription = AppState.addEventListener('change', (nextState: AppStateStatus) => {
    if (nextState === 'active') {
      // App came to foreground — reconnect if disconnected
      if (!socket.connected) socket.connect();
    } else if (nextState === 'background') {
      // App went to background — disconnect to save battery
      socket.disconnect();
    }
  });
  return () => subscription.remove();
}, []);

Best Practices

  • Always clean up WebSocket connections in the useEffect cleanup function — failing to do so creates memory leaks and ghost connections.
  • Use ws.readyState === WebSocket.OPEN before calling ws.send() — sending on a closing/closed socket causes an error.
  • Implement exponential backoff for reconnection delays instead of a fixed interval — prevents thundering herd if your server restarts.
  • Disconnect WebSockets when the app goes to the background and reconnect on foreground to preserve battery life.
