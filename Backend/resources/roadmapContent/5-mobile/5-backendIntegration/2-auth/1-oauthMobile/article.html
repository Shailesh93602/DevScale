OAuth 2.0 & Social Login in React Native

Most users dislike creating yet another username and password. OAuth 2.0 social login — "Sign in with Google/Apple/GitHub" — reduces signup friction dramatically and eliminates the need for your app to store and hash passwords at all. OAuth 2.0 is the industry standard authorization framework used by virtually every major platform. Understanding how it works prevents security mistakes that commonly appear in mobile implementations.

The OAuth 2.0 Authorization Code Flow

For mobile apps, the secure flow is the Authorization Code Flow with PKCE (Proof Key for Code Exchange). PKCE prevents a class of attacks where a malicious app intercepts the authorization code:

  1. App generates a random code_verifier and computes code_challenge = SHA256(code_verifier).
  2. App opens the browser with the provider's auth URL, including the code_challenge.
  3. User authenticates in the browser (Google/Apple UI — your app never sees the password).
  4. Browser redirects back to your app via a deep link with an authorization code.
  5. Your app exchanges the code + code_verifier for access/refresh tokens (server-to-server).

Implementation with Expo Auth Session

import * as WebBrowser from 'expo-web-browser';
import * as Google from 'expo-auth-session/providers/google';
import { makeRedirectUri } from 'expo-auth-session';
import AsyncStorage from '@react-native-async-storage/async-storage';

// Required — enables Auth Session to handle the redirect URI
WebBrowser.maybeCompleteAuthSession();

export const useGoogleAuth = () => {
  const [request, response, promptAsync] = Google.useAuthRequest({
    clientId: process.env.EXPO_PUBLIC_GOOGLE_CLIENT_ID,
    iosClientId: process.env.EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID,
    androidClientId: process.env.EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID,
    // Scopes determine what user data you will have access to
    scopes: ['profile', 'email'],
  });

  React.useEffect(() => {
    handleGoogleResponse();
  }, [response]);

  const handleGoogleResponse = async () => {
    if (response?.type !== 'success') return;

    const { authentication } = response;

    // Don't use the Google access token directly in your app!
    // Send it to YOUR backend to verify and issue your own JWT
    try {
      const backendResponse = await fetch('https://api.yourapp.com/auth/google', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ googleAccessToken: authentication?.accessToken })
      });

      const { accessToken, user } = await backendResponse.json();

      // Store your backend's JWT securely
      await AsyncStorage.setItem('auth_token', accessToken);
    } catch (error) {
      console.error('Auth failed:', error);
    }
  };

  return { signInWithGoogle: () => promptAsync(), request };
};

Sign in with Apple (Required for iOS)

Apple requires that any iOS app offering third-party social login must also offer Sign in with Apple. Failure to include it results in App Store rejection.

import * as AppleAuthentication from 'expo-apple-authentication';

const SignInWithAppleButton = () => {
  const handleAppleSignIn = async () => {
    try {
      const credential = await AppleAuthentication.signInAsync({
        requestedScopes: [
          AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
          AppleAuthentication.AppleAuthenticationScope.EMAIL,
        ],
      });

      // credential.identityToken is a JWT signed by Apple
      // Send this to your backend to verify and issue your own token
      const response = await fetch('https://api.yourapp.com/auth/apple', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ identityToken: credential.identityToken })
      });

      const { accessToken } = await response.json();
      await AsyncStorage.setItem('auth_token', accessToken);
    } catch (error: any) {
      if (error.code === 'ERR_CANCELED') return; // User dismissed
      console.error('Apple sign in failed:', error);
    }
  };

  return (
    <AppleAuthentication.AppleAuthenticationButton
      buttonType={AppleAuthentication.AppleAuthenticationButtonType.SIGN_IN}
      buttonStyle={AppleAuthentication.AppleAuthenticationButtonStyle.BLACK}
      cornerRadius={5}
      style={{ width: '100%', height: 44 }}
      onPress={handleAppleSignIn}
    />
  );
};

Storing Tokens Securely

import * as SecureStore from 'expo-secure-store';

// ❌ WRONG: AsyncStorage stores in plaintext
await AsyncStorage.setItem('auth_token', token);

// ✅ CORRECT: SecureStore uses iOS Keychain / Android Keystore (hardware-backed encryption)
await SecureStore.setItemAsync('auth_token', token);
const token = await SecureStore.getItemAsync('auth_token');
await SecureStore.deleteItemAsync('auth_token'); // On logout

Best Practices

  • Never trust the OAuth token client-side for authorization decisions. Always send it to your backend to verify and issue your own short-lived JWT.
  • Store auth tokens in expo-secure-store (iOS Keychain / Android Keystore), not plain AsyncStorage.
  • If you offer Google login on iOS, you MUST also include Sign in with Apple — App Store will reject your app otherwise.
  • Implement token refresh — access tokens typically expire in 1 hour. Use a refresh token to silently obtain new access tokens without forcing re-login.
