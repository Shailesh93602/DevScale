CSS for Accessibility: Styling with Inclusivity

CSS is not just for looks; it's a critical tool for making your website accessible to everyone, including users who have visual, hearing, or motor impairments.

1. Color Contrast

Always ensure that the contrast between your text and background is high enough to be readable. The **WCAG guidelines** recommend a minimum contrast ratio of **4.5:1** for normal text.

2. Focus Indicators

Keyboard users rely on "focus" styles to see where they are on the page. **Never remove the default focus outline** (outline: none;) unless you are replacing it with a clearly visible custom focus style.

3. Visually Hidden Content

Sometimes you need to add text for screen reader users that shouldn't be visible on the screen. Use a special .sr-only class instead of display: none; (which hides content from screen readers too).

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  border: 0;
}

4. Respecting User Preferences

Modern CSS allows you to detect the user's system-level accessibility settings, such as **reduced motion** (for vestibular disorders) and **forced-colors** (for high contrast mode in Windows).

@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

💡 Pro Tip: Use the **Accessibility** panel in your browser's DevTools to automatically check for color contrast violations and other common accessibility issues as you develop your styles.
