CSS Display: Defining Element Behavior

The display property is one of the most fundamental in CSS. It determines how an element is treated by the browser layout engine. Every element has a default display type (usually block or inline), but you can change this to achieve different layouts.

Common Display Values

  • block: The element starts on a new line and takes up the full width available. (e.g., <div>, <h1>).
  • inline: The element does not start on a new line and only takes up as much width as necessary. You cannot set a width or height on inline elements. (e.g., <span>, <a>).
  • inline-block: Behaves like an inline element (doesn't start a new line) but allows you to set a specific width and height.
  • none: The element is completely removed from the page layout and is not visible. (Unlike visibility: hidden, which leaves an empty space).

Modern Layout Values

  • flex: Turns the element into a "flex container," enabling the Flexbox layout model for its children.
  • grid: Turns the element into a "grid container," enabling the CSS Grid layout model for its children.

Display: Contents

A newer value, contents, makes the element itself "disappear" from the layout tree, while its children behave as if they were direct children of the parent element. This is useful for semantic wrappings that shouldn't affect layout.

💡 Pro Tip: If you find yourself struggling to center an element or make things sit side-by-side using margins and floats, you should probably switch to display: flex. It's the modern way to handle almost all alignment issues.
