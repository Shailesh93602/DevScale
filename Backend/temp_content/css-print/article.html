CSS for Print: Designing for the Physical World

Sometimes users want to print a physical copy of your web page. Without specific "Print CSS," the result can be a mess of unnecessary sidebars, dark backgrounds, and broken layouts. Designing for print is an essential but often overlooked part of web development.

The @media print Query

To write styles that only apply when the user prints the page, you use a specific media query.

@media print {
  /* Print-specific styles here */
}

Hiding Non-Essential Content

When printing, you should hide elements that have no value on paper, like navigation bars, sidebars, ads, and interactive buttons.

@media print {
  .nav, .sidebar, .ads, .btn-submit { display: none; }
}

Page Break Control

You can control where the printer starts a new page using properties like break-before, break-after, and break-inside: avoid;. This prevents things like a heading appearing at the bottom of one page while its content starts on the next.

Styling Links for Print

On paper, links are useless because they aren't clickable. A common trick is to use CSS to display the actual URL next to the link text so users can still find the information later.

@media print {
  a::after { content: " (" attr(href) ")"; }
}

Standardizing for Paper

  • Change your text color to black (to save ink) and your background to white.
  • Use standard page sizes (like 8.5in x 11in) or let the browser handle it.
  • Keep margins generous so no text is cut off by the printer's edge.

💡 Pro Tip: You can test your print styles without wasting paper! In Chrome DevTools, open the **Rendering** tab and find the "Emulate CSS media type" option, then select **print**.
