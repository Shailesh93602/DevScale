CSS Selectors and Specificity: The Hierarchy of Design

CSS Selectors tell the browser which HTML elements to style. If multiple selectors target the same element, the browser uses **Specificity** to determine which styling rule "wins." Understanding this concept is the key to solving "stubborn" CSS issues.

Basic Selectors

  • Universal Selector (*): Matches everything on the page.
  • Type Selector (h1, p): Matches all elements of a specific tag type.
  • Class Selector (.my-class): Matches elements with a specific class.
  • ID Selector (#my-id): Matches a single element with a specific ID.

Attribute and Psuedo Selectors

  • Attribute ([type="text"]): Matches based on an HTML attribute value.
  • Psuedo-class (:hover, :nth-child): Styles an element in a specific state.
  • Psuedo-element (::before, ::after): Styles a specific part of an element.

Specificity Calculation (the "Points" System)

The browser gives each selector a "score" to decide who wins. The higher the score, the more specific (and powerful) the rule.

  1. ID (1,0,0): Strongest. 100 points.
  2. Class, Attribute, Psuedo-class (0,1,0): Moderate. 10 points.
  3. Type (0,0,1): Weakest. 1 point.

Example: #header .nav p has a score of 111. .header p has a score of 11. The first rule wins.

Inheritance

Some properties, like font-family and color, are "inherited." They are passed down from a parent element to its children automatically.

Important Rule (!important)

Adding !important to a declaration will force it to "win" over almost any other specificity. **Warning:** Use this sparingly, as it makes debugging very difficult later!

💡 Pro Tip: If your CSS isn't working, don't just reach for !important. First, check the **Styles** panel in your browser's DevTools to see which rule is overwriting yours and increase your selector's specificity intelligently instead!
