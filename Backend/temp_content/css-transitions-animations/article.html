CSS Transitions and Animations: Making Your Site Move

Motion is a powerful way to draw attention and provide feedback to your users. CSS provides two main ways to create motion: Transitions (simple changes from state A to state B) and Animations (complex, multi-step movements).

1. CSS Transitions

Transitions allow you to change property values smoothly over a given duration. They are usually triggered by a change in state, like a mouse hover.

.button {
  background-color: blue;
  transition: background-color 0.3s ease-in-out;
}
.button:hover {
  background-color: darkblue;
}

2. CSS Animations (@keyframes)

Animations allow you to create complex movement by defining "keyframes"—specific points in the animation where properties change.

@keyframes fade-in {
  from { opacity: 0; }
  to { opacity: 1; }
}
.hero-title {
  animation: fade-in 2s ease;
}

Easing Functions

The "timing" of your animation (how it accelerates and decelerates) is controlled by **Easing Functions**. ease-in starts slow, ease-out ends slow, and linear stays the same speed.

Performance: Use Transforms and Opacity

For the smoothest animations, try to animate only transform (scale, rotate, translate) and opacity. These properties are handled by the computer's GPU, making them much faster than animating things like margin or width.

💡 Pro Tip: Less is more! While it's tempting to animate everything, too much motion can be distracting or even nauseating for some users. Use subtle animations to enhance the experience, not to dominate it.
