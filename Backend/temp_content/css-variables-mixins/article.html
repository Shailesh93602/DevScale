CSS Variables and Mixins: Advanced Style Logic

In modern CSS (especially when using preprocessors like SASS), the combination of **Variables** and **Mixins** allows you to build a powerful and reusable "Design System." This makes your codebase much cleaner and easier to update over time.

CSS Variables (The "Properties")

Use variables to store your atomic design tokens—things like your primary color, main border radius, and standard spacing unit.

:root {
  --btn-radius: 8px;
  --spacing-main: 1rem;
}

SASS Mixins (The "Behaviors")

A Mixin is a block of CSS that can be "included" anywhere. It's especially powerful because you can pass **Arguments** to it, similar to a JavaScript function.

@mixin button-style($bg-color) {
  background-color: $bg-color;
  border-radius: var(--btn-radius);
  padding: var(--spacing-main);
  border: none;
}
.primary-btn { @include button-style(blue); }
.danger-btn { @include button-style(red); }

Why Combine Them?

By using native CSS variables inside your mixins, you get the best of both worlds:

  • Mixins: Handle the logic of which properties are needed.
  • Variables: Allow for "live" updates and easy theming at runtime.

Practical Use Cases

  • Centered Flexbox: A mixin that applies all the alignment rules for a flex container.
  • Media Query Snippets: Mixins that make writing desktop-specific CSS easier.
  • Custom Scrollbars: A reusable block of rules to style scrollbars consistently.

💡 Pro Tip: If you're building a design system, create a separate _mixins.scss or _variables.scss file. This keeps your main stylesheets focused and makes it easy to share your core styles across multiple projects.
