Forms and Input: Interacting with Users

HTML forms are the primary way for users to send data to a web server. Whether it's a login page, a search bar, or a complex registration form, understanding how to structure inputs correctly is essential for any web developer.

The <form> Element

The <form> element acts as a container for all interactive controls. It typically has two important attributes:

  • action: The URL where the form data should be sent.
  • method: The HTTP method to use (usually GET or POST).
<form action="/submit" method="POST">
    <!-- Inputs go here -->
</form>

Essential Input Types

The <input> element is versatile and changes its behavior based on the type attribute:

  • type="text": A single-line text field.
  • type="password": A text field that masks the characters.
  • type="email": Validates that the input is a correctly formatted email address.
  • type="checkbox": Allows users to select one or more options.
  • type="radio": Allows users to select only ONE option from a group.
  • type="submit": A button that sends the form data to the server.

Labels and Accessibility

Every input should have an associated <label>. This is not only good for visual clarity but also critical for screen readers. You link a label to an input using the for attribute (on the label) and the id attribute (on the input).

<label for="username">Username:</label>
<input type="text" id="username" name="user_name">

Textareas and Select Menus

For multi-line text, use <textarea>. For dropdown menus, use the <select> and <option> tags.

<label for="country">Country:</label>
<select id="country" name="user_country">
    <option value="usa">USA</option>
    <option value="uk">UK</option>
</select>

💡 Pro Tip: Always use the name attribute for your inputs. This is the key that the server uses to identify the data sent in the request.
