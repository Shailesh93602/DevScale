HTML Accessibility (A11y): Designing for Everyone

Web accessibility (A11y) means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. As a developer, building an accessible web is both a moral and often a legal requirement. HTML is the first and most important layer of accessibility.

The Importance of Semantic HTML

The foundation of an accessible page is using tags for their intended purpose. Screen readers (software that reads page content aloud) rely on semantic tags to provide a "map" of the page.

  • Use <nav> for navigation links.
  • Use <h1> through <h6> in a logical, nested order.
  • Use <button> for actions and <a> for navigation between pages.

Alternative Text for Images (alt)

The alt attribute is the single most important accessibility feature for images. If an image fails to load or if a user is using a screen reader, the alt text provides the necessary context.

<img src="chart.png" alt="Bar chart showing a 20% increase in web traffic over the last quarter">

Labels for Form Inputs

Never use plain text as a label for an input. Always use the <label> element and link it using the for and id attributes. This allows users to click the text to focus on the input and ensures screen readers announce the correct label.

ARIA Roles and Attributes

When native HTML tags aren't enough (e.g., for complex widgets like tabs or modal windows), use ARIA (Accessible Rich Internet Applications) landmark roles and states like aria-label, aria-hidden, and aria-expanded to provide extra information to assistive technologies.

💡 Pro Tip: Use the Lighthouse tool in Chrome DevTools to run an accessibility audit on your pages. It will give you a score and specific tips for improvement.
