HTML Comments: Leaving Roadmap for Yourself and Others

Comments are lines of text in your code that are completely ignored by the browser. They are used to explain parts of your code, leave reminders, or temporarily "disable" a section of HTML during debugging.

Syntax

An HTML comment starts with <!-- and ends with -->. Anything placed between these markers will not be displayed on the web page.

<!-- This is a single-line comment -->

<!-- 
  This is a
  multi-line
  comment
-->

Best Practices for Comments

  • Organize your code: Use comments to mark the start and end of large sections (e.g., <!-- Start of Sidebar -->).
  • Keep them concise: Comments should explain why something is done, not necessarily what (the code should ideally be readable enough to explain what).
  • Remove debugging comments: Before launching your site, remove any comments that were only meant for your own debugging process.

Security Warning

While comments are not rendered on the page, they are still sent to the user's browser. Anyone who "Views Page Source" can read your comments. Never include sensitive information like passwords, API keys, or private internal notes in your HTML comments.

Commenting Out Code

If you want to hide a piece of HTML without deleting it, you can wrap it in comment tags. This is very useful when testing different layouts.

<!-- <button>Click Me!</button> -->

💡 Pro Tip: Most code editors have a keyboard shortcut (like Cmd + / or Ctrl + /) that will automatically wrap the selected lines in HTML comment tags for you.
