HTML and CSS Integration: Three Ways to Style

To apply CSS rules to your HTML elements, you must first "integrate" the CSS into your document. There are three standard ways to do this, ranging from the least efficient (inline) to the most powerful (external).

1. Inline Styles

Inline styles are applied directly to a specific HTML element using the style attribute. This should generally be avoided because it mixes design with content and makes the code difficult to maintain.

<h2 style="color: red; font-size: 20px;">Styled Title</h2>

2. Internal Style Sheets

Internal styles are placed inside a <style> tag within the <head> of your HTML document. This is useful for single-page websites or for applying styles that only affect one specific page.

<head>
  <style>
    body { background-color: #f0f0f0; }
    h1 { color: blue; }
  </style>
</head>

3. External Style Sheets

External styles are stored in a separate file (usually with a .css extension) and linked to the HTML document using the <link> tag. This is the best practice for professional websites because it allows you to change the look of an entire site by editing just one file.

<head>
  <link rel="stylesheet" href="styles.css">
</head>

Performance and Maintenance

Using external style sheets is the most performant method because browsers can cache the CSS file separately from the HTML. It also provides a cleaner "separation of concerns," making your project much easier to scale and debug.

💡 Pro Tip: If you use all three methods, CSS follows a hierarchy. Inline styles have the highest priority, followed by internal styles, and finally external styles (unless a rule has the !important flag).
