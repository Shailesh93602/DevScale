HTML Email Design: Coding for the Inbox

Coding an HTML email is very different from coding a modern web page. Because email clients (like Outlook, Gmail, and Apple Mail) have wildly different levels of CSS support, you often have to use "retro" coding techniques to ensure your message looks good everywhere.

The Grid Shortcut: HTML Tables

While tables are discouraged for web layouts, they are the gold standard for HTML email. They provide the most consistent structure across different email clients, especially older versions of Microsoft Outlook.

Inline Everything

Many email clients strip out <style> tags in the <head>. To ensure your styles are applied, you must use inline CSS for every element. There are many "CSS Inliner" tools available to automate this process.

<td style="font-family: Arial, sans-serif; font-size: 16px; color: #333333;">
  Hello, Subscriber!
</td>

Images and Alt Text

Many email clients disable images by default. To ensure your message is still understandable, always provide descriptive alt text and set a background color for your image containers.

The "Ghost Table" Technique

For advanced responsive layouts in Outlook, developers use "Microsoft Conditional Comments" to create "ghost tables" that only the desktop version of Outlook sees.

<!--[if mso]>
<table role="presentation" width="600">
<![endif]-->

Email Accessibility

  • Add role="presentation" to layout tables so screen readers skip them.
  • Keep your font size at 14px or higher for readability.
  • Use high-contrast colors for text and buttons.

💡 Pro Tip: Always test your emails using a service like Litmus or Email on Acid. They provide screenshots of how your email will look in dozens of different real-world inbox environments.
