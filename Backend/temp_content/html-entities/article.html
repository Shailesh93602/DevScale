HTML Entities: Handling Reserved and Special Characters

Some characters in HTML have special meanings. For example, the less-than symbol (<) tells the browser that a new tag is beginning. If you want to display these characters as actual text on your page, you must use HTML Entities.

Common Reserved Characters

These characters must always be replaced by entities to avoid confusing the browser's HTML parser:

  • &lt; represents < (less than)
  • &gt; represents > (greater than)
  • &amp; represents & (ampersand)
  • &quot; represents " (double quote)
  • &apos; represents ' (single quote)

The Non-Breaking Space (&nbsp;)

The &nbsp; entity is used when you want a space between two words but do not want the browser to break the line between them. It is also used to add multiple spaces in a row, as browsers normally collapse multiple spaces into one.

Symbols and Currency

HTML entities are also used for symbols that aren't on a standard keyboard:

  • &copy; represents © (copyright)
  • &reg; represents ® (registered trademark)
  • &euro; represents € (euro)
  • &trade; represents ™ (trademark)

Mathematical Symbols

  • &plusmn; represents ± (plus-minus)
  • &times; represents × (multiplication)
  • &divide; represents ÷ (division)

💡 Pro Tip: Entities always begin with an ampersand (&) and end with a semicolon (;). While you can use numerical codes (e.g., &#169;), entity names are much easier to remember and read.
