HTML and RESTful APIs: Connecting Your Front-End and Back-End

In a modern web application, the HTML is just the user interface. The actual data (like user profiles or product lists) is usually stored on a database and retrieved through a RESTful API (REpresentational State Transfer).

How HTML and APIs Communicate

The communication happens using standard HTTP methods. Your HTML sends a request, and the API sends back data—usually in **JSON** (JavaScript Object Notation) format.

Common HTTP Methods

  • GET: Used by your HTML to retrieve data (e.g., getting a list of products).
  • POST: Used to send new data (e.g., submitting a registration form).
  • PUT / PATCH: Used to update existing data (e.g., editing a profile).
  • DELETE: Used to remove data from the database.

Traditional HTML Forms vs. API Calls

Traditional HTML forms (using <form action="/submit" method="POST">) refresh the page upon submission. In modern apps, we use JavaScript's fetch() API to send the data in the background, keeping the user on the same page.

Data Attributes as API Hooks

You can store API-related information (like an ID) directly in your HTML using data-* attributes. This makes it easy for your JavaScript to know which resource an element represents.

<button data-user-id="123" onclick="deleteUser(this)">Delete</button>

Security: CORS and Auth Tokens

To prevent malicious cross-site requests, browsers use **CORS** (Cross-Origin Resource Sharing). Your API must specifically allow your website's domain to access its data. Additionally, most APIs require an **Authentication Token** (like a JWT) to verify who the user is.

💡 Pro Tip: Use the browser's Network panel in DevTools to see exactly what data is being sent to and received from your API. It's the best way to debug a broken connection.
