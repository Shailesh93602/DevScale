HTML Scripting: Adding Interactivity

While HTML provides the structure of a page, scripting (primarily via JavaScript) provides the behavior. The <script> element is used to either embed executable code directly or to reference an external script file.

The <script> Tag

You can include JavaScript directly inside your HTML file using the <script> tag:

<script>
  console.log("Hello from the console!");
</script>

External Scripts

For better organization and cacheability, it's best practice to keep your JavaScript in separate .js files and link them using the src attribute.

<script src="script.js"></script>

Async and Defer

Browsers normally stop rendering the HTML page when they encounter a script tag, which can lead to slow load times. HTML5 introduced two attributes to change this behavior:

  • async: The script is downloaded in the background and executed as soon as it's available, without blocking the HTML parser. (Best for independent scripts like analytics).
  • defer: The script is downloaded in the background but only executed after the entire HTML document has been parsed. (Best for scripts that rely on the DOM).

The <noscript> Element

This tag defines an alternate content for users who have disabled scripting in their browser or have a browser that doesn't support it.

<noscript>
  <p>This site requires JavaScript to function correctly. Please enable it in your browser settings.</p>
</noscript>

💡 Pro Tip: Unless you have a specific reason otherwise, always place your script tags just before the closing </body> tag or use the defer attribute to ensure the page renders quickly.
