JavaScript Control Structures: Making Decisions

Control structures allow you to control the "flow" of your program. Instead of running every line in order, you can use **Conditionals** to skip certain parts and run them only when needed (e.g., IF the user's password is correct, let them in).

1. The if...else Statement

The most basic way to make a decision. The browser checks the condition in parentheses. If it's true, it runs the first block of code. If not, it skips it (or runs the else block if provided).

if (age >= 18) {
  console.log("You can vote!");
} else {
  console.log("You are too young.");
}

2. The switch Statement

A cleaner way to handle many different conditions for a single value. It's often used for things like handling different levels in a game or different roles for a user.

switch (day) {
  case 'Monday': console.log("Start work!"); break;
  case 'Friday': console.log("Weekend!"); break;
  default: console.log("Just a regular day.");
}

3. Comparisons inside Conditions

Conditions always result in a **Boolean (true or false)**. You can use >, <, ===, and !== (not equal to) to create complex logic.

4. Values that are "Truthy" or "Falsy"

In JavaScript, every value is either **Truthy** or **Falsy**. A value like an empty string ("") or the number zero (0) is considered false, while a string with text or a non-zero number is considered true.

💡 Pro Tip: If you find yourself writing a huge if...else if...else chain, consider switching to a switch statement or a "Lookup Table" (using an Object) to make your code much easier to read and maintain.
