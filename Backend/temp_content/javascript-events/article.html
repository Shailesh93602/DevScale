JavaScript Events: Responding to User Actions

Events are "things" that happen to elements on your web page (e.g., a button is clicked, an image finishes loading, or the user scrolls down). **Event Listeners** are the way you tell JavaScript to "listen" for these actions and do something in response.

1. Common Events

  • click: The user clicks (or taps) an element (e.g., submitting a form).
  • submit: A <form> is being sent (this is the best place to validate user data!).
  • keydown: The user presses a key on their keyboard (like pressing Enter to search).
  • mouseover: The user's mouse moves over an element.
  • scroll: The user scrolls the page (often used for "back to top" buttons).

2. addEventListener() (The Modern Standard)

The best way to handle events. It allows you to add multiple listeners to the same element without overwriting them.

const btn = document.querySelector("#myBtn");
btn.addEventListener("click", () => {
  console.log("Button Clicked!");
});

3. The "Event" Object

When an event occurs, JavaScript automatically provides an **Event Object (e)**. This object contains useful information (like which key was pressed or exactly where the mouse clicked).

4. preventDefault(): Taking Control

Some events have a default behavior—for example, a link will try to navigate to a new page. You can use e.preventDefault() to stop this behavior and handle the action yourself with JavaScript.

Event Bubbling

Events in JavaScript don't just happen on one element; they "bubble up" from the child to the parent and all the way to the top of the page. This is a powerful concept for handling events on many elements at once (using **Event Delegation**).

💡 Pro Tip: Use **Event Delegation**! Instead of adding 100 listeners to 100 different list items, add ONE listener to their common parent. It's much faster and easier to manage as your project grows.
