JavaScript Functions: Rebuilding and Reusing Code

Functions are the building blocks of JavaScript applications. Instead of writing the same code over and over, you write a **Function** once and "call" it as many times as you need, often with different inputs (parameters).

1. Defining a Function

To create a function, you use the function keyword, give it a name, and put your code inside curly braces {}.

function sayHello() {
  console.log("Hello, EduScale!");
}
sayHello(); // This "calls" the function.

2. Using Parameters and Arguments

Parameters allow your function to accept "inputs" and work differently based on them. This makes your function much more flexible.

function greetUser(name) {
  console.log("Hello, " + name);
}
greetUser("Alice"); // Hello, Alice

3. The return Keyword

If you want a function to "give back" a value (like the result of a calculation), you use the return keyword. Once a function returns, it stops running immediately.

function add(a, b) {
  return a + b;
}
let result = add(10, 20); // result is now 30

4. Arrow Functions (Modern)

The most popular way to write functions in modern JavaScript! It provides a cleaner, shorter syntax (using =>) and has slightly different rules for "scoping" (handling the this keyword).

const multiply = (a, b) => a * b;

Anonymous and Callback Functions

Functions that don't have a name are called **Anonymous**. You can actually pass one function *into* another function as a parameter (this is called a **Callback**), which is the key to handling events and asynchronous data.

💡 Pro Tip: Keep your functions small and focused on **just one task**. This makes your code much easier to test, debug, and reuse in different parts of your project.
