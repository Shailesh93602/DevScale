JavaScript Numbers: Handling Math and Logic

In JavaScript, all numbers (whether they are simple integers like 5 or complex decimals like 3.14) belong to a single "Number" data type. Understanding how to manipulate these values is essential for building everything from simple counters to complex financial applications.

1. Integers vs. Decimals (Floats)

JavaScript doesn't make a distinction between "integers" and "floating-point numbers" in its basic type system. They are all treated as 64-bit double-precision values.

2. Built-in Number Methods

  • toString(): Converts a number into a string (e.g., for displaying it).
  • toFixed(n): Rounds a number to a specific number of decimal places. Essential for currency!
  • Number.isInteger(): Checks if a value is a whole number.

3. The NaN (Not-a-Number) Special Value

This is a common "bug" you'll find when you try to perform math on a non-numerical value (like trying to multiply "Hello" by 5). Always check if a value is a valid number before doing calculations!

4. Converting Strings to Numbers

If you get a value from a user input (which is always a string), you must convert it to a number before doing math.

  • Number(): Converts the entire thing.
  • parseInt(): Converts the beginning of the string into a whole number (e.g., "10px" becomes 10).
  • parseFloat(): Converts the beginning of the string into a decimal.

The BigInt (Modern)

For very, very large numbers that are bigger than 9 quadrillion, standard JavaScript numbers can lose accuracy. The newer **BigInt** type was added specifically to handle these massive values with perfect precision.

💡 Pro Tip: For financial calculation (like money), avoid using floating-point numbers directly. Convert all values to cents (integers) first, do your math, and then convert back to dollars for display. This prevents "mystery" cents appearing in your totals.
