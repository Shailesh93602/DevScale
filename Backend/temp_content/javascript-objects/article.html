JavaScript Objects: Organizing Complex Data

In JavaScript, an **Object** is a standalone entity with **Properties** (characteristics) and **Methods** (actions). While a simple variable stores one value, an object allows you to group together many related values in one place (e.g., a "User" object containing a name, email, and age).

1. Defining an Object

Objects are enclosed in curly braces {} and use **Key-Value pairs** to store data.

const user = {
  name: "EduScale",
  age: 2,
  isPremium: true
};

2. Accessing Properties

  • Dot Notation (user.name): The most common way to access values.
  • Bracket Notation (user["name"]): Useful when the key is stored in another variable.

3. Methods: Adding Actions to Objects

A "Method" is simply a function that belongs to an object. It's used to perform actions related specifically to that object.

const car = {
  brand: "Tesla",
  start: function() { console.log("Engines ready!"); }
};
car.start();

4. The this Keyword

Inside a method, the this keyword refers to the **current object** itself. It allows the method to access its own properties.

Constructors and Classes (Modern)

If you need to create many similar objects (like 100 different products), you use a **Class** as a "Blueprint" to define the structure once and generate objects (instances) from it.

💡 Pro Tip: Use **Objects** to group related data! Instead of having five separate variables for one user, put them in a single user object. This makes your code much cleaner and easier to pass around in functions.
