JavaScript Page Printing: Moving from Digital to Physical

In addition to using **Print-specific CSS**, JavaScript provides a simple way to trigger the browser's built-in print dialog directly. This allows you to add a "Print this receipt" or "Download as PDF" (via the print menu) button for your users.

1. window.print()

The core command! It's that simple. When this function is called, the browser opens its standard print preview window immediately.

<button onclick="window.print()">Print Page</button>

2. Print Events

JavaScript provides two specific events to help you handle printing:

  • beforeprint: This event triggers JUST before the print dialog opens. Use it to show hidden content or hide sidebars if you're not using @media print CSS.
  • afterprint: This event triggers AFTER the user closes the print dialog. Use it to restore your UI to its normal state.

3. Performance and User Experience

Never, EVER trigger a print dialog automatically when a page loads. This is extremely annoying and is a major violation of user experience (UX) best practices.

Modern Alternatives (PDF Libraries)

If you need a perfectly formatted document (like a formal invoice or a certificate), triggering the print menu may not be enough. In those cases, developers use JavaScript libraries like jsPDF or html2pdf to generate a custom PDF file directly in the browser.

💡 Pro Tip: Combine window.print() with @media print CSS. This allows you to create a "Print-Friendly" button that results in a perfectly formatted physical document for your users every time.
