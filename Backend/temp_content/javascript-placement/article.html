JavaScript Placement: Where to Put Your Code

Where you place your <script> tags in your HTML document can have a significant impact on your website's performance and functionality. There are three main ways to include JavaScript in your web pages.

1. Internal JavaScript (in the <head>)

Putting scripts in the <head> is usually not recommended unless they are absolutely required for the initial paint (like a theme-switching script). This is because the browser will stop parsing the HTML until the script finishes loading, which can cause a "blank screen" for the user.

2. Internal JavaScript (before </body>)

This is the traditional "best practice." Because the browser has already parsed and displayed all your HTML before it reaches the script, your page will feel much faster to the user.

3. External JavaScript (The modern standard)

For any real project, you should put your JavaScript in separate .js files. This keeps your HTML clean and allows the browser to multi-task and cache the files for faster subsequent page loads.

<script src="script.js"></script>

The async and defer Attributes

  • async: The script is downloaded in the background while the HTML is being parsed, and then executed immediately when it finishes. Best for independent scripts (like ads or analytics).
  • defer: The script is downloaded in the background and executed ONLY after the entire HTML has been parsed. This is almost always the best choice for your main application script.

💡 Pro Tip: Use **defer** for almost all your external scripts. It gives you the best of both worlds: your scripts start downloading early, but they won't block the visual rendering of your page for the user.
