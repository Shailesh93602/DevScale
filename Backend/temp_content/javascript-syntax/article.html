JavaScript Syntax: The Rules of the Language

Every language—whether it's English, Spanish, or JavaScript—has a set of rules for how to combine words and symbols into meaningful "sentences." These rules are called **Syntax**.

1. Case Sensitivity

JavaScript is **case-sensitive**. This means that myVariable and myvariable are seen as two completely different things by the browser.

2. Statements and Semicolons

A "statement" is a single command. While modern JavaScript can often "guess" where a statement ends, it's a best practice to end each one with a **Semicolon (;)** to avoid errors in complex code.

let x = 5;
let y = 10;

3. Comments (//)

Use comments to explain your code to yourself and others. These lines are completely ignored by the computer.

// This is a single-line comment.
/* This is a 
   multi-line comment. */

4. Variables (Keywords: let, const)

Variables are containers for storing data. You use let for variables that might change and const for values that will always stay the same (Constant).

5. Naming Conventions (CamelCase)

Most JavaScript developers use camelCase for variable and function names. The first word is lowercase, and every following word starts with a Capital letter (e.g., totalPrice, getUserName).

💡 Pro Tip: If your JavaScript isn't working, check the **Console** in your browser's DevTools. It will tell you the exact line number where you have a "Syntax Error" (like a missing parenthesis or a typo).
