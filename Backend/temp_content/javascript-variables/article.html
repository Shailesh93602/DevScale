JavaScript Variables: Storing Your Data

In programming, a **Variable** is a container for a value. You can think of it like a labeled box where you can store information (like a user's name, the price of an item, or the result of a calculation) and retrieve it later.

1. The Keywords: let, const, and var

  • let: The modern way to declare a variable that might change later (e.g., a counter or a user's score).
  • const: The modern way to declare a "Constant"—a value that will NEVER change (e.g., your API key or the value of Pi). Always use this by default!
  • var: The original way to declare variables. Today, it is mostly considered obsolete because it has confusing "scoping" rules. Avoid using it in modern projects.

2. Naming Your Variables

  • Must start with a letter, $, or _.
  • Cannot start with a number.
  • Cannot be a "Reserved Word" (like let, function, or class).
  • Use camelCase (e.g., userAge, totalPriceWithTax).

3. Data Types

JavaScript is "dynamically typed," which means you don't have to tell the variable what kind of data it's holding. It will figure it out automatically!

let score = 100; // Number
let playerName = "EduScale"; // String
let isActive = true; // Boolean

4. Variable Scope

The "Scope" of a variable determines where in your code it can be accessed. Variables declared with let and const have **Block Scope**, meaning they only exist inside the curly braces {} where they were created.

💡 Pro Tip: Use const for every variable you create unless you specifically know you're going to change its value later. This makes your code safer and much easier for other developers to read and debug.
