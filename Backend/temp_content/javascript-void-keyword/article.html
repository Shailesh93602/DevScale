JavaScript Void Keyword: Controlling Expression Results

The void operator is a specialized tool in JavaScript that evaluates an expression and then returns undefined. While not commonly used in modern application logic, it is very common in "Hacky" links and specific edge cases.

1. What does void do?

It "swallows" the result of an expression. For example, void(5 + 5) will return undefined instead of the expected 10.

2. The javascript:void(0) Link

Probably the most common use. It's used to create a link that looks and acts like a clickable element but DOES NOT navigate to a new page or refresh the current one.

<a href="javascript:void(0)" onclick="openModal()">Click Me</a>

3. Performance and Function Results

Some developers use void to ensure that a function returns undefined even if its last line happens to be a value. This is rare in professional modern code but good to recognize if you see it in legacy projects.

Modern Best Practice

Instead of javascript:void(0), modern web development recommends using a <button> element styled to look like a link, or using e.preventDefault() in your event listener. This is much better for accessibility and search engine optimization (SEO).

💡 Pro Tip: If you're building for accessibility, avoid **void(0)** links. A screen reader user expects a link to go somewhere. If it doesn't navigate, it should almost always be a <button> instead!
