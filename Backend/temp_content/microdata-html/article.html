Microdata and HTML: Communicating with the Semantic Web

While semantic tags like <article> and <nav> help search engines identify parts of a page, Microdata helps them understand the meaning of the data inside those tags. It allows you to label structured information like people, products, events, and reviews.

What is Microdata?

Microdata is a set of attributes introduced in HTML5 for nesting metadata within existing content on web pages. It is primarily used by search engines (via the Schema.org vocabulary) to create "Rich Snippets" in search results.

Core Microdata Attributes

  • itemscope: Defines the scope of an item (e.g., this div contains information about a person).
  • itemtype: Specifies the type of item using a URL from Schema.org (e.g., https://schema.org/Person).
  • itemprop: Identifies a specific property of the item (e.g., the person's name, email, or job title).

Example: Marking Up a Person

<div itemscope itemtype="https://schema.org/Person">
  <p>My name is <span itemprop="name">Jane Doe</span></p>
  <p>I am a <span itemprop="jobTitle">Web Developer</span></p>
</div>

Rich Snippets and SEO

By using microdata for product reviews, prices, and availability, search engines can display this information directly on the search results page. This "Rich Snippet" makes your link much more attractive to users and significantly increases your click-through rate.

Verification

You can test your microdata implementation using Google's Rich Results Test or the Schema Markup Validator.

💡 Pro Tip: Schema.org is a collaborative project between Google, Bing, and Yahoo. By using their vocabulary, you ensure your data is understood by all major search engines.
