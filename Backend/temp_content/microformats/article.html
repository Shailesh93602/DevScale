Microformats: Simpler Semantic Markup

While Microdata and RDFa are powerful, they can be complex to implement. **Microformats** are a simpler way to add structured data to your web pages using existing HTML classes. They allow software (like browser extensions or search engines) to "find" data like contact info, events, and reviews automatically.

How Microformats Work

Microformats leverage the class attribute to assign meaning to elements. Think of them as a "reused" naming convention that many different systems have agreed upon.

Popular Microformats

  • hCard: Used for representing people, companies, and organizations (based on the vCard standard).
  • hCalendar: Used for events and schedules (based on the iCalendar standard).
  • hReview: Used for publishing reviews of products, services, or businesses.

Example: Marking Up a Person (hCard)

<div class="vcard">
  <a class="url fn" href="https://example.com">Jane Doe</a>
  <div class="org">Example Corp</div>
  <div class="tel">555-1234</div>
</div>

Microformats vs. Microdata

Microformats were popular before HTML5 introduced formal Microdata. While still used today, modern developers often prefer Microdata (Schema.org) because it is more robust and explicitly supported by major search engines. However, Microformats remain the easiest way to add basic semantic meaning without extra tags.

The "Microformats2" Update

The latest version (Microformats2) simplifies the naming convention. Instead of relying on specific vCard strings, it uses a standardized prefix system: h-* for objects, p-* for properties, u-* for URLs, and dt-* for datetime.

💡 Pro Tip: Even if you don't use the full Microformat spec, following their naming conventions (like using class="tel" for phone numbers) is a good habit that makes your code more predictable for other developers.
