Progressive Enhancement and Graceful Degradation

These are two different philosophies for handling the varying capabilities of browsers and devices. Both aim to ensure that your website remains usable for as many people as possible, but they approach the problem from opposite directions.

1. Progressive Enhancement (Bottom-Up)

Progressive enhancement starts with a basic, functional experience that works in *all* browsers (even very old ones or text-only browsers). Then, you add layers of sophistication (CSS, JavaScript, advanced APIs) for browsers that support them.

  • Layer 1: Semantic HTML (Content & Structure - works everywhere).
  • Layer 2: CSS (Presentation - enhances the look in modern browsers).
  • Layer 3: JavaScript (Interaction - adds advanced features).

2. Graceful Degradation (Top-Down)

Graceful degradation starts by building the experience for the most modern browsers using all the latest features. Then, you ensure that the site "degrades gracefully" (still works, but with a simplified experience) in older browsers.

Which One Should You Use?

Modern developers generally prefer Progressive Enhancement. It is more robust because it ensures that even if your JavaScript fails to load or the user is on a slow connection, the essential content of your site is still available.

Feature Detection vs. Browser Sniffing

To implement these strategies, you should use **Feature Detection** (checking if a specific API exists) rather than "Browser Sniffing" (checking the user agent string). The @supports rule in CSS and the in operator in JavaScript are essential tools for this.

if ('serviceWorker' in navigator) {
  // Browser supports Service Workers!
}

💡 Pro Tip: Treat JavaScript as an enhancement, not a requirement. If your site’s core functionality (like reading an article or submitting a simple form) requires JavaScript to work at all, you are likely not following Progressive Enhancement.
