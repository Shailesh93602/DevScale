Server-Sent Events (SSE): Real-Time Updates Made Simple

Server-Sent Events (SSE) is a standard allowing servers to push real-time data to web pages over a single HTTP connection. Unlike WebSockets, which are bi-directional (two-way), SSE is **uni-directional** (one-way: server to client). This makes it ideal for live feeds, notifications, and stock tickers where the client doesn't need to talk back constantly.

How SSE Works

To use SSE, the client creates a new EventSource object pointing to a server endpoint. The server keeps the connection open and sends data in a specific text format (text/event-stream).

const source = new EventSource('/api/updates');
source.onmessage = (event) => {
  console.log("New data from server:", event.data);
};

The text/event-stream Format

Simple SSE messages look like this. Each message starts with data: and ends with two newlines (\n\n).

data: The weather is sunny!

data: New notification: You have 5 messages.

SSE vs. WebSockets

  • SSE: Uni-directional, works over standard HTTP, includes automatic reconnection, and is easier to implement for simple notifications.
  • WebSockets: Bi-directional, requires a custom protocol, and is better for high-frequency interactive apps like games or chat rooms.

Automatic Reconnection

One of the best features of SSE is that the browser handles reconnections automatically. If the connection drops, the browser will wait a few seconds and then try to reconnect without any extra JavaScript from the developer.

Browser Support

SSE is widely supported in all modern browsers (except for very old versions of Internet Explorer). For IE support, polyfills are readily available.

💡 Pro Tip: Because SSE uses standard HTTP, it works perfectly with existing load balancers and firewalls that might struggle with the custom protocol used by WebSockets.
